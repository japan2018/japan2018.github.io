[Python] Installation procedure of virtualenv and pythonz to Mountain Lion environment + usage

Oct 13, 2013 Python 

## 1.First of all

Mountain Lion comes with Python installed by default. However, if you want to use the latest Python environment but want to keep the Python that is installed as standard on the Mac, you cannot easily install the latest Python. This time, I would like to write about “virtualenv” and “pythonz” that can leave Python installed as standard on Mac and use various Python environments.

2. Prerequisites

  • Easy_install is installed

*) If easy_install is not installed, execute the following command.

$ curl -O https://svn.apache.org/repos/asf/oodt/tools/oodtsite.publisher/trunk/distribute_setup.py
$ sudo python distribute_setup.py

3. Installation procedure

(1) Install virtualenv using easy_install.

$ sudo easy_install virtualenv

(2) Execute the following command to install pythonz.

$ curl -kL https://raw.github.com/saghul/pythonz/master/pythonz-install | bash

(3) Add the following contents to .bashrc.

[[ -s $HOME/.pythonz/etc/bashrc ]] && source $HOME/.pythonz/etc/bashrc

(4) Reflect the modified .bashrc contents.

$ source ~/.bashrc

4. pythonz command collection

(1) cleanup

  • Delete the files and folders downloaded when you installed Python using pythonz.
$pythonz cleanup

(2) install

  • Install Python.
$pythonz install (version)

*) You can check the Python version that can be installed using the following command.

$ pythonz list -a

(3) list

  • Display Python installed and installable Python versions using pythonz.
$ pythonz list # Display currently installed Python version
$ pythonz list -p # show path of currently installed python
$ pythonz list -a # install pythonz to show Python version

(4) uninstall

  • Remove Python installed using pythonz.
$pythonz uninstall (version)

(5) update

  • Update pythonz to the latest version.
$ pythonz update

5. Build a Python virtual environment with virtualenv

(1) Create a working folder for virtualenv. This time, create a .virtualenv folder on your home directory.

$ mkdir ~/.virtualenv
$ cd ~/.virtualenv

(2) Use the virtualenv command to create a Python virtual environment.

#Basic command
$ virtualenv -p (installation path of Python executable file) (virtual environment name) # If you do not specify "-p", the Python version will be 2.5.
# Example of use
$ virtualenv -p ~/.pythonz/pythons/CPython-2.7.5/bin/python py27

(3) Start the Python virtual environment with the following command.

$ source py27/bin/activate # "py27" is the virtual environment name created above. Please replace it as appropriate.

(4) To terminate the Python virtual environment, execute the following command.

$ deactivate
