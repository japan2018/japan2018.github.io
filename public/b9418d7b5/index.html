1.01. Output and comment

Key Point

  • Python code can directly write the processing contents -There is no need to write a main function. You can write it, but you don’t have to write it for small-scale code like competition pros.
  • You can print a string with print("string")
  • By writing #, comments after # on that line become comments

Basic form of program

The following program is a program that does nothing.

In other words, unlike C++, there is no need to write a main function etc. The program works just by writing the processing contents directly. You can write a main function, but it’s okay if you don’t write it for small-scale programs such as competitive programming.

output

Let’s take a look at the program we submitted first. This program is a program that outputs the character string “Hello, world!”.

print("Hello, world!")

Hello, world!

print()

To print a string in Python, use print(). The part that specifies the output string is the part “Hello, world!". You can output a string by writing it in () of print(). When dealing with strings in Python programs, you need to enclose them in " ". Also, a line break is automatically added at the end of the sentence.

end of line

C++ requires a ; at the end of the line, but Python doesn’t require you to write anything. (Even if you write it, it works without any problem)

Output another string

If you want to output another character string, change the contents of " " as follows.

print ( "Hello World")

Hello World

Multiple outputs

You can also output multiple times.

print("a")
print("b")
print("c", "d")

a
b
c d

As I wrote earlier, Python’s print() automatically inserts a line break at the end of a sentence. Therefore, b is printed on the line following a. Next, write multiple lines separated by ,, and the output will be separated by spaces, like c d.

Numerical output

To output a numerical value, you can write it directly without using " ".

print(2525)

2525

Comment

Comments are for leaving notes to explain the behavior of the program, and writing them does not affect the behavior of the program. Let’s look at an example comment in Python.

print("Hello, world!") # Hello, world!
# Comment here

Hello, world!

By writing #, the comment after # on that line becomes a comment.

Double-byte character

Unlike C++, Python basically works without problems even if “double-byte characters” are included in the program. However, it is difficult to read, so it is recommended to write in “half-width characters”.

1.02. How to write programs and errors

Key Point

  • Python makes sense for spaces, line breaks and indentation
  • Python has a coding convention called “PEP8”
  • Python is an interpreted language, so no compilation errors occur

How to write a program

This section describes spaces, line breaks, and indents in the program. It also describes the Python coding standard, PEP8.

Spaces and line breaks

In Python, spaces can be omitted, but line breaks are significant and cannot be omitted.

In C++, ; means end of line; in Python, newline means end of line.

indent

Indentation is done in C++ for readability, but in Python it also makes sense.

Let’s take a look at the difference between C++ and Python using the if statement below as an example.

#include <bits/stdc++.h>
using namespace std;
 
int main() {
    if (conditional expression) {
        processing
    }
}

#include <bits/stdc++.h>
using namespace std;
 
int main() {
if (conditional expression) {
processing
}
}

For C++, there is no difference in behavior with or without indentation inside the { } as above.

if conditional expression:
    processing

Indentation is required because Python uses : and an indent instead of { } to express what is inside an if statement.

PEP8

PEP8 is a Python coding standard. Coding conventions are rules designed for readability.

The program works fine even if you do not follow PEP8, but you can write a program that is easy to read by doing so.

Here’s just a few of them, because introducing them all can be lengthy.

  • Indentation is 4 spaces
  • Maximum line length is 79 characters
  • Do not put extra space

Details can be found at https://pep8-ja.readthedocs.io/ja/latest/.

Program error

Basically it is as written in the original APG4b, but since Python is an interpreted language, no compilation error occurs.

Interpreted languages do not compile like C++ and are executed one by one. Therefore, if there is a syntax error, a runtime error will occur instead of a compile error.

if statement

By using if statement, you can write a program that processes **only when a certain condition is correct. The format is as follows.

if conditional expression:
    processing

When the conditional expression is correct, the process after the indent after the : is decreased by one, and when the conditional expression is incorrect, the process is skipped.

In the following example, if the input value is less than 10, then “x is less than 10” is output and then “end” is output. If the input value is not less than 10, only “End” is output.

x = int(input())

if x <10:
    print("x is less than 10")

print("end")

    print("x is less than 10")

Return statement behavior

The processing of the function ends when the processing reaches the return statement.

There is no problem if there are multiple return statements in one function, but be aware that the processing written after the return statement will not be executed.

In the following program, since the line that outputs ““Hello, “ is written as "return”, the subsequent processing is not executed.

def hello():
    print("Hello, ", end="")
    return # Only this line is executed

    print("world!")
    return


hello()

Hello,
