02. * and ** are attached to the parameter that receives the variable length argument

I saw some code that says *args or **kwargs in the function parameters. This is described in the parameter that receives the variable length argument, and the number of arguments passed to the parameter when calling the function can be set to an arbitrary number instead of a fixed number.

A variable length argument allows you to specify any number of arguments, where normally only one can be specified.

  • ~ Normally, you can only store one item per cardboard box. However, if you use variable length arguments, you can put as many items as you want per cardboard, and they will be packed in one bubble wrap. ~*
def kansu(*args):
    print(f'{args} received!')

kansu('Apple','Banana', 100)
Received # ('Apple','Banana', 100)!

For the parameter, only one args is passed, and three arguments are passed, but it is operating without error.

In familiar examples, variable length arguments are also used in the print() function and .format() method.

print('a','b','c') # a b c

Also, the names args and kwargs are customary, so any name you choose will work.

def x(*z):
    y = []
    for i in z:
        y.append(i*10)
    print(y)
    
x(1,2,3,4,5,6,7,8)
# [10, 20, 30, 40, 50, 60, 70, 80]

03. Difference between one * and two *

There are two types of variable length arguments, and the way to pass and receive the arguments are different.

  1. Parameters that accept variable length positional arguments <- *args Receive an arbitrary number of arguments as tuples (empty tuples if no arguments are passed) Pass by positional argument

-Pack all items in one bubble wrap and go to cardboard-

  1. Parameter that receives variable length keyword arguments <- *kwargs Receive an arbitrary number of arguments in the form of keyword=value in dictionary type (key value) (If you do not pass the argument, it will be an empty dictionary) Pass with keyword argument

-After naming all items with post-it, packing all items with one bubble wrap and going to cardboard-

04. How to use parameters that accept variable length position arguments

By adding * to the argument, the parameter will receive the variable length positional argument. That is, it accepts any number of arguments. How you use the function depends on where you define the variable-length positional argument in the function’s parameters.

def myfunction(x, y, *z):
    print(f'{x}-{y}-{z}')

myfunction(2, 3, 5, 6, 7) # 2-3-(5, 6, 7)

The first and second arguments correspond to x and y respectively. Subsequent arguments correspond to z and are stored as tuples.

def myfunction(x, *y, z):
    print(f'{x}-{y}-{z}')

myfunction(2, 3, 5, 6, 7) # TypeError: myfunction() missing 1 required keyword-only argument:'z'

It throws an error because no argument is passed to parameter z. In this case, z is a keyword-only argument, so you must set a default value or use it as a keyword argument when calling.

def myfunction(x, *y, z=0):
    print(f'{x}-{y}-{z}')

myfunction(2, 3, 5, 6, 7) # 2-(3, 5, 6, 7)-0

def myfunction(x, *y, z):
    print(f'{x}-{y}-{z}')

myfunction(2, 3, 5, 6, 7, z=0) # 2-(3, 5, 6, 7)-0

05. Using variable length keyword arguments

By adding ** to the argument, the parameter receives a variable length keyword argument. That is, any number of keyword=value is received, and the keyword and argument correspond to the dictionary type key and value, respectively. How you use the function depends on where you define ** in the function’s parameters.

def myfunction(param1, param2, **param):
    print(f'{param1} | {param2} | {param}')

myfunction(2, 3, param3=4, param4=5, param5=6) # 2 | 3 | {'param3': 4,'param4': 5,'param5': 6}

The value obtained by the variable length keyword argument becomes a dictionary type.

def myfunction(param1, **param, param2):
    print(f'{param1} | {param2} | {param}')

The result is a syntax error Variable length keyword arguments cannot be defined in the middle.

06. When using two variable length arguments together

Normal parameters placed after parameters that accept variable length positional arguments become keyword-only arguments.

def Test(a, *b, c): # c is a keyword only
  pass

Normal parameters cannot follow a parameter that accepts variable length keyword arguments.

def myfunction(param1, **param, param2):
    print(f'{param1} | {param2} | {param}')

Therefore, when they are used together, the order should be as follows.

1. A parameter that receives both positional and keyword arguments
2. Parameter that receives variable length positional argument
3. Parameter that receives the keyword-only argument
4. Parameter that receives variable length keyword arguments
