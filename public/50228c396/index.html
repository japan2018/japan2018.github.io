## debug() instead of print()

Although print() is the output method used in Python, in TouchDesigner you can use the function debug() instead of where print() is available. The debug() can see the output as well as where and on which line of which operator it was called. This is useful in most cases when debugging with Touch Designer.

CrashAutoSave for files that cannot be opened or have initial failure

Sometimes I write an infinite loop, or something I don’t know causes the file to crash or not open. When a crash occurs, CrashAutoSave.toe is automatically generated, but you can also start your toe file in safe mode by adding CrashAutoSave. to the beginning of the file name. If you start it in safe mode, you can check and modify the contents without running any process without a project. After correcting the contents, you can fix the file that could not be opened by removing CrashAutoSave. from the file name and saving and reopening the file.

When you open a file with CrashutoSave., the dialog shown below appears first.

When you press OK and Touch Designer starts up, the timeline does not run and the word Safe Mode appears in the center of the screen. In this state, you can edit the operator etc., so you can fix the defective part or take out only the necessary part and copy and paste it to another file.

Open in non-Commercial mode

For those who do not have a license at a workshop etc., it can be used to check whether Non-Commercial also works when distributing sample files etc. Since it is not possible to change the licenses quickly, it is easy to put restrictions from the script.

app.addNonCommercialLimit('password') # limit
app.removeNonCommercialLimit('password') # unlock

app.addResolutionLimit(1280, 720,'secret123') # limit
app.removeResolutionLimit('secret123') # release
