I usually use C# to develop medical systems. I decided to focus on machine learning, so I started learning python.

Even though I learned about it, I went around Progate’s python course, but I will briefly summarize the differences between C# and how to write.

If you usually use C#, C++, VB and want to learn python from now on, I think that learning will be better if you know the difference between the two in advance.

Simple difference between C# and python seen from code

No semicolon at end of sentence

Console.WriteLine("Hello C#"); //be sure to end the statement with a semicolon

print("Hello python")

** Not required if the semicolon at the end of the statement is python. ** First of all, I was not used to this when using a language that normally requires a semicolon, such as C#.

Variables do not need to be typed

string[] animals = {"dog","cat"}; // all variables specify type

animals = ["dog","cat"]

**python is a “dynamically typed language”, and there is no need to specify the data type when declaring variables. **

Even in statically typed languages such as *C#, it is possible to infer the type of data that actually enters (type inference) by using the var keyword, but the data type is not specified when reading the code. Feels a little difficult. *

When I write code, I also feel that it seems easy.

**By the way, python doesn’t specify the return value or argument of the function. **

No curly braces required, indentation affects code

foreach(string animal in animals)
  {
    Console.WriteLine(animal);
  }

for animal in animals:
    print(animal)

Processing to output the contents of the array in the code As you can see by looking at (foreach in C#, for part in python), **python distinguishes the processing after conditional expression by: (colon) and indent. ** The same applies to if statements.

It’s also interesting that the presence or absence of indentation affects movement. This makes it easy for anyone to write.

List elements can be of any type

string[] animals = {"dog","cat"}; // all variables specify type

animals = ["dog","cat",1,2]

A list is an array in C#. ** It is a feature that C# does not have, that you can mix anything such as string, number type, etc. in one list element. **

*C# also has a collection called ArrayList, which can contain any element type. However, a cast (type conversion) is required when extracting. *

By the way, in python, the process to retrieve the contents of an array is described as for. Processing is different from the C# for statement. This area may be complicated.
