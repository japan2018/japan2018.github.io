40-32/2=4!

Jun 4, 2020 Python math 

I think I saw it about 10 years ago...

Let’s verify this as the subject.

Brute force

First of all, it will be exhausted by brain death. You will have the following loop.

from math import factorial

for a in range(1,100):
    for b in range(1,a):
        for c in range(1,100):
            if (a-b)%c != 0:
                continue
            d = int((a-b)/c)
            if a-b/c == factorial(d):
                print("{}-{}/{}={}!".format(a,b,c,d))

When $c=1$, $a-b/c$ and $(a-b)/c$ have the same value. In addition, when $d=1,2$, $d! = d$, so it can be seen that the objective is satisfied for any $a,b=a-1 or a-2$. …But this isn’t what Tonchi really meant.

Brute force 2

▽ We will tighten the conditions a little. Respect the subject as $c \geq 2,d \geq 3$.

from math import factorial

for a in range(1,1000):

It has become more like Tonchi. Probably infinite, but no proof has been given. At a glance, I’m wondering if there is a case where $3!$ is the answer. Also, since the amount of calculation has increased, we would like to have an algorithm that can generate $a,b,c,d$ in a shorter time.

#Mathematical considerations

$$a-b/c = d!$$$$(a-b)/c = d$$

This can be a matrix like the one below.

\begin{pmatrix}
c & -1 \\
1 & -1
\end{pmatrix}
\times
\begin{pmatrix}
a \\
b
\end{pmatrix}
=
c
\begin{pmatrix}
d! \\
d
\end{pmatrix}

I will solve it.

\begin{pmatrix}
a \\
b
\end{pmatrix}
=
c
\begin{pmatrix}
c & -1 \\
1 & -1
\end{pmatrix}^{-1}
\times
\begin{pmatrix}
d! \\
d
\end{pmatrix}

\begin{pmatrix}
a \\
b
\end{pmatrix}
=
c
\cdot
\frac{1}{-c+1}
\begin{pmatrix}
- 1 & 1 \\
- 1 & c
\end{pmatrix}
\times
\begin{pmatrix}
d! \\

\end{pmatrix}

\begin{pmatrix}
a \\

\end{pmatrix}
=
\frac{c}{1-c}
\begin{pmatrix}
- d! + d \\
- d! + cd
\end{pmatrix}

\begin{pmatrix}

\end{pmatrix}
=
\frac{c}{c-1}
\begin{pmatrix}

\end{pmatrix}

\begin{pmatrix}

\end{pmatrix}
=
\frac{cd}{c-1}
\begin{pmatrix}
(d-1)!-1 \\
(d-1)!-c
\end{pmatrix}

▽ Of these, the combination of $c,d$ where $a,b$ is an integer is the condition of interest.

When #d=3

Consider the case when $d=3$. At this time, the condition is $$a = \frac{3c}{c-1}$$$$b = \frac{3c}{c-1}(2!-c)$$, but $c \ From geq 2$, we can see that $b$ cannot be a positive integer. Therefore, $d=3$ may not satisfy the subject.

Simple general solution

When you want $\frac{cd}{c-1}$ to be an integer for a certain $d$, you can see that at least $c=d+1$ satisfies it. So if you fix c, the condition is $$a = (d+1) \Bigl\{(d-1)!-1\Bigr\}$$$$b = (d+1) \Bigl It will be \{ (d-1)!-d-1\Bigr\}$$.

Since d can take any integer of 4 or more, it has been shown that there are innumerable $a,b,c,d$ that satisfy the subject.

Next, let’s enumerate by program.

from math import factorial

for d in range(4,100):
    c = d+1
    a = c*(factorial(d-1)-1)
    b = c*(factorial(d-1)-c)
    print("{}-{}/{}={}!".format(a,b,c,d))
