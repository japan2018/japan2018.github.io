I made Django’s Middleware and made it possible to access request information from anywhere

Mar 10, 2020 Python Django middleware request 

I started thinking about whether the login user could be stored in the user_created (creator) and user_updated (updater) of the model by common processing.

Until then, I wrote the process to pass user information from view to form and store it in user_created or user_updated.

What is Django middleware?

From Official

Middleware is a framework for adding hooks to Django’s request/response handling.

You can define the process to be performed when sending a request or before returning a response.

middleware implementation

Define a function to operate threadlocal

from threading import local


THREAD_LOCAL = local()


def set_thread_variable(key, value):
    """ Store value in thread local data
    """
    setattr(THREAD_LOCAL, key, value)


def get_thread_variable(key, default=None):
    """ Get value from thread-local data
    """
    return getattr(THREAD_LOCAL, key, default)


def get_request():
    """ Get request information from thread local data
    """
    return get_thread_variable('request')

define middleware class

from common.middlewares.threadlocals import set_thread_variable
from django.utils.deprecation import MiddlewareMixin


class RequestStoreMiddleware(MiddlewareMixin):
    def process_request(self, reuqest):
        set_thread_variable('request', reuqest)

Since I want request information this time, I will define only process_request.

Just call get_request wherever you like

def set_user_receiver(sender, instance, *args, **kwargs)
    request = get_request() # It may be possible to perform authentication check before storing user information
    instance.created_user = request.user
    instance.updated_user = request.user


pre_save.connect(set_user_receiver, sender=Company) # pre_save can be used to register the function you want to call before executing save().
