## Variables may not be value entities

When dealing with specific types such as list and dictionary, variables refer to entities, rather than value entities. The properties of variables can be confirmed from the execution results of the following programs.

l1 = []
l1.append(0)

l2 = l1
l2.append(1)

print('l1 =',l1)
print('l2 =',l2)

Execution result

l1 = [0, 1] l2 = [0, 1]

After assigning the list l1 to the list l2, the value was added only to l2, but the same value is added to l1. This is because both l1 and l2 are reference information, and the entities at the end of the reference share the same thing.

Copy the entity itself

Use the copy() method to solve the problem that the value itself is not copied just by making an assignment.

l1 = []
l1.append(0)

l3 = l1.copy()
l3.append(1)

print('l1 =',l1)
print('l3 =',l3)

Execution result

l1 = [0] l3 = [0, 1]

However, even if you use the copy() method, if the element of the list is a reference, it will not be copied to the reference destination.

l4 = [[]]
l4[0].append(0)

l5 = l4.copy()
l5[0].append(1)

print('l4 =',l4)
print('l5 =',l5)

Execution result

l4 = [[0, 1]] l5 = [[0, 1]]

If you want to copy all referenced elements of the list, use the deepcopy() method included in the copy package.

import copy

l4 = [[]]
l4[0].append(0)

l6 = copy.deepcopy(l4)
l6[0].append(1)

print('l4 =',l4)
print('l6 =',l6)

Execution result

l4 = [[0]] l6 = [[0, 1]]
