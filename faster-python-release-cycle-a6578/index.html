Python that was released as 0.9.0 in 1991 also overlapped versions, 2.0 in 2000, 3.0 in 2008, Python 2 and Python 3 are developing in parallel, at the moment 2.7.17 and 3.8.0 Is coming up. I will talk about it, so I will write about it a little.

The original story is a PEP (Python Enhancement Proposal) called PEP 602 –Annual Release Cycle for Python.

Previous release cycle

Exporting 3.1.0 or later, where the release interval has stabilized, is like this.

Version Release Date
3.1 2009-06-27
3.2 2011-02-20
3.3 2012-09-29
3.4 2014-03-16
3.5 2015-09-13
3.6 2016-12-23
3.7 2018-06-27
3.8 2019-10-14

Although there are some places around, it seems that it is released at a cycle of about one and a half years when I look at it. And the development of one version started when the previous version β came out, so it took almost two years since the development started as a whole. This is a bit too long, so a new release cycle proposal has been proposed and adopted.

New release cycle

First of all, the main feature of this release cycle is “every year we release a new version of Python”. Every year in October, a new version is released with a cycle of one and a half years until now. Therefore, the scheduling is like this.

Phase Period
Development 5 months (overlap with previous β/RC period)
α version 7 months
β version 3 months
RC version 2 months

Development overlaps with the previous version, and the period of α + β + RC is exactly 12 months, so it can be released every year. In other words, as soon as the official version of the previous version comes out, the next version α comes out! Up until now, it had been more than half a year since the previous release, so it’s a great thing.

And the support period after the release is clearly decided like this.

  • 18 months (1 and a half years) after the release, minor bugs will be updated once every 2 months with bug fixes. Binary release for Windows and Mac.
  • For the next 42 months (3 and a half years), we will make security fixes as necessary. Source code release only.
  • Support ends 5 years after release.

Python 3.9 looks like this

The schedule for Python 3.9 has already been decided on PEP-596, and it looks like this.

3.9 Start of development: 2019-06-04 (Completed) 3.9.0 alpha 1: 2019-11-19 (completed) 3.9.0 alpha 2: 2019-12-16 3.9.0 alpha 3: 2020-01-14 3.9.0 alpha 4: 2020-02-17 3.9.0 alpha 5: 2020-03-16 3.9.0 alpha 6: 2020-04-13 3.9.0 beta 1: 2020-05-18 (No new features added after this) 3.9.0 beta 2: 2020-06-08 3.9.0 beta 3: 2020-06-29 3.9.0 beta 4: 2020-07-20 3.9.0 candidate 1: 2020-08-10 3.9.0 candidate 2: 2020-09-14 3.9.0 final: 2020-10-05

As you can see, the first alpha version is already out. Up to now, there were only 4 alpha versions, but now there are 6 which seems to make up for the shortened development period. I think it’s a good change that it has become an agile approach, where features are gradually added while being released as an alpha version and used.

Summary

I have summarized the new release cycle introduced in Python 3.9. I think it was good that the cycle became shorter and new functions could be used in a timely manner.
