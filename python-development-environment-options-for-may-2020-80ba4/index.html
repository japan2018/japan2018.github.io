May 26, 2020 Python VSCode 

#Introduction

Python is attractive because you can write and run it immediately after installing it. If you really want to write it in business

  • Python version control tool
  • Package manager
  • Editor (IDE)
  • Linter
  • Formatter
  • Type checker

At least you need to prepare. However, in this area, new things are released every day due to the rage, so it is difficult to unequivocally present “this is the best practice”. So, this time, I would like to introduce that there are such things for each of the above tools.

TLDR

  • Version/Package management is over when pyenv + Pipenv was the standard
  • If you use VS Code or Vim, introduce Pyright for type analysis
  • I prepared a template

1. Version/Package Manager

It is a tool to use different Python versions and packages for each project. For a long time, pyenv + Pipenv has been said to be the standard of a new era, but now it is back in the age of warriors.

  • pyenv + Pipenv
  • pyenv + Poetry
  • pyenv + DepHell
  • pythonloc
  • Pyflow
  • Such…

Do you think the top two Japanese documents are substantial? Personally I like pyenv + Poetry. For more details, please read this article as it is wonderful.

2020 Python Package Management Best Practices

In Python 3.8, there is a proposal like PEP 582 “Can’t you stop virtual environment?” It is expected that management around here will become easier if this is officially incorporated. Light on Windows (NT kernel).

2. Editor (IDE)

It is a memo pad with many useful functions for writing codes. I can’t write the choices, so I will omit them. We recommend VS Code unless you have a particular preference.

  • Code completion works
  • Can be debugged
  • Linter/Formatter/Type checker works

Any editor can be used as long as it meets the three requirements. Conversely, let’s stop using editors that do not meet these requirements (such as XX and XX) in business.

3. Linter

It is a tool to check whether the written code is against the coding rules. I think this is an option to customize flake8 according to your formatter. If you know other good ones, please let me know.

In addition, flake8 has to update pycodestyle from GitHub until about one month ago due to a dependency package problem There was an error in Walrus operator, but it was solved when I checked it now.

4. Formatter

It is a tool that automatically formats the written code according to the coding rules. I used to think that it was like using autopep8, but now

  • autopep8
  • yapf
  • black

And there are three options. For details on each feature, see the following articles.

**It’s decided! 3 strongest automatic code shaping tools! **

I’m using yapf because the black format wasn’t accepted for certain religious reasons.

5. Type checker

A tool for writing Python like a statically typed language. The choices are

  • If you use VS Code or Vim as the editor Pyright
  • Mypy if not used
  • If you can not stand the slowness of mypy Pyre

Is it three? I don’t know how to use pytype…

Pyright is faster than mypy (MS says it’s 5x) and can perform type checking on large projects without stress. To be honest, this is good enough to use VS Code. mypy works with Vim, Emacs, Sublime Text, Atom, PyCharm and VS Code. Like Pyright, Pyre is a type checker that aims to move faster than mypy, The usage is different from others (it notifies the server of file changes), so be sure to read the official documentation.
