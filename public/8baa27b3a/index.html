What is bleak?

A library for BLE communication Python that runs on Windows 10/Linux/macOS.

  • https://pypi.org/project/bleak/
  • https://github.com/hbldh/bleak

There are various python libraries for BLE communication, but I chose bleak as the one that can (almost) move in the same way on 3 types of OS, Windows 10, Linux (raspbian, ubuntu), and macOS.

How bleak works

Bleak is implemented by wrapping the BLE communication API of each OS.

  • On Windows 10, I call the dll using the common language runtime (CLR) and call the UWP API in it.
  • Linux calls the bluez stack.
  • On macOS, pyobjc is used to call Core Bluetooth API of Objective-C

Because the BLE communication part of each OS is hidden, it is made into a python library, so that it is possible to perform BLE communication with (almost) the same python script on any OS.

check the BLE device address of the toio core cube

This is [bleak sample code discover.py](https://github.com/hbldh/bleak/blob/master/examples/discover.py That’s it.

import asyncio

from bleak import discover

async def run():
    devices = await discover()
    for d in devices:
        print(d)

loop = asyncio.get_event_loop()
loop.run_until_complete(run())

The device address is a 6-byte hexadecimal number that is called toio Core Cube. Make a note of this value. You will set this value later in the script. In the execution example above, it is “D0:8B:7F:12:34:56”. For macOS, it is not a 6-byte hexadecimal number, but a long UUID string such as 243E23AE-4A99-406C-B317-18F1BD7B4CBE.

toio Read the battery level of the core cube, turn on the LED, and turn the motor

To read the remaining battery capacity of the toio core cube, connect to the oio core cube and then read the characteristic of the battery (Battery Information) (10B20108-5B3B-4571-9508-CF3EFCD7BBAE). To light the LED, write to the characteristic (10B20103-5B3B-4571-9508-CF3EFCD7BBAE) of the lamp (Light Control). To move the motor, write a value to the character list (10B20102-5B3B-4571-9508-CF3EFCD7BBAE) of the motor (Motor Control).

Let’s write the values referring to the examples of “lamp” and “motor” in the communication specifications of the toio core cube technical specifications.

  • Communication specifications of toio core cube technical specifications -Battery -Lamp -Motor
import asyncio
import platform

from bleak import BleakClient

#Characteristics of battery, LED and motor
BATTERY_CHARACTERISTIC_UUID = ("10b20108-5b3b-4571-9508-cf3efcd7bbae")
LAMP_CHARACTERISTIC_UUID = ("10b20103-5b3b-4571-9508-cf3efcd7bbae")
MOTOR_CHARACTERISTIC_UUID = ("10b20102-5b3b-4571-9508-cf3efcd7bbae")
#bleak seems to be useless unless the characteristic UUID is written in lower case

async def run(address, loop):
    async with BleakClient(address, loop=loop) as client:
        x = await client.is_connected()
        #logger.info("Connected: {0}".format(x))
        print("Connected: {0}".format(x))
        # Battery level reading
        battery = await client.read_gatt_char(BATTERY_CHARACTERISTIC_UUID)
        print("battery: {0}".format(int(battery[0])))
        #LED lights red for 160 ms
        write_value = bytearray(b'\x03\x10\x01\x01\xff\x00\x00')
        await client.write_gatt_char(LAMP_CHARACTERISTIC_UUID, write_value)
        # Motor Left 100 front speed, right right 20 speed
        write_value = bytearray(b'\x01\x01\x01\x64\x02\x02\x14')await client.write_gatt_char(MOTOR_CHARACTERISTIC_UUID, write_value)
        # Finish in 5 seconds
        await asyncio.sleep(5.0, loop=loop)

if __name__ == "__main__":
    address = (
        Set the device address of toio Core Cube found in # discovery.py here
        "D0:8B:7F:12:34:56" #Specify a hexadecimal 6-byte device address when using Windows or Linux
        if platform.system() != "Darwin"
        else "243E23AE-4A99-406C-B317-18F1BD7B4CBE" # UUID attached to macOS when macOS
    )

toio Notify the button state of the core cube and the ID read by the reading sensor

To notify (to notify) the button status of the toio core cube and the ID read by the reading sensor, the characteristic of the button (Button Information) (10B20107-5B3B-4571-9508-CF3EFCD7BBAE) and the reading sensor (ID Information) Get notified of the characteristic (10B20101-5B3B-4571-9508-CF3EFCD7BBAE).

For this, write the following script referring to bleak sample code enable_notifications.py.

import asyncio
import platform

from bleak import BleakClient

# Button and reading sensor characteristic
BUTTON_CHARACTERISTIC_UUID = ("10b20107-5b3b-4571-9508-cf3efcd7bbae")
ID_READER_CHARACTERISTIC_UUID = ("10b20101-5b3b-4571-9508-cf3efcd7bbae")
#bleak seems to be useless unless the characteristic UUID is written in lower case

#Button state notification handler
def button_notification_handler(sender, data):
    print("BUTTON {0}: {1}".format(sender, data))

#Read sensor notification handler
def id_reader_notification_handler(sender, data):
    print("toio ID {0}: {1}".format(sender, data))

async def run(address, loop):
    async with BleakClient(address, loop=loop) as client:
        x = await client.is_connected()
        print("Connected: {0}".format(x))
        # Button state notification start
        await client.start_notify(BUTTON_CHARACTERISTIC_UUID, button_notification_handler)
        # Read sensor notification start
        await client.start_notify(ID_READER_CHARACTERISTIC_UUID, id_reader_notification_handler)
        # Finish in 10 seconds
        await asyncio.sleep(10.0, loop=loop)
        # Button state, reading sensor notification end
        await client.stop_notify(BUTTON_CHARACTERISTIC_UUID)
        await client.stop_notify(ID_READER_CHARACTERISTIC_UUID)

if __name__ == "__main__":
    address = (
        Set the device address of toio Core Cube found in # discovery.py here
        "D0:8B:7F:12:34:56" #Specify a hexadecimal 6-byte device address when using Windows or Linux
        if platform.system() != "Darwin"
        else "243E23AE-4A99-406C-B317-18F1BD7B4CBE" # UUID attached to macOS when macOS
    )
    loop = asyncio.get_event_loop()
    loop.run_until_complete(run(address, loop))
