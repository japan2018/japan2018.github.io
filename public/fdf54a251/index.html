[Python] Use Bokeh with IPython Notebook

Dec 17, 2014 Python 

You can use [Bokeh](http://bokeh.pydata.org/) to create interactive graphs. With HTML output, integration with IPython Notebook and more, you will be able to represent data graphically in Python alone.

You can refer to the gallery on the official website and the notebook below for usage.

The main developers of Bokeh are Continuum Analytics who provide Anaconda etc. You can also see the author’s presentation on YouTube. It is about 50 minutes long, but it is also helpful as there are architectural topics such as BokehJS.

  • Bokeh: Interactive Web Plots & Dashboards-YouTube

Last week, the latest version was released. The content is described in the release article-Bokeh 0.7 Released!. It is not stable yet, such as adding new features and changing APIs, but I think it is very useful when you want to pursue data exploratoryly. Recently, implementations of Scala and Julia have also been added, and it seems that it can be used in implementations other than Python.

For Python, you can easily install it with pip. However, I think that it is often used in combination with NumPy and Pandas, so I added it to the Docker image. You can use skitazaki/python34-ipython along with Bokeh for some scientific computing libraries in IPython Notebook. Since functions such as zooming are effective in the graph part, it can be said that it is an advantage over a static graph to be able to roughly plot and then adjust. It is also possible to download it as an image when the display is just right by zooming. It can be a useful tool for visually checking affordable datasets.
