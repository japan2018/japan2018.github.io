[Python] About Python sort() and reverse()

Sep 6, 2016 Python 

Python's List has functions that change the order, such as sort() and reverse(), but if you use these, you'll be in trouble if you don't understand the specifications.

I am addicted to the trouble the other day, so I will leave it.

Return value is None

For example, like any other function,

print([1,3,8,7,4].sort())

Even if you write, None is displayed.

In the same way

print([1,3,8,7,4].reverse())

But the result is the same.

Read Python documentation

sort()

For the time being, when I read the Python documentation, I found something like this:

This method turns a sequence into place, to save space when sorting a large sequence. As a side effect, it does not return a sorted sequence, to remind the user that this operation is done. http://docs.python.jp/3/library/stdtypes.html#list.sort

This means that sort() will only make a breaking change, not a sorted list.

Continue to the documentation

Use sorted() to explicitly request a new list instance

There is.

That is,

l = [1,3,8,7,4]
l.sort()
print(l)

Or write

print(sorted([1,3,8,7,4]))

Should be written

reverse()

reverse() also read the document,

The reverse() method turns a sequence in-place to save space when reversing a large sequence. This does not return the inverted sequence, to remind the user that this operation is done as a side effect http://docs.python.jp/3/library/stdtypes.html#mutable-sequence-types

There is. In other words, like sort(), you need to put it in a variable once, make a destructive change, and then print.

Then, one question comes up.

Is there no reversed() that returns a reversed list? When.

In conclusion, the reversed() function itself exists.

However, unlike sorted(), which returns a sorted list, reversed() returns an iterator of the given list in reverse order.

Therefore, since it is necessary to generate the list from the iterator,

print([i for i in reversed([1,3,8,7,4])]))

And you need to.

But it’s much easier to write if you just flip the list.

print([1,3,8,7,4][::-1])

[::-1] of [1,3,8,7,4][::-1] is called a slice. I will omit the details, but a function to extract a part from a list and create a new list. is.

You can specify three values separated by colons (or you can omit them), the first is the point to start the extraction, the second is the point to end the extraction, and the third is the extraction interval.

This time, the interval is -1, so it is incremented by 1 and extracted (= all are extracted), and since there is a minus, extraction will be started from the end of the list.

So the list in reverse order comes back.

As the proper use of the list inversion method,

I want to save memory → reverse()

I want to add some processing to each value while making an inverted list (double, etc.) → reversed()

I want to flip it for the time being → Slice

I think you can use it properly.

#Summary

If you’re not sure what it does, read the documentation.
