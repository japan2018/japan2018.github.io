# Where is it

Here Please use it. Since it uses a GPU, the processing speed is quite fast. The best Google that lends GPU for free! !

#How to use

If you get an error, restart the runtime and run the cells one by one from the preparation section. I think most errors will be resolved by doing so. If you get a network error when downloading, please execute the cell again. It seems to occur depending on the timing of file generation. If you change your browser to Chrome, you may not get the error.

Preparation

Load the program

First, open this note in playground mode. It is at the top of the screen. Next, execute (ctrl+enter) all the cells in the “Preparation” chapter. Then, execute (ctrl+enter) the cell in the “Load module/method” chapter.

You can also execute by clicking [] in each cell as shown in the picture below.

Song preparation

If you execute the cell of “Upload songs to divide from here”, the following message will be displayed. Click Browse to upload any sound source. __ Please re-execute this cell every time even when uploading multiple sound sources. __ When the image below appears, the file has been uploaded.

Also, if you want to upload multiple sound sources at the same time, click the “Upload button” in the folder displayed in “File” on the left tab, or drag and drop to the tab to upload the file. is.

__ When you add a song, in order to reflect the song on the viewing etc., please re-read the cell for viewing etc. again. __

Viewing

To watch, run the cell in the Watch section. After a while, the following is displayed and you can watch it.

Split and download songs

Combine and download each part

Known bugs

There is a bug that Download and viewing are sometimes executed twice. Once you restart the runtime, it will recover. However, it is left as it is because there is no problem in practice if the root cause is unknown. Please let me know if anyone knows the cause. Probably because the handling of scope is bad, I think it is happening. I want to study and fix it.
