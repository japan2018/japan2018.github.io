[Python] Data analysis Appropriate use of data analysis tools for beginners

Dec 24, 2019 Python data analysis data visualization BI data science 

# Purpose of this article and target audience

There are many tools for data analysis. Here, data analysis tools include Excel, programming, dashboard tools, BI tools, etc.

In this article, for those who are new to data analysis and those who are going to do data analysis, we will explain what kind of analysis tool should be used for what kind of situation.

The most important reason to decide what kind of analysis tool to use in what situation is that the right tool can make data analysis efficient. Where programming isn’t necessary, you may not need to force it, In some situations, a handy BI tool is useless and may require some programming or SQL knowledge.

Also, the person who is going to analyze the data will run into the question of how much knowledge they should have for the analysis they want (or are entrusted to). At that time, if you have some idea of what kind of analysis tool to use for what situation, you can predict how much knowledge you will need.

Analysis tools targeted by this article and their features

Data analysis classification

There are two main types of data analysis: “ad hoc analysis” and “regular analysis”.

For companies and research that do not have a well-established data analysis platform and are not automated, they will manually analyze the data each time it is needed. This is called an ad hoc analysis in the sense of an ad hoc analysis.

Ad hoc analysis uses interactive analysis tools that allow you to execute queries and data transformation/aggregation processing and immediately check the results.

On the other hand, a dashboard tool that automatically aggregates and displays data to some extent is used when regular reporting is required rather than manual analysis each time.

What Excel is not good at

Excel is a very useful tool, but there are limits to data integration. Although increasing the number of columns in the horizontal direction is an Excel-like process, it may be difficult to integrate data in the vertical direction.

For example, consider the case of handling time series data from a sensor. (When abstracted, Google Analytics is also like a sensor, right? lol)

Suppose sensor (1) records data every 1 second and sensor (2) records data every 10 seconds. When collecting and integrating these data, the data of the sensor ① should be averaged every 10 seconds, and the representative value should be fetched and merged.

It is a hassle to do these processing in Excel, and if the number of data reaches hundreds of thousands or millions, Excel will slow down the calculation.

Although, For example, Excel may be sufficient when dealing with management indicators. At best, we only discuss the results by counting the weekly and monthly results. Therefore, consulting companies and accounting firms still recognize that Excel can be used as an important skill.

What BI tools cannot do

BI tools are great and very valuable as they promote democratization of data.

However, even if BI tools can handle integrated data, they are not good at (they can do) the integration itself. You must write your own SQL queries for data aggregation and integration before using BI tools.

So, in the end, for a meaningful analysis, you need a data preprocessing team that integrates the data and makes it easy for BI tools to read.

Dashboard Tool

In contrast to ad hoc analysis, dashboard tools are used for the purposes of regularly querying and creating reports and charts.

#Summary For companies that have a solid data analysis platform and can afford to properly aggregate and integrate data, it is advisable to introduce BI tools and visualization tools. If members other than engineers want to analyze data and utilize it in their daily discussions, they have no choice but to introduce BI tools and visualization tools.

However, if the KPI you want to see has not been set, or if you are not sure what to consider, it is a good idea to use BI tools to discuss it with everyone.

For companies that have already standardized the flow of consideration from data analysis, it would be better to use a visualization tool to automate the analysis.

If the data analysis infrastructure is not yet in place and you are just accumulating data for the time being, do ad hoc analysis using a programming language By doing a proper ad hoc analysis here, it becomes clear what KPIs are important to the company and what you want to consider using the data. This is a very important step.

Even if you suddenly automate an analysis, it may end up being obsolete. Since it takes time and money to set up the data analysis base, it is absolutely not good to start creating the analysis base without thinking deeply.
