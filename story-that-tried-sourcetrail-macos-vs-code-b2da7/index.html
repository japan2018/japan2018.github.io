Story that tried Sourcetrail × macOS × VS Code

Nov 26, 2019 Python VSCode macOSCatalina sourcetrail 

## Introduction

I will try it immediately because the code visualization tool called Sourcetrail has been made open source. It looks interesting.

What is Source Trail

@ IT’s article (Sourcetrail, a code visualization tool for C++ and Python is open source)

Sourcetrail combines features such as interactive dependency graphs, concise code views, and efficient code searching to provide source code overviews and details.

By statically analyzing the source code and visualizing it, developers are freed from reading a huge amount of existing source code. Sourcetrail supports C, C++, Java, Python, etc., and it seems possible to connect with editors such as Atom, Eclipse, VS Code, and IDE.

I tried Sourcetrail × macOS × VSCode

0. Assumption

Try setup in the following environment.

  • macOS Catalina 10.15.1
  • Visual Studio Code 1.40.1
  • Python 3.6.9

1. Sourcetrail installation and project creation

  • It can be downloaded from here (https://github.com/CoatiSoftware/Sourcetrail/releases ). This time, select “Source trail_2019_4_61_macOS_64bit.zip”.
  • Unzip the downloaded zip and copy Sourcetrail.app in it to the application folder.

I will start it immediately.

  • Create project

You can create a new project with Command + N.

| Item | Input value | | ————————— | ——————— ——————————- | Sourcetail Project Name | Enter any project name. | Sourcetail Project Location | Specify the location where Sourcetail project files are saved. |

  • Select Source Group

When you click “Add Source Group” in General, the following window will be displayed.

Since there is only “Empty Python Source Group” for Python, select it and click the “Next” button.

  • Source Group settings

Click the “Next” button in the New Source Group, and the following setting window will be displayed.

| Item | Input value | | —————————- | ——————– —————————————- | | Python Environment | Enter the full Python path. | | Files & Directories to index | Specify the directory for source code analysis this time with full path. | | Excluded Files & Directory | You can probably specify excluded folders. (I haven’t tried it, so I don’t know the details…) | Source File Extensions | I’m not sure, so I’m going through. |

  • Create!!

When you click the “Next” button in the Source Group settings window, the following window will be displayed. Click “Create” to complete the project creation.

The following display will appear, so click “Start”.

・・・This time it was a trial so there was only one file, so it ended in an instant. Click “Quit” to see the result.

(Although it’s a lonely display) It seems that it has succeeded.

2. VS Code side settings

I have seen the success of Sourcetail setup, so next I will configure VS Code.

  • Extension installation

Search for “source trail” from Extensions and install the extension.

  • Change settings

Open the setting with command + ,, search with “source trail” and set “Source trail: Start Server At Startup” to True.

Official documentation (https://www.sourcetrail.com/documentation/)

The communication between Sourcetrail and the code editor is achieved using a local TCP connection.Sourcetrail uses the port 6667 to listen for incoming messages.Outgoing messages will be sent to the port 6666.

3. Sourcetrail ⇄ VS Code connection confirmation

Both Sourcetrail and VS Code will end once, and then restart.

Connection is completed when the following displays appear on the lower right of each. Thank you for your hard work.

App View
Sourcetrail Connected to VS Code
VSCode ☑︎ Sourcetrail

Connect Sourcetrail→VSCode with Command + left Click, VSCode→Sourcetrail with right Click→“Sourcetrail: Send Location”.

Conclusion

Very convenient. (Small feeling) This time, I was introduced to setup on macOS, but I was able to introduce it smoothly on Windows 10.
