Install Anaconda

Python environment uses Anaconda.

As a matter of fact, once I created a machine learning environment with a plain Python 3.7 environment. And I thought I would write an article like ““Create a machine learning environment on Windows 10 without Anaconda””.

However, while searching, I found some articles that the environment installed with conda is faster, “Eh? Then recreate the environment with Anaconnda?” Again, I built a Python environment with Anaconda.

conda python pip Execute the three and check if the path is correct.

Create virtual environment

Anaconda makes it easy to create a Python environment. If you would like to know more about what advantages it has, leave it to Google Sensei this time.

The command to create a virtual environment is

> conda create --name myenv python=3.7

The part of myenv is the name of the virtual environment, so please name it whatever you like. With this alone, the environment can be created.

Environment switching

> conda activate myenv

You can switch with.

*As a caution, I often see that there is no conda

> activate myenv

However, it seems like an old version, At first glance it looks like it switched without any errors, The environment has not been switched even after checking with info described below.

To see which environment you are currently active in

> conda info -e

You can check with:

# conda environments:
#
base C:\Users\<username>\Anaconda3
myenv * C:\Users\<username>\Anaconda3\envs\myenv

The current environment is marked with “*”.
