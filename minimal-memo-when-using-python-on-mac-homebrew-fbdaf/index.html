[Python] Minimal memo when using Python on Mac (Homebrew)

Dec 26, 2015 Python 

First, Python is installed on the Mac from the beginning.

You can use this, but I’d like to use the homebrew version.

You can also use pyenv to manage installation and version usage. Here.

###environment

  • Mac OSX 10.11.2 (El Capitan)

Standard condition

which python
/usr/bin/python

###brew version installation

In the brew version, Python 2.x series and Python 3.x series can be installed, and Python 3.x series is installed as Python 3, so they can coexist without any special consideration. For now, let’s use Python 2.x series.

brew info As you can see from python, pip seems to be installed together.

brew install python

After installation, try which python again,

/usr/local/bin/python

The brew version (/usr/local/bin/ and below) is used. This is because /usr/local/bin is prioritized over /usr/bin by default, so if PATH is not set for some reason, it will be prioritized in .bash_profile etc. Well define the ranking.

Update modules etc.

Since pip seems to wrap setuptools, update each tool.

pip install --upgrade setuptools
pip install --upgrade pip

Standard state python doesn’t seem to include pip. In that case, you can install with easy_install pip.

Check installed modules

The list is

pip freeze

In can be confirmed. For details on each module,

pip show <package_name>

Can be seen at. The package is

/usr/local/lib/python2.7/site-packages

It seems to be installed in, but this seems to look at the same place as the original python as well as the brew version of python.

Module installation

Install the required modules. For the time being, I need the following.

pip install numpy
pip install matplotlib
pip install pillow

If you include matplotlib, numpy will also be installed.

###Django

For the time being, I will also write the Django setup. Until It worked!

Install

pip install django

pip install django==1.9 etc. You can also do the version.

Create project

django-admin.py startproject django_test

Start and check the test server

cd django_test
python manage.py runserver

Try to access http://127.0.0.1:8000/ with your browser. OK if it worked!
