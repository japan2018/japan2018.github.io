A summary of the contents of this error is something like “The numpy version I read did not work because it was duplicated.”

#How did you solve it? It’s a method of doing various research and uninstalling numpy and reinstalling again. First, see the following blog and move to the corresponding folder and uninstall. http://sechiro.hatenablog.com/entry/2016/04/02/Mac%E3%81%ABTensorFlow%E3%82%92%E5%85%A5%E3%82%8C%E3%82%88% E3%81%86%E3%81%A8%E3%81%97%E3%81%9F%E3%82%89%E3%80%81Numpy%E3%81%AE%E3%82%A8%E3% 83%A9%E3%83%BC%E3%81%8C%E5%87%BA%E3%81%9F%E3%81%AE%E3%81%A7

However, the content of the error message did not change, so I searched various information and found another candidate for the installation location of numpy. Deleted and reinstalled as per the blog below. http://d.hatena.ne.jp/aremokoremo/20140507/1399475248

Then I was able to successfully import TensowFlow and start the programming life with TensowFlow.

If you get hooked up with an error message like me…

Check if numpy is in the following two folders. Then delete all numpy from the following folders and then reinstall the new numpy.

  • /System/Library/Frameworks/Python.framework/Versions/2.7/Extras/lib/python/

  • /Library/Python/2.7/site-packages/
