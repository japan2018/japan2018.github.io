First machine learning project

This is the main part of this article. After the elementary learning, I was able to understand the theory to some extent. I was able to explain machine learning to some extent, and finally I decided to start a machine learning project.

However, listening to the story, it seems that what to do with machine learning has not been decided at all, and rather the purpose of introduction is a strong promotion element. In other words, it’s OK if you can say “This app has artificial intelligence!”

The app was a search system for certain data, and I also had to create a data search engine (SQL), so I decided to use machine learning for this search engine part. At that time, I was silently listening to what happened. For maintenance of object detection cases

I can’t talk to you in detail because of the contract, but I have decided to participate in the maintenance of another project for object detection using SSD300. The learning environment and the deployment environment were all built on the ml engine (now AI-platform) of GCP, which was also my first cloud project.

It wasn’t so much of a problem, and I was informed that the initial delivery date (50% accuracy in the verification data) was cleared without any problems. In the next step, it was supposed that the accuracy of the model would not be improved much and that the final goal would be __90% accuracy. In addition, in parallel, we were refraining from developing apps that used machine learning models, so we were required to make no major mistakes in the direction of accuracy improvement.

However, the setting of this __ precision 90% _ was not good as a matter of course. There was no fixed point about what number was 90% (accuracy or recall or mAP…etc). Neither the front desk nor the other party understood that the measured accuracy was different from the accuracy of production operation.Normally, this side should strictly define the accuracy and explain it to the other side, and also determine the method of production verification, but the person who can stand the front side and explain the above story on this side Because there is no, there was a situation like this. As a result, I noticed that the final definition of delivery was messy, and it was not possible to achieve the target of 90% accuracy even further.

In fact, it was a very burning project, and it was a project that I seriously regret.

What we learned from this project & start a new learning

Now, of course, if you think about it, when you try to use machine learning as a business, you may overlook it, or even if you know it, you tend to go through it. Export in list format

  • Machine learning is not universal, and it is natural that the accuracy does not improve -> it does not just take time, sometimes it is necessary to reconsider models and specifications
  • For that purpose, it is necessary to strictly define what measures the performance of the model.
  • The accuracy obtained from the verification data is not always the same as when released, so do not deliver the model and create a state where it will end without maintenance
  • The accuracy goal that you have told us cannot always be achieved, so do not concurrently perform simultaneous development (apps and web) with a view to achieving that goal.
  • Don’t leave the description and definition of delivery to the front desk (sales) only. Engineers need to take the initiative to stand and explain

Especially the parallel development of the model and the application is the most common. This is definitely not recommended. It’s just a matter of trouble, and if you fail, you can’t take responsibility. If you can’t answer the question “Why didn’t the accuracy reach the target? How can you achieve the accuracy of the target? (Set amount and deadline)”, stop it. Good

The knowledge written here may be understood somehow by reading DeepLearning made from scratch, etc., but it is difficult to understand without experience and experience through actual projects. Considering that it is not the end of making and that the created machine learning model operates as an actual service, it tends to be insufficient from the viewpoint of developers only

I became interested in what I needed to do in order not to fail as a machine learning project. I bought “Machine Learning Beginning at Work” here and read it carefully.

Just like the title of the book, I had a lot of know-how on what to think in order to handle machine learning at work.

To accompany the business

Exactly one more employee (T-kun) working on machine learning at this time. Originally T-kun was a high-spec person who majored in theoretical physics, so I left it to him to implement and develop machine learning models. Along with that, I followed the sales person

  • Is there a major problem with the requirements definition (what is input and what is output)
  • Avoiding unreasonable demands and precision settings
  • Answers to technical questions posed on the spot

I started talking about it. Then, expand the story that I brought back to T, and check whether there is no problem in implementation, and whether the accuracy setting line is far from the accuracy of the latest paper with similar genre. I made a cycle to By creating this cycle, there are no major mistakes in requirement definition or unreasonable accuracy settings.

Also, as mentioned earlier, parallel development of apps and machine learning models is quite a land mine, so when receiving a challenging machine learning project, always receive it in the form of PoC. .. Be sure to pass the story that it will not be incorporated into the app (this can avoid the risk of spending extra money for the other party)

Harmful consequences of this cycle

It’s a cycle I made to avoid burning, but it’s actually bad. In short, it means that the period will be delayed by the time the project starts. __Before, even if it was a little unreasonable or a strange requirement was defined, I started the project with OK, so if you compare it, the sense of speed will slow down. In addition, it was often found that the other party did not finalize the specifications while developing the requirements definition. When I said, “How do you proceed here?”, there were often no responses, and replies would come weeks later.

However, I think that this is unavoidable for avoiding the flame, but inevitably there will be a collision with the sales side.

Model development project after a long absence

After a while, I was interacting with the sales person and the other party, but since the machine learning development team was running out, I decided to return to model development. This case aims to create a model that searches for similar questions using Doc2Vec.

This case was received in the form of PoC. As an improvement this time, we adopted a form that prepares a report of the transition and progress of the accuracy of the model and delivers it every designated period. Even if this is PoC, it doesn’t end with the delivery “It was bad”, and even if the accuracy does not come out __ Why the result and consideration __ By passing in the form of a report, we aim to ensure that failures do not end in failure

The final result was not good. We considered that the main reason was that the number of data prepared for learning was quite small (about 4,000 unique data) and that the type of data might be too wide. If there was no report, there was a possibility that it would burn like “Why is this result?”, but since I delivered the report every time, I was able to avoid it and talk about the next case. This is the latest project I participated in, and now it’s a handyman state to participate in web development.
