How to learn & infer a transformer English-Japanese translation model with Cloud TPU

Jan 29, 2020 Python translation gce gcp Transformer 

[Official Tutorial](https://cloud.google.com/tpu/docs/tutorials/transformer?hl=ja) If you proceeded according to the street, it didn't work, so take a note.

In this article, we will explain how to launch CloudTPU and GCE VM instances and build an English-Japanese translation model using transformer, which is one of the NMT models.

Premise

  • You have already created a project on Google Cloud Platform
  • Billing is enabled for created projects

Cloud Console

In Cloud Console, enter the following to launch a new CloudTPU and GCE VM instance.

# Set project ID
gcloud config set project <project_id>
# Start ctpu (ctpu name is transformer)
# Start GCE instance
ctpu up --name=transformer --tf-version=1.14

In the official tutorial, it is supposed to start with ctpu up, but the version does not match the default tensorflow of the GCE VM instance, so if you follow the tutorial you will get an error. To proceed according to the tutorial, you need to align the version of CloudTPU’s tensorflow with that of the GCE VM instance.

GCE (Google Computing Engine)

This section describes the procedure for learning and inferring using the transformer model based on the own dataset (English-Japanese parallel translation) stored in GCS (Google Cloud Strage). Below, we will proceed with connecting to the VM instance of GCE created with ctpu up with SSH.

Directory structure in VM instance

.
├── src
│  ├── __init__.py
│  └── myproblem.py
└── tmp
    └── t2t_tmp
        └── sample.picke

Download learning dataset from GCS to GCE

gsutil cp gs://<budge_name>/sample.pickle ./tmp/t2t_tmp/sample.pickle

Here, sample.pickle is a data frame with two columns, english (English) and japanese (Japanese).

PROBLE definition

If you want to use your own dataset, you need to implement and register PROBME. Reference: https://tensorflow.github.io/tensor2tensor/new_problem.html Here, create the following two Python scripts.

from .import myproblem

import pickle

import numpy as np

from tensor2tensor.data_generators import problem
from tensor2tensor.data_generators import text_problems
from tensor2tensor.utils import registry


@registry.register_problem
class Translate_JPEN(text_problems.Text2TextProblem):
    @property
    def approx_vocab_size(self):
        return 2**13

    @property
    def is_generate_per_split(self):
        return False

    @property
    def dataset_splits(self):
        return [{
            "split": problem.DatasetSplit.TRAIN,
            "shards": 9,
        }, {
            "split": problem.DatasetSplit.EVAL,
            "shards": 1,
        }]

    def generate_samples(self, data_dir, tmp_dir, dataset_split):
        with open('./tmp/t2t_tmp/sample.pickle','rb') as fin:
            sentences = pickle.load(fin)
        for row in np.array(sentences):
            yield {'inputs': row[0],'targets': row[1]}

Set environment variables in VM instance

# Set of environment variables
export STORAGE_BUCKET=gs://<project_name>
export DATA_DIR=$STORAGE_BUCKET/transformer
export TMP_DIR=/tmp/t2t_tmp
export PATH=.local/bin:$PATH
export PROBLEM=translate_jpen
export TRAIN_DIR=$STORAGE_BUCKET/training/transformer_ende
export MODEL=transformer
export HPARAMS=transformer_tpu
# Self-made script
export USR_DIR=./src

#Preprocessing and learning After preprocessing based on your own ./src/myproblem.py, you will learn. Here, cloud_tpu_name directly specifies the name specified in ctpu up. (An error will occur if you specify it with $TPU_NAME) Reference: https://stackoverflow.com/questions/59089613/tpu-core-error-on-google-cloud-platform-cannot-find-any-tpu-cores-in-the-system

It took about 3 hours with a data set of about 60,000 parallel translations, depending on the amount of data.

# Preprocessing
t2t-datagen \
  --problem=$PROBLEM \
  --data_dir=$DATA_DIR \
  --tmp_dir=$TMP_DIR \
  --t2t_usr_dir=$USR_DIR

#Learning
t2t-trainer \
  --data_dir=$DATA_DIR \
  --problem=$PROBLEM \
  --train_steps=40000 \
  --eval_steps=3 \
  --model=$MODEL \
  --hparams_set=$HPARAMS \
  --output_dir=$TRAIN_DIR \
  --t2t_usr_dir=$USR_DIR \
  --use_tpu=True \
  --cloud_tpu_name=transformer

Reasoning

After learning, perform inference. You can run the translation in an interactive shell by setting the decode_interactive parameter to True. Please refer to the following if you want to infer locally based on the results of learning with _CloudTPU. _ https://qiita.com/yolo_kiyoshi/items/209750f27f582ed48257

# Inference
t2t-decoder \
   --data_dir=$DATA_DIR \
   --problem=$PROBLEM \
   --model=$MODEL \
   --hparams_set=$HPARAMS \
   --output_dir=$TRAIN_DIR \
   --t2t_usr_dir=$USR_DIR \
   --decode_hparams="beam_size=4,alpha=0.6 \
   --decode_interactive=true
