Drawing a tree structure with D3.js in Jupyter Notebook

Nov 28, 2019 Python d3.js Jupyter 

# Thing you want to do

I want to draw tree-structured data beautifully with Jupyter Notebook or a web application! That is my motivation. I use Python because of the convenience of analyzing the original data, and I wanted to repeat analysis and visualization interactively.

Python

There seem to be various ways to draw a tree structure in Python, and the first thing I can think of is networkxor[graphviz].AlthoughIuseagraphlibrarysuchas(https://graphviz.readthedocs.io/en/stable/), I think that it is difficult to express elaborately because it is a library intended to analyze the graph structure in the first place.

On the other hand, ETE Toolkit is a dedicated analysis/visualization tool for tree structures. The model is expressed using The Newick format which is well known as a tree structure. This also seems to be pretty good, but considering the affinity on the web, I wanted to use something more versatile such as JSON for describing the model.

Javascript

There is a good summary about drawing graph structures in Javascript at here. There are various things, but I think it’s probably D3.js in terms of versatility and the amount of information. There is also a library for drawing tree structures.

So, I will try drawing a tree structure with D3.js using Jupyter Notebook. Jupyter Notebook has a good affinity from the beginning because it can execute Javascript and draw HTML without using the Python kernel by using cell magic (%%javascript).

Using D3.js with Jupyter Notebook

As a method, it is possible to use Javascript cell magic and py_d3.TheformeriswellwrittenusingRequireJS,whichisdifficulttounderstandintuitively(seehere). The latter seems to wrap Javascript cell magic nicely. Here, we will explain the drawing using py_d3.

Preparation

Install py_d3 anyway. Use pip. pip install py_d3 First, load the module as follows:

import py_d3
py_d3.load_ipython_extension(get_ipython())

Below is a quote from a sample home page. The first %%d3 5.12.0 specifies the version of d3.js. You can find out which version you can use by typing %d3 version.


%%d3 5.12.0

<g></g>

<style>

    height: 25px;

div.bar {

d3.select("g").selectAll("div")

    });
</script>

Data integration with Python

If you make full use of RequireJS, you can input objects and variables calculated in Python to the defined Javascript functions. However, it is not much different from outputting as JSON and reading with d3.js, so if it is not very large data, it is easy to go through the intermediate file. For example, prepare a JSON file with a tree structure as follows.

import json
data = {
    "name": "A",
    "children": [
      {"name": "B" },
      {
        "name": "C",
        "children": [{ "name": "D" }, {"name": "E" }, {"name": "F" }]
      },
      {"name": "G" },
      {
        "name": "H",
        "children": [{ "name": "I" }, {"name": "J" }]
      },
      {"name": "K"}
    ]
  };
json_file = open('test.json','w')
json.dump(data, json_file)

Tree drawing

To draw the tree structure, use the tree layout of d3 hierarchy. After loading the JSON data structure and creating a tree based on that data, all you have to do is adjust the SVG drawing. Please refer to the d3.js and SVG documentation for detailed drawing adjustments.

%%d3 5.12.0

<style>
.link {
fill: none;
stroke: #555;
stroke-opacity: 0.4;
stroke-width: 1.5px;
}
</style>

<svg width="800" height="600"></svg>

<script>
var width = 800;
var height = 600;

var g = d3.select("svg").append("g")
    .attr("transform", "translate(80,0)");
     console.log("data");


d3.json("test.json")
    .then((data) => {
        console.log(data);
        var root = d3.hierarchy(data);
        var tree = d3.tree(root).size([height, width-160]);
        tree(root);

        var link = g.selectAll(".link")
            .data(root.descendants().slice(1))
            .enter()
            .append("path")
            .attr("class", "link")
            .attr("d", (d) => {
                return "M" + d.y + "," + d.x +
                    "C" + (d.parent.y + 100) + "," + d.x +
                    "" + (d.parent.y + 100) + "," + d.parent.x +
                    "" + d.parent.y + "," + d.parent.x;
            });

        var node = g.selectAll(".node")
            .data(root.descendants())
            .enter()
            .append("g")
            .attr("class", "node")
            .attr("transform", function(d) (return "translate(" + d.y + "," + d.x + ")"; })

        node.append("circle")
            .attr("r", 8)
            .attr("fill", "#999");

        node.append("text")
            .attr("dy", 3)
            .attr("x", function(d) {return d.children? -12 :12; })
            .style("text-anchor", function(d) {return d.children? "end" :"start"; })
            .attr("font-size", "200%")
            .text(function(d) {return d.data.name; });
        })
   .catch((error)=>{
    
   }
)
</script>
