Recommended for people like this

First of all, as an introductory chapter, I am familiar with Qt to some extent, but PyQt is for those who have never used it. Or maybe you’re using Python and want to take advantage of the rich features of Qt.

What is #PyQt? Qt’s Python bindings, said to have been developed by Riverbank Computing, UK. Binding means connecting. In short, it’s the one that connects Python and Qt. More specifically, I think it’s meant to allow you to call Qt from Python.

History

I didn’t know exactly when the development started, but it seems that it existed around 2007. PySide developed by Nokia (now The Qt Company) is said to be the same, but it seems that it was developed as a substitute because PyQt’s license agreement with Riverbank Computing Corp. failed.

License

PyQt is GPL. For reference, PySide is LGPL. You may want to use PySide if you give priority to corporate profits, but PyQt seems to be more convenient. Which one to employ as a company is a problem.

Install

Below is a brief summary of the steps to use PyQt.

  1. Anaconda Install (Python language distribution)
  2. Qt Install
  3. Build a virtual environment for PyQt from Anaconda
  4. Install PyQt in the created virtual environment

Run # Sample program

Below is the python code to display a simple MainWindow.


#-*-coding: utf8-*-
  
import sys

from PyQt5.QtWidgets import QApplication, QMainWindow, QLabel

class MainWindow(QMainWindow):

    def __init__(self):
        super().__init__()
        self.title ='PyQt test(QMainWindow)'
        self.width = 400
        self.height = 200
        self.setWindowTitle(self.title)
        self.setGeometry(0, 0, self.width, self.height)
        label = QLabel('This is PyQt test.', self)
        self.show()


if __name__ =='__main__':
    app = QApplication(sys.argv)
    window = MainWindow()
    sys.exit(app.exec_())
