Features of functional programming

  • Variables and functions are transparent to references and have side effects suppressed or eliminated altogether.
  • That is, the inside of the function doesn’t rely on any external data and doesn’t modify them
  • In the pure function type with side effects completely eliminated, there is no concept of assigning a value to a variable, and the value is bound to the variable.
  • Pipelines, higher-order functions, closures, etc. are used to divide the processing into functions and use them in combination
  • Typical languages: Haskell etc.

What is reference transparency?

  1. Variables that are not affected by the environment and have the same value (immutable (immutable)) no matter when they are referenced
  2. Includes functions that always return the same value if the arguments are the same
  3. Human error can be suppressed because the value is unchanged.
  4. Strong in parallel processing because it does not generate side effects (hard to cause data conflict)

# Measure function processing time
from functools import wraps
import time

# Define a function that receives a function
# Return the function inside
def time_measurement(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        exe = func(*args, **kwargs)
        end = time.time()
        print(f"time_measurement: {end-start}")
        return exe
    return wrapper

Call with # @ (decorator) and enter the argument of the called function

def func(num):

“Is the last one object-oriented? It’s a guy who can be compared to an apple or a car!”

class Meat:
    # Instance initialization
    # Each instance has the attributes defined here
    def __init__(self, name, weight):
        if weight <0:
            raise ValueError ("Weight must not be less than 0g")
        self.name = name
        self.weight = weight
        # Add _ so that it cannot be used directly from the outside
        # Discount set to initial value 5
        self._unit_price = 5
        self._price = 0
    # Set the string that is displayed when you call the instance
    def __repr__(self):
        return f"{self.name}: {self.price} Yen"
    You can call a method from the outside like an instance variable by adding a # property decorator
    @property
    def unit_price(self):
        # Set the value you want to reference as the return value
        return self._unit_price
    @property
    def price(self):
        return self.weight * self._unit_price
    #Use setter to change variables
    If you want to change the # variable, do not overwrite it directly
    # By doing this, external reference will be made through this method, preventing illegal value mixing.
    @unit_price.setter
    def unit_price(self, value):
        if not (1 <= value <= 10):
            raise ValueError ("Set the unit price between 1 yen and 10 yen")
        self._unit_price = int(value)

class Item:
    def __init__(self, name):
        self.name = name
        self._price = 0
    def __repr__(self):
        return f"{self.name}: {self.price} Yen"
    @property
    def price(self):
        return self._price

inheritance

“When I want to make a meat class again, I’m going to inherit the above abstract class (Item class) to make a concrete class.”

  • Inherit a parent abstracted object and create a more concrete class
  • If it is python, it can be inherited by taking the parent class as an argument when defining the class
  • The properties and methods of the parent class cannot be used unless explicitly inherited by using the super() method at initialization.
class Meat(Item):
    def __init__(self, name, weight):
        Explicitly call __init__() of the parent class using #super()
        super().__init__(name)
        self.weight = weight
        self._unit_price = 5
    @property
    def unit_price(self):
        return self._unit_price
    Override # method
    @property
    def price(self):
        return self.weight * self._unit_price
    @unit_price.setter
    def unit_price(self, value):
        if not (1 <= value <= 10):
            raise ValueError ("Set the unit price between 1 yen and 10 yen")
        self._unit_price = int(value)
