A-We Love Golf

It is a question to answer whether K is in the range from A to B inclusive.

From the constraint of $1\leq A \leq B \leq 1000$, $1\leq K \leq 1000$, it is no problem if you simply search for a value that satisfies the condition in the range from $A$ to $B$ ..

N = int(input())
A, B = map(int, input().split())
for i in range(A, B+1):
  if i %N == 0:
    print('OK')
    break
else:
  print('NG')

B-1%

It is a question of how many years later it will exceed the given amount of X yen when 1% of annual interest is attached to 100 yen deposited.

Amounts below the decimal point are rounded down. I answered the number of times I turned while until the specified amount was exceeded.

X = int(input())
n = 100
count = 0
while n <X:
  count += 1
  n = int(n * 1.01)
print(count)

C-Many Requirements

You will get $d$ points when $A_b-A_a = c$ for inputs $a, b, c, d$. The question is, what is the maximum total score when the sequence $\boldsymbol{A}$ can be created freely?

It looks like a messy problem, but the sequence length $N$, maximum $M$ is $1\leq N \leq 10, 1\leq M \leq 10$, given $a, b The quantity $Q$ of ,c,d$ is also specified as $1\leq Q \leq 50$. I thought this would be a brute force push.

We used itertools.combinations_with_replacement() for brute force. Note that it is decided that the sequence $A$ is a sorted value, but duplicate values are possible.

import itertools

N, M, Q = map(int, input().split())
abcd = [tuple(map(int, input().split())) for _ in range(Q)]

Max = 0
for C in itertools.combinations_with_replacement(range(1, M + 1), N):
  score = sum([d for a, b, c, d in abcd if C[b-1]-C[a-1] == c])
  Max = max(Max, score)
print(Max)

D-Floor Function

$$ floor (Ax/B)-A\times floor (x/B) $$

The problem is to find $x$ where is the maximum value.

Aside from the mathematical proof, I wondered if I should find the maximum value of $x$ where $floor (x/B)=0$. The condition that satisfies the condition is $B-1$ which is the maximum of $N$ or less, that is, the smaller of $B-1$ and $N$.

So I wrote the following.

A, B, N = map(int, input().split())

x = min(B-1, N)
print(A*x // B-A * (x//B))

Mathematical explanation

I didn’t understand the mathematical explanation, so I read the explanation.

$$ f(x) = floor (Ax/B)-A\times floor (x/B)$$

And put. When the value of $x$ is increased by $B$ in this formula, the first term on the right side is always increased by $A$. The second term is always reduced by $A$. These cancel each other out and the total remains unchanged. That is, the following formula is established.

$$f(x) = f(x+B)$$

So we don’t have to think about the range of $B\leq x$. Find the maximum value in the range $0\leq x <B$, $0\leq x \leq N$.

In this range, the value of the second term, which is originally negative, is always 0, so the value of the first term, which is monotonically increasing, should be maximized within the range. That is, maximize $x$ in the range.

Since the second term is not used, the following writing is also good.

A, B, N = map(int, input().split())
print(A * min(B-1, N) // B)

E-Rotation Matching

When $N$ participants play 1vs1 rock-paper-scissors at the same time for $M$ games and $N$ times, it is a problem to think about how to make a combination where the opponents do not overlap.

The problem statement is extremely complicated. What are you listening to?

Suppose $N=9$ people have participated in this competition. The game will consist of 9 sets. From here, let’s set $M = 3$ and try to make three pairs.

Let’s select a pair of 1 and 2 appropriately.

Even if you rotate it 9 times in this state, opponents will not overlap.

Next, let’s take a pair of 3 and 5.

Again, there is no duplication.

Next, let’s take a pair of 6 and 7.

This will cause duplication. A staggered pair of 1, 2 and a staggered pair of 6, 7 will get the same thing on the way.

How to fix this? Since it has been shifted by one or two, let’s shift three this time.

This time there is no duplication. The problem is to make such a pair.

In other words, this problem can be restated as follows.

“Output $M$ pairs of ($A_i, B_i$) for which $B_i-A_i = 1, 2, 3, …, M$ without duplicate values.”

You can’t do a decent search to solve this. (TLE has been rolled out.) From the beginning, decide the shape and place it.

So I implemented it as follows.

N, M = map(int, input().split())

halfPos = -(-N//2)
qPos = halfPos // 2
q3Pos = qPos + halfPos

for m in range(1, M+1):
  if m %2:
    print(qPos-m//2, qPos+m//2+1)
  else:
    print(q3Pos-m//2, q3Pos+m//2)
