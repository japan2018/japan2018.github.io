[Python] Make the default value of the argument immutable

May 25, 2012 Python 

You can set default values for arguments in function definitions

def foo(bar=[]):
    bar.append('baz')
    return bar

Executing foo() returns a list called ['baz']

>>> foo()
['baz']

What happens if you run the same code again?

>>> foo()
['baz','baz']

The results have changed. The reason for this is that Python’s default value for a function is evaluated only once. And the Python list is a mutable object, and the append() destructively adds the argument to the list, so the second execution will return an unexpected result. ..

To clear this problem, immutable object should be the default. In the case of this foo function

def foo(bar=None):
    if bar is None:
        bar = []
    bar.append('baz')
    return bar

Like No matter how many times this function is called with no arguments, None returns the same result each time because it is immutable.

>>> foo()
['baz']
>>> foo()
['baz']

In Python, str is immutable, so using a string as a default value is not a problem.
