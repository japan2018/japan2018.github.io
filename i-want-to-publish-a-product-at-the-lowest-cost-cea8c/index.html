#What can a free plan do? Most things can be done. You can use it without problems ** even if you deploy and operate the application on the server.

There are two restrictions you should be aware of with Heroku’s free plan.

-Free Dyno hours (uptime) 550 hours/month -App sleeps after 30 minutes of inactivity

Free Dyno hours (uptime) 550 hours/month

Heroku’s free plan defaults to 550 hours of Dyno hours per month.

30 days x 24 hours = 720 hours, so if you try to run it for a full month, the free plan will not be enough.

However, this can be solved if you have a credit card.

Even if you have a free plan and register a credit card to your account 450 hours/month of free Dyno hours will be added for a total of 1,000 hours/month.

App sleeps after 30 minutes of inactivity

With the free plan, the app will sleep if there is no access for 30 minutes, The response will be slow due to the amount of time that the application starts up the next time you access.

However, this can be solved with Heroku Scheduler.

Heroku Scheduler is a function that allows you to periodically execute specified commands. It’s like Linux cron.

By executing curl command for the URL of the application with Heroku Scheduler You can continue running without dropping the application.

▷ The setting method of Heroku Scheduler is summarized in this article.

How to deploy to server

*This article assumes the Django project is deployed.

  • In case of Rails project, Gemfile modification etc. are required, but the Heroku command to execute is the same.

1. Install required libraries

Install the libraries required for your project to run on Heroku.

pip install gunicorn django-heroku

2. Add the execution environment file

Create a file directly under the project directory that tells Heroku the project execution environment.

runtime.txt

This file describes the Python version.

python-3.6.6

Procfile

This is a file that describes the startup command of the Heroku process. Replace <your-project-name> with your project name.

web: gunicorn <your-project-name>.wsgi --log-file-

requirements.txt

Here is the list of dependent packages. Let’s make the output result of the pip freeze command as a file.

pip freeze> requirements.txt

3. Modify the project settings file

Modify the DEBUG = Ture part of settings.py for production use.

DEBUG = False

4. Commit with Git

Create & commit a local repository with Git.

git init
git config user.name "Your Name"
git config user.email you@example.com
git add.
git commit -m "first commit"

5. Deploy with Heroku command

Deploy to Heroku with the Heroku command. When you execute the heroku login command, the browser will be launched and you will be asked for Heroku authentication. After logging in, you can go to the terminal without dropping your browser for further work.

heroku login
heroku create <your-app-name>
git push heroku master
heroku ps:scale web=1
heroku run python manage.py migrate
