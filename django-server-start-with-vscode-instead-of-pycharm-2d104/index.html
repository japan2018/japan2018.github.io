#Environment

  • Windows10
  • Python: 3.7
  • Django: 2.2.2
  • VScode: 1.4.1
  • Working directory is shown on the right: /Users/marun/test

1. Interpreter settings

Since I was developing using venv, I created a new settings.json and reconfigured venv’s Python.exe as follows.

{
    "files.eol": "\n",
    "terminal.integrated.env.windows": {
        "PATH": "${workspaceFolder}/venv/Scripts;${env:PATH}"
    },
    // ${workspaceFolder}: C:Users/marun/test.
    "python.pythonPath": "${workspaceFolder}/venv/Scripts/python.exe"
}

2. Set environment variables

I needed an environment variable to start the Django server on Windows, but I will create it so that launch.json can read the .env file.

PYTHONUNBUFFERED=1
DB_USER="[user name]"
DB_PASSWORD="[password]"

This is a setting for me, so please change the contents if necessary. For mac, don’t need environment variables?

3. Create launch.json

Finally, we will create launch.json which will be the basis of the debug function configuration.

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Python: Django",
            "type": "python",
            "request": "launch",
            "program": "${workspaceFolder}/manage.py",
            "console": "integratedTerminal",
            "args": [
                "runserver",
                "--noreload"
            ],
            "envFile": "${workspaceFolder}/.vscode/.env",
            "django": true
        }
    ]
}

By defining envFile, I was able to read the environment variables safely. When you create this, I think the following buttons will appear in the debug mode.
