After changing the account name on Mac, I could not use pip or ipython, so I re-installed python and it was cured

Mar 29, 2020 Python Mac brew pyenv 

#Overview

When I changed my mac account name, I couldn’t use pip or ipython installed by pyenv, so I could use python again. #environment macOS Catalina 10.15.4 Homebrew 2.2.11 pyenv 1.2.17 python 3.8.0 #Background I used brew on mac to install pyenv, and then pyenv was used to install python. After that, I changed my mac account name (and home directory name) for various reasons. Follow the official description to change the account name. The changes were successful, but when I run pip or ipython, I get the following error and cannot use it.

>> ipython
/usr/local/Cellar/pyenv/1.2.17/pyenv.d/exec/pip-rehash/pip: /Users/[new home directory name]/.pyenv/versions/3.8.0/bin/pip: /Users /[old home directory name]/.pyenv/versions/3.8.0/bin/python3.8: bad interpreter: No such file or directory

It looks like I’m angry that I can’t find my old home directory name python. I think the old settings are left somewhere, but I often find where “/Users/[old home directory name]/.pyenv/versions/3.8.0/bin/python3.8” is defined. I didn’t know, so I gave up and re-installed python 3.8.0 with pyenv and it worked.

pyenv uninstall 3.8.0
pyenv install 3.8.0
