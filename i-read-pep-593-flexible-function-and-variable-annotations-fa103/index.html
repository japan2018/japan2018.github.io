I read PEP-593 (Flexible function and variable annotations)

Feb 16, 2020 Python Python3 mypy pep 

In a certain discussion flow, I decided to read [PEP-593 (Flexible function and variable annotations)](https://www.python.org/dev/peps/pep-0593/), so I wrote down my understanding. Leave it in.

Overview

  • PEP-3107 (Function Annotations) introduced annotations into the Python syntax. At that time, the type hints, DB query mapping, and RPS marshalling information were mentioned as the uses of annotations.
  • PEP-484 (Type Hints) introduced a method to implement type hints using annotation notation. It can be said that this is the main usage of annotations.
  • Now that PEP-484 has become the de facto standard, it has become difficult to use annotations for purposes other than type hints.
  • So, let’s redefine annotations to add type hints as well as our own metadata.

Approach

  • Add new typing.Annotated
  • To annotate a data or function with type T and metadata x, y, write Annotated[T, x, y]
    • The first element is regarded as a type
  • Type checkers should refer to T, and tools and libraries that use metadata should refer to x and y.
    • In other words, tools and libraries ignore annotations that they don’t support

Example

Numerical value in the range 3-10, which is considered a character in the ctype (fictitious) annotation.

Annotated[int, ValueRange(3, 10), ctype("char")]

Impression

  • It is difficult to write Annotated. I wanted to write a more rough tuple…
  • Since the contents of annotations can be acquired even at runtime (get_type_hints()), it seems that libraries and frameworks that use this will appear.
  • I hope that it will not become a chaotic code as a result of abuse.
  • PEP-3107 (function annotation) is introduced, but PEP-526 (Syntax for Variable Annotations; variable annotation) Is not mentioned at all. A pitiful child…
    • Of course not only functions but also variables can be annotated.
