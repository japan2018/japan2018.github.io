#About school

It was useful

  • I learned the basics of HTML/CSS/JavaScript/Linux and it is still useful when creating a system.
  • The language has changed from Ruby to Python, but both types are dynamically typed, so there was no learning cost for learning the basic grammar.
  • I learned the basics of creating and operating EC2 and S3, and since I used AWS for all system deployment in my current position, I was able to enter the business smoothly.
  • I don’t use it frequently in my current job, but the experience of developing multiple people using Git is useful everywhere in my work.
  • By observing how to write a question sheet for mentors, when asking a senior engineer etc., it is possible to clearly tell what the problem is and where it is stuck, and it is easy to get an answer.

What didn’t help

Basically, I think that what I learned is useful to some extent, but I thought that the following points could not be helped.

  • I learned Ruby at school, but I should not use it at all in my current job (because I was aspiring to a machine learning project, I was diligent)

What you should have done before changing jobs

  1. It’s easy, so bring a simple system or app in the language you’ll use in your new workplace to deploy
Reason

I thought that I should make more efforts to quickly adapt to the workplace by touching the code writing and framework beforehand.

  1. A better understanding of AWS services
Reason

I think most of the recent service deployments use AWS. It’s nice to study in the free frame, but in reality, you will spend tens of thousands of dollars and hundreds of thousands of yen to operate the system. Therefore, I thought it would have been better if I could understand the AWS service as much as possible, such as spending a little money to learn how to take a domain and learn how to distribute access.

  1. Awareness and practice of writing clean code
Reason

It’s very important to be aware of clean code, because it’s a tough reviewer and you won’t even know it when you fix it later. (The author wrote it so redundantly that it took a long time to review it at first)

The hard thing now

  1. You need to learn various functions because you use JavaScript more than you can imagine
  2. Basically, there are many parts that I do not understand because I create a system with a small number of people and analyze data, so it is easy to stop work

There are a lot of other difficult things to do, but in the end, ask someone who seems to understand if you have a problem! ! I think this is a good way to get the job done. I wasn’t good at listening to people, and I tended to try it alone. However, if you ask a senior engineer, they will solve it in a few minutes or give you advice, so it is 100% faster to ask a person who understands it than to spend an hour investigating it! !
