[Python] Let’s put in Glances (Python command line monitoring tool).

Mar 24, 2014 Python use 

I just picked it up on TL on Twitter, so I put it in.
  • https://pypi.python.org/pypi/Glances -A cross-platform curses-based monitoring tool

The reason is that the screenshots in pipy were good ^^;

Install

CentOS provides rpm so you can install it with yum. (Python-glance-2012.2.4-3.el6.noarch)

In my case, I added it to CentOS 6.5 of Vagrant. (Not yum, easy-install or pip can be easily entered)

Try to start

Just type the command.

% glances

screenshot

A short time after starting, the resource status is drawn in batch.

Help screen. By default, various information is displayed on one screen, but it seems that you can narrow it down as an option.

Start in client & server mode

Looking at the docs, there seems to be some client & server mode. If you check the expression,

  • Side that provides resource information: server
  • Resource status acquisition side: Client

Will be. In addition, Glances must be installed in both environments.

This time, I tried CentOS side of Vagrant as a server and MacOS side as a client.

[vagrant@centos-tare ~]$ glances -s # vagrant side / Assigning IP to 192.168.33.11.
% glances -c @192.168.33.11 # MacOS side

Like this, you can get information from the client side through port: 61209.

It is displayed in green when there is no particular load.

When you are busy?

If there is any load, it will appear colored on the terminal depending on the situation.

There was no good idea to load it, but I decided to put a write load on the OS on Vagrant side with a disk IO benchmark tool called bonnie++.

Performance measurement is not the purpose (^^;

When I tried it, the process of bonnie++ spun around, the CPU value increased, and it changed to green -> pink -> red.

Has API

By the way, in addition to listening in server mode, it seems that it also has an API with XML-RPC. With python, it seems that data can be obtained in json format.

  • https://github.com/nicolargo/glances/wiki/The-Glances-API-How-To

The listening Port is also 61209.

If you try tcpdump while launching the process of the client on the MacOS side, you can see the information like this. (Let’s pick up the character string with the A option)

% sudo tcpdump -i vboxnet1 -A
