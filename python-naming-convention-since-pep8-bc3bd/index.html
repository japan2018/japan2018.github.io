Python naming convention (since PEP8)

Jan 21, 2020 Python Python3 naming rules PEP8 

Python has "[PEP8: Python code style guide](https://pep8-ja.readthedocs.io/ja/latest/)", which also describes naming conventions such as variable names and function names.

By matching the naming convention and coding style, you can reduce the burden on people who read the source code and those who review it. Please refer not only to the naming convention but also to the naming method described in “Readable Code”.

PEP8

Usage Naming Convention
Private, private One underscore at the beginning
Name collision avoidance in subclasses Two underscores at the beginning
Special property, special method Add two underscores before and after
(Because it is prepared in the language specification, do not define it yourself)
Collision avoidance with reserved words and built-in function names Last one underscore
Usage Naming Convention
Package Short names in all lowercase, no underscores
Module All names in lowercase, separated by underscores
Classes, exceptions CapWords method (connect uppercase words only at the beginning, do not use underscore)
Function, Method Lowercase letters only, with words separated by underscores
Constant Uppercase letters only, words separated by underscores
Variable lowercase only, with words separated by underscores if necessary
1-character variable Never use l (lowercase ell), O (uppercase O), I (uppercase eye)
(depending on the font the numbers 1, 0 (I can’t tell that))

Other than PEP8

Usage Naming Convention
Discard variable 1 underscore
(when a variable is needed but not used in a for loop or split assignment)

Reedable code

  • Don’t abbreviate words (Obviously omitted names are OK)
  • Information and meaning can be added to the name (flag and check do not know what flag or what value was checked)
  • Communicate information by name so that no comment is required
  • Use single-character variables for temporary variables that can be used within the viewable range (up to 25 lines)
