[Python] TensorFlow 2.1 is here!

Jan 11, 2020 Python TensorFlow 

#Introduction

This is the last article in the Advent Calendar for Tensorflow 2.0. (Large delay) Tensorflow 2.0 has been released! Then Tensorflow 2.1 had already arrived, so Here are the changes and evolved points based on Release Note.

Main topic

  • GPU package integration
  • Cloud TPU support
  • Ragged Tensor Dataset support
  • Last release in Python 2

GPU package integration

From Tensorflow 2.1, the purpose of inserting tensorflow-gpu is gone, Only pip install tensorflow will support tensorflow as GPU! With this, there was no confusion and it felt smarter.

keras API Cloud TPU support

GCP has a function called Cloud TPU. There are also TPU items in Colabratory. It is Experimental, but with keras API such as keras.fit,keras.predict Cloud TPU is now available. In the old days, I had to do various things before doing keras.fit, In the first place, it cannot be used in Colabratory (This information) It was troublesome in various ways. It would be nice to see this improved and become TPU friendly!

#Ragged Tensor Dataset support

Ragged Tensor can finally be handled by Dataset, You can now do the following:

rt_x = tf.ragged.constant([[1,2,3],[],[1,2]])
x_data = tf.data.Dataset.from_tensor_slices(rt_x)

for x in x_data:
    print(x)

tf.Tensor([1 2 3], shape=(3,), dtype=int32)
tf.Tensor([], shape=(0,), dtype=int32)
tf.Tensor([1 2], shape=(2,), dtype=int32)

This may increase the number of situations in which natural language processing and various uses of Ragged Tensor can be utilized.

Python2 support is final

TensorFlow 2.1 will be the last TF release supporting Python 2.Python 2 support officially ends an January 1, 2020. As announced earlier, TensorFlow will also stop supporting Python 2 starting January 1, 2020, and no more releases are expected in 2019.

Therefore, finally, Tensorflow 2.1 will end support for Python 2. Well, I have lived as long as Python2 has a long life, Even though official support grew, it was the last in 2019, so I want to make a solid transition to Python 3 soon. (Before that, I want Google to make Gcloud SDK native to Python3…)
