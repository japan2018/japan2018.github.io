# Overview

PEP 484: Type HintswasintroducedinPython3.5.Formoreinformation,seeOfficialDocumentTypeHintSupport.

Recently, I’ve heard a lot of stories introduced in the community. I would like to proceed with the introduction in future projects, and I will put it here as a reference for my own study and explanations to other people.

Type hints were introduced in Python 3.5 and above as a Python grammar.

  • It’s a hint, so it doesn’t affect anything at runtime
  • No built-in official tools

tool

  • There is a de facto check tool called mypy (it can be installed with pip)
  • Supports editors such as PyCharm, code interpolation becomes powerful, and it also checks
  • Effective only at the place where the hint is written (in module or function)
  • The more you write the hint, the more effective the code interpolation and the more you check it

mypy cheat sheet

Install mypy

(env) $ pip install mypy

check with mypy

(env) $ mypy module name

#Other

Optional

Optional is a type that allows the specified type or None. For example, to allow either integer or None, use Optional[int].

It is used in the following cases.


from typing import Optional

def f1(a: int, b: int) -> Optional[int]:
    if b == 0:
        return None
    return a // b

It is convenient because it can be used as follows.


def f2(c: Optional[int]) -> int:
    if c is None:
        c = 0
    return c + 100

If you write the above case as follows, mypy will notify you of the error.


def f2(c: Optional[int]) -> int:
    return c + 100

$ mypy sample.py
sample.py:**: error: Unsupported operand types for + ("None" and "int")
sample.py:**: note: Left operand is of type "Optional[int]"
Found 1 error in 1 file (checked 1 source file)

That is, by doing if c is None, the time when c is None is added, and the variable c Is guaranteed to be an integer type (int), so it can be processed without problems.

Union

Union can represent one of two types. For example, if you say integer type or string type, specify as Union[int, str].

When you receive an argument as shown below, you can use it when you do not know whether it is an integer or a character string, and by using isinstance to divide the process, you can determine the type of the variable like Optional above. I can do it.


from typing import Union

def f3(d: Union[int, str]) -> str:
    if not isinstance(d, str):
        d = str(d)
    return d.upper()

Here, as if not isinstance(d, str):, when the variable d is not a string type (str), the process of converting it to a string is added and processing continues. With return d.upper(), we know that d is a string, so we can use a string-type method that safely converts it to uppercase.

int and float

A float cannot be entered by specifying int, but an int can be entered by specifying the opposite float. In the following cases, only num4 will give an error.


num1: float = 1.0
num2: float = 1
num3: int = 1
num4: int = 1.0 # this is an error

$ mypy sample/sample.py
sample.py:**: error: Incompatible types in assignment (expression has type "float", variable has type "int")

This seems to be because it is possible to automatically convert int to float, but if you change the opposite float to int, you may lose information.
