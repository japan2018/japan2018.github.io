Nowadays, Houdini is used in various industries, and I think various HDAs are being developed by various companies. On the other hand, if management and documentation are not maintained, **you can not understand how to use HDA created by others, and you can not know what kind of HDA exists **.

This article will show you how to create tooltips and your own documentation for the parameters shown below.

* Since I have only a Windows environment for checking, there may be some differences such as directories on other platforms, but please understand.

#Where to write help

Help can be written in either of the following two places.

  • Type Properties window Help tab
  • $HOUDINI_PATH/help/nodes/category/operator type name.txt
Help tab Text file

Which one is written depends on the studio’s method and preference, but since I want to manage help and HDA separately, I am writing a text file. By the way, standard nodes and Game Development Toolset are managed in the same way.

The Help format can be written using Wiki markup. For details, please see the following documents.

  • Wiki Markup Reference

As you can see in the documentation, this Wiki markup formatting is pretty advanced. However, I feel that the threshold is slightly higher for those who are not used to it and those who are not. Those people can see the help source of Houdini standard node, so it is better to modify it from there.

See standard node markup

There are two ways to see the standard node markup.

  • ** Press the help button of the parameter editor and add .txt at the end of the URL field. ** Put out the node you want to see the source, click the help button in the parameter editor, and add .txt after the displayed URL (it will not be displayed properly until you wait for a while). You can see the markup of.

  • ** View local text files. ** Local text files are located at $HH/help/nodes.zip/category/operator type name.txt. ** *$HH is Houdini installation directory/houdini (Example: C:/Program Files/Side Effects Software/Houdini 17.5.391/houdini**)

#A frequently used format for asset help

Now that you know how to read the source, let’s summarize the asset format using the Align Sop source as an example. ――

  1. The title. Enclose with = Basically, the operator label is written.

  2. icon is the setting of the icon of the node, and the other parts describe the information used for filtering etc. It does not affect the appearance of help.

  3. Summary. Enclose it in **"""**. A simple description that fits within two basic lines is written.

  4. A description of the purpose and operation of the node. Enclose it in ==, write the title, and write a description with a line break.

  5. Insert image. ../ represents $HOUDINI_PATH/help/. To load an animated gif, use **[Anim:filepath.gif]**.

  6. It is an explanation by input. **@inputs** starts the section that describes input. Write a colon (:) after the input label for each input. Below that, describe the behavior of the node for input, etc. with indentation.

  7. Begin the section describing parameters in **@paramters**. When the parameters are divided by folders, they can be divided by setting **== folder label ==**. If you have not set a tooltip on a parameter, it will automatically extract the first paragraph of this parameter’s description and use it as the tooltip for the parameter. **#id:** If you do not add a property and parameter name, it will try to match the parameter using its label. However, it’s better to add the **#id:** property to all parameters. That way, some parameters will be linked properly, even if they have duplicate labels.

  8. **@related** can be used to create a section for links to related pages.

#Create custom sample

You can also add a custom sample to the help. To create the sample, you need to have the following two files in place.

  • .Hda file created by object network
  • ** Text file with sample description **

** *If the .hda file is other than the object network, for example, if it is Dop, it is necessary to create Dop Network and include it in the object network. **

The above two files have the same file name and are placed in the following directory.

  • $HOUDINI_PATH/help/examples/nodes/ category/operator type name/

For example, to display a custom sample named MyNodeExample on the HDA of Sop called mynode Put the sample file below.

  • $HOUDINI_PATH/help/examples/nodes/sop/mynode/MyNodeExample.hda
  • $HOUDINI_PATH/help/examples/nodes/sop/mynode/MyNodeExample.txt

When the file is prepared, you can display the custom sample by adding the following description to the help.

:load_example: My Node Example
    #examplefile: /examples/nodes/sop/mynode/MyNodeExample.hda
    #path: /examples/nodes/sop/mynode/MyNodeExample
    #include: yes
