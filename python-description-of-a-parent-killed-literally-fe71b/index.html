Python programming whose parents were killed by literals of strings, numbers and truth values I think it’s very interesting :raised_hands:, but at the same time I couldn’t understand what was happening, so I looked into it. It’s a commentary or my own study.

I mainly write “small numerical values”. If you make a mistake, throw Masakari.

#0

int() # 0

This is as per @Raclett3’s link.

or return 0 if no arguments are given

It returns 0 if no argument is given, yes. This is easy.

#1

len([()]) # 1

This just returns a length of 1 in len because the list has a tuple in its element. In other words, is it easy to understand what is going on in the example below?

len([(), ()]) # 2
len([1, 2, 3]) # 3

#2

len(((),())) # 2

Obviously it’s all tuples, but all I’m doing is that the list outside 1 is a tuple. It is easy to understand if you insert a little blank.

len( ((), () )) # 2
len( ((), (), () )) # 3

#3

len(hex(not())) # 3

For not(), the explanation of @Raclett3 in the original article is very easy to understand, so please use that. I will transform it little by little.

len(hex(not()))

# ↓ not() == True

len(hex(True))

# ↓ True == 1

len(hex(1))

# ↓ hex() converts the argument into a hexadecimal string.

len('0x1') #That is 3!!

Four

len(str(not())) # 4

This is easy if you understand 3! The 4-character True is set to str type, and the length is acquired by len.

len(str(not()))

# ↓ not() == True

len(str(True))

# ↓ True is converted to a character string.

len('True') # 4!!

Five

len(str(not[()])) # 5

This is the False version of 4.

len(str(not[()]))

# ↓ not[()] == False

len(str(False))

# ↓ False is converted to a character string.

len('False') #5

#6

len(str(complex(not(),int()))) # 6

It’s a headache from here: scream_cat: complex takes the real part as the first argument and the imaginary part as the second argument, and returns a complex type complex number. I think it is easier to understand this too.

len(str(complex(not(),int())))

# ↓ not() is True, int() is 0

len(str(complex(True, 0)))

# ↓ True is 1

len(str(complex(1, 0)))

# ↓ Returns a complex type complex number with real part 1 and imaginary part 1, (1+0j).

len(str((1+0j))) # len( str( (1+0j) ))

# ↓ (1+0j) is set to str type.

len('(1+0j)') # () is 6 characters, so 6 is returned.

#7

len(str(bytes(len(((),))))) # 7

(((( ；゜Д゜))) It trembles for a moment, but if you look closely, you can see that 1 is a bytes type. Regarding the string conversion of bytes type, there is also a sample code in here as described in @Raclett3’s article.

len(str(bytes(len(((),))))) # bytes is len( ((), ))

# ↓ In other words, the length of the tuple with 1 element is the argument of bytes.

len(str(bytes(1)))

# ↓ Make 1 a bytes type.

len(str(b'\\x00') # \\ is an escape sequence, so it is 1 character.

# ↓ b'\\x00' is made into a character string and the length is obtained.

len("b'\\x00") # 7!

#8

len(str(((),()))) # 8

There are a lot of () in the second half, so it looks complicated…

len(str(((),()))) # len( str( ((),()) ))

# ↓ "A tuple with two empty tuples as elements" is the argument of str().

len('((),())') # A tuple with 2 elements is converted to a string as it is, and the length 8 of the string is returned.

#9

len(str(([int()],[]))) # 9
len(str(((int()),()))) # It's hard to see all tuples

It’s finally the end. This is the same as 8 :honeybee:

len(str(([int()],[])))

# ↓ int() == 0

len(str(([0],[])))

As with #↓ 8, the argument of str is ([0],[]), so...

len('([0],[])') # 9!!!
