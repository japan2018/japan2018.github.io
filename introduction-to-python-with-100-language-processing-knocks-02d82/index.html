for statement

Now that we understand the list, we’ll deal with a for statement that we can use to iterate over something. For example, when calculating the total sum of array elements in C, it was written as follows.

int i;
int squares[6] = {1, 4, 6, 16, 25, 36};
int total = 0;
for(i = 0; i <6; i++) {
    total += squares[i];
}

The Python for statement looks like this:

total = 0
for square in squares:
    total += square

In each loop, one element of squares is taken out and assigned to square. That is, square=1, square=4,, and so on. The for statement looks like this:

A variable in list representing a for element or something: TAB processing content

I am fooling myself into saying “list” just after “in”, but the string type is also OK (because each character can be regarded as an element of the string). The generic term for what can be placed immediately after the in of a for statement is iterable (object)(iterable).

Note that indentation was optional in C but mandatory in Python!

Tuple packing and sequence unpacking assignment method

A tuple pack is a method of collectively assigning multiple values separated by , to one variable by using the specification that can be described by omitting the tuple (). By contrast, assigning a sequence to multiple variables together is called unpacking the sequence.

>>> a ='a'
>>> b ='b'
>>> t = a, b
>>> t

('a','b')

>>> x, y = t
>>> print(x)
>>> print(y)

b

Dictionary type

Implement a function that accepts arguments x, y, z and returns the string “y at z is z”. Furthermore, confirm the execution result by setting x=12, y=“temperature”, z=22.4.

It is difficult to do this by combining strings with +. There are three ways in Python.

printf format

>>> x = 12
>>> y ='temperature'
>>>'%s at %d' %(x, y)

'12 o'clock temperature'

str.format() (Python 2.6 or later)

>>>'{}' when {}'.format(x, y)

'12 o'clock temperature'

f-string (Python3.6 or later)

>>> {y}' when f'{x}

'12 o'clock temperature'

Which one should I use?

Basically, f-string is the easiest. However, backslash cannot be used in {}. Also, printf format seems to be faster when specifying the number of digits (Reference) With this, you can do No. 07 without any hesitation.
