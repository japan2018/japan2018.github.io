Add words to GiNZA(SudachiPy)’s user dictionary

Jan 2, 2020 Py Teun Ubuntsu Sudachi Ginza 

#Introduction

I’ve been involved in natural language processing for a few weeks, and until now I’ve often used R and MeCab, but recently I’m moving to hear that Python’s GiNZA is an excellent tool for natural language processing. First of all, I needed to add words that are not in the dictionary, so I tried to summarize the flow.

#environment

  • Ubuntu 16.04
  • Python 3.6 (pyenv/anaconda virtual environment)
  • GiNZA 2.2.0

#0. Install GiNZA It can be easily installed with the pip command.

$ pip install "https://github.com/megagonlabs/ginza/releases/download/latest/ginza-latest.tar.gz"

#1. Prepare a dictionary It seems that “Sudachi Py” is running for morphological analysis of GiNZA. Therefore, adding words is basically the same as adding “Sudachi” to the user dictionary. First of all, prepare a dictionary with a csv file. The format of the dictionary is Heading, left concatenation ID, right concatenation ID, cost, heading, part of speech 1,2,3,4, part of speech (adjuvant type), part of speech (adjuvant), reading, normalized notation, dictionary ID, division type, A Unit division information, B unit division information, unused Arrange in order.

$ vim add_term.csv
Anna and the Snow Queen, 4786,4786,5000, Anna and the Snow Queen, noun, proper noun, general, *,*,*, Anat Yukinojo, Anna and the Snow Queen, *,*,*,*, *

Each item is described in detail in here, but this time I assigned the recommended ID and cost .. Enter “*” for unnecessary items. Since examples are posted on the above site, I think that you should choose a close one from there.

  • Create a csv file with “UTF-8”.

#2. Create a dictionary SudachiPy is also installed when you install GiNZA, so build the dictionary using the sudachipy command. The command is sudachipy ubuild -s [path of system dictionary] [path of created csv file].

$ sudachipy ubuild \
  -s .pyenv/versions/anaconda3-5.2.0/envs/ginza/lib/python3.6/site-packages/sudachidict/resources/system.dic \
  add_term.csv

I have a long path because I build the environment with pyenv, but please change it according to your environment. When executed, the following message will be displayed.

reading the source file...2 words
writing the POS table...2 bytes
writing the connection matrix...4 bytes
building the trie...done
writing the trie...1028 bytes
writing the word-ID table...14 bytes
writing the word parameters...16 bytes
writing the word_infos...96 bytes
writing word_info offsets...8 bytes

If successful, “user.dic” will be added to the current directory.

#3. Add the created user dictionary to the SudachiPy configuration file Add the generated user.dic path. Add the path to the following location in the configuration file.

$ vim ./pyenv/versions/anaconda3-5.2.0/envs/ginza/lib/python3.6/site-packages/sudachipy/resources/sudachi.json

{
  "characterDefinitionFile": "char.def"
  "userDict": ["user.dic path"] # add user.dic path here
  "inputTextPlugin" :[
  ...
  ...

  • use with GiNZA in python
import spacy

nlp = spacy.load('ja_ginza')
doc = nlp ('I want to go see Anna and the Snow Queen')

for sent in doc.sents:
    for token in sent:
        print(token.orth_)
