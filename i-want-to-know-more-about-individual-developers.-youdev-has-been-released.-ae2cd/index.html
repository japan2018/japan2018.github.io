Production environment

Like this.

I used MySQL because DB hadn’t used PostgreSQL much. Actually I wanted to use RDS of AWS, but since it is not a monetizing service, I installed it quietly in EC2. But I still want to use RDS. .

I wanted to avoid using S3 if possible, but I decided to use S3 because there is image data that the user will register. In the first place, the storage capacity of EC2 is only 8GB, so I made it S3 quietly here as well.

The source code is managed by private github.

#Using Amazon S3 with Django Immediately after the production environment started, I used the media files in the same directory as the static of EC2, so I have to move them to S3 first. .

Create S3 bucket

Really do S3 bucket creation.

There were basically no stumbling points here.

  1. Create bucket
  2. Add IAM user (You may want to copy the access key here.)
  3. Creating a group (policy applies AmazonS3FullAccess) Add the user created in 4.2 to the group created in 3.
  4. Add the user created in 2 to the bucket user

Transition to S3

First, check the authentication below

$ aws configure list

      Name Value Type Location
      ---- ----- ---- --------
   profile <not set> None None
access_key <not set> None None
secret_key <not set> None None
    region <not set> None None

If it is not set, you can not connect to S3, so set the access key and secret key of the user created earlier

$ aws configure

AWS Access Key ID [None]: <access key>
AWS Secret Access Key [None]: <secret key>
Default region name [None]: <Nothing entered>
Default output format [None]: <Nothing entered>

Now that you can connect to S3, upload!


$ cd /usr/share/nginx/html/media/ # This is the directory containing the files you want to migrate

$ aws s3 cp --recursive uploads s3://<bucket name>/<folder name>

It’s done!

By the way

  • If you add –recursive, the directory will be the target.
  • If sync is used instead of cp, synchronization is used instead of copy.

Django side settings

You have to upload the media file on Django and change the reference so that it points to S3.

Installed because the following modules are required to use S3 with Django

$ pip install django-storages
$ pip install boto3

Add the following to setting


INSTALLED_APPS = [
    ...
    'storages', # add
]

...

TEMPLATES = [
    {
        'BACKEND':'django.template.backends.django.DjangoTemplates',
        'DIRS': [],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
                'social_django.context_processors.backends',
                'social_django.context_processors.login_redirect',
                
                'django.template.context_processors.media', #Added
                #By adding this, you can refer to S3 from template with {{MEDIA_URL}}.
            ],
        },
    },
]

...

# MEDIA_URL ='/media/'
AWS_ACCESS_KEY_ID ='<access key>'
AWS_SECRET_ACCESS_KEY ='<secret key>'
AWS_STORAGE_BUCKET_NAME ='<bucket name>'

AWS_S3_CUSTOM_DOMAIN ='%s.s3.amazonaws.com' %AWS_STORAGE_BUCKET_NAME
AWS_LOCATION ='media'
DEFAULT_FILE_STORAGE ='storages.backends.s3boto3.S3Boto3Storage'
MEDIA_URL = "https://%s/%s/" %(AWS_S3_CUSTOM_DOMAIN, AWS_LOCATION)

Now you can upload to S3 and browse S3.

Register emoji in mysql

Originally I had to set the charset to utf-8, so I couldn’t register emoji.

Change the configuration file

[mysqld]
character-set-server=utf8mb4

[client]
default-character-set=utf8mb4

restart mysql

sudo service mysqld restart

Change because the table was already created in utf-8

ALTER TABLE targettable CONVERT TO CHARACTER SET utf8mb4;

In the case of Django, it is necessary to set it on the Django side as well.

DATABASES = {
    'default': {
        'ENGINE':'django.db.backends.mysql',
        'NAME':'xxx',
        'USER':'xxx',
        'PASSWORD':'xxx',
        'HOST': ``,
        'PORT':'',
        Add #OPTIONS
        'OPTIONS': {
            'charset':'utf8mb4',
        },
    }
}

You can now output and register pictograms!
