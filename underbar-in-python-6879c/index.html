Python has a lot of libraries, and I often import and use functions. In such a case, if there are multiple return values from the function and there is a part that is not used, you can use underscore and discard it without occupying the memory of the return value

Now you can build a program without wasting memory. This is most likely to be used

Follow function name _

There are four types depending on the location. Moreover, it is defined by the PEP8 coding standard. I didn’t know there was such a definition. https://www.python.org/dev/peps/pep-0008/

It is written in detail in PEP8>Naming Convention. Let’s put together four below

##_function(x): One before function

def _single_leading_underscore(x):
    return n
#weak "internal use" indicator. E.g. from M import *
#does not import objects whose names start with an underscore.

If you write this, it will not be called from from func import* from another place, but it seems that you can call a function with an underbar as a function in class such as classx._func(). Basically, it seems that it is a python’s practice to not accept external references, but to add one underbar to things. Is this also a read measure?

##function(x)_: One after the function

def list_(x):
    return n
#used by convention to avoid conflicts with Python keyword

If you want to use a python reserved word such as list or class for the function name, it seems that you should add one _ at the end and use it.

##__function(x): Two before the class name

def __double_leading_underscore:
     return n
#when naming a class attribute, invokes name mangling
#(inside class FooBar, __boo becomes _FooBar__boo; see below).

If you put two underscores in front of the function in the class, it will call the mangling of the name. Mangled is a name modification, and in short, it doesn’t seem to be treated as ordinary internally. As written in the above example, the variable __boo in class FooBar cannot be called in FooBar.__boo, and must be called _FooBar__boo before it can be called. This will not be accepted from outside, so I think it may be one of the countermeasures.

##__function__(x): Two before and after the function

def __double_leading_and_trailing_underscore__:
    return n
# "magic" objects or attributes that live in user-controlled
# namespaces. E.g. __init__, __import__ or __file__. Never invent
# such names; only use them as documented.

If you add two underbars before and after the function in class, it becomes Magic Method (special method). It’s a story of a special method, but it seems that it is used when you want to change the behavior of an object or to make it move according to the nature of the class. For example __init__ uses the passed arguments. Initializes a newly created object in the class definition. There are many magic methods. I have a list, but it’s full http://www.ops.dti.ne.jp/ironpython.beginner/method.html

Make numbers easier to read _

>>> 1_000_000
Out: 1000000

Reference

In Python 3.6 and above, you can add underscores inside numbers to make them easier to read. You can achieve a comma every 3 digits that is commonly used.

It seems that even if you type an underscore instead of a comma every 3 digits, it will be recognized as a proper number. It is convenient when you want to enter a large number of digits.

Indicates the last value displayed by the interpreter_

>>> 10
Out: 10
>>> _ * 3
Out: 30
>>> _ * 10
Out: 300

I didn’t know this, but when I’m using the python interpreter, it seems that I can get the number displayed previously with the underbar.
