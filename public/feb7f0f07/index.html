[Python] Build a numerical computing environment with pyenv and miniconda3

Apr 6, 2014 Python 

It seems that almost all of Python's numerical calculation libraries have been migrated to Python 3 since 2012, so I tried to build the environment with Python 3.

It seems that pyenv is a standard to install and use multiple versions of Python properly. It’s easy to use Homebrew on a Mac.

brew install pyenv

â†“ Don’t forget to write this as .zshrc.

export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"

eval "$(pyenv init -)"

You can see the list of versions that can be installed with pyenv install.

pyenv install

So I found anaconda and miniconda, so at first I thought about using virtualenv, but I decided to use pyenv alone.

Install miniconda3,

pyenv install miniconda3-3.0.5
pyenv global miniconda3-3.0.5

Maybe I needed pyenv rehash here.

Install other libraries with the conda command.

conda install numpy scipy matplotlib ipython pandas
conda install pyzmq Jinja2 tornado

(The one in the back seems to be necessary for ipython notebook)

You should also need pyenv rehash here. (If you do eval "$(pyenv init -)", it also does pyenv rehash, so if you write this in zshrc, you just need to start another shell.)

You just installed the package that has been built and distributed, so you do not need to build it yourself. great.

What is miniconda3

It seems that building a numerical computing environment for Python is a hassle for everyone, and Anaconda is created as a package containing all of them. Anaconda has a Mac installer, but you can also add the minimal tools called miniconda and add the libraries yourself. This time I used miniconda3 based on Python3.

Another package, Enthought Python Distribution (EPD), is a full package that I used to use. It seems to be Canopy now, but it doesn’t seem to support Python3 yet.
