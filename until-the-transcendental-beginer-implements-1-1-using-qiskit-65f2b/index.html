What is Qiskit

Qiskit is an open source framework for quantum computers. Provides tools for creating quantum circuits and running them on prototype quantum devices and simulations. It is compatible with the quantum circuit (English version) of the quantum tuning machine (English version), and can be used for quantum hardware that complies with this. (From wikipedia)

Implementation

Create a suitable environment. This time, I use Jupyter notebook.

  1. First import the required libraries.
from qiskit import * # For now, import all
from qiskit.tools.monitor import backend_overview
from qiskit.tools.visualization import plot_histogram
from qiskit.providers.ibmq import least_busy
  1. Register your IBMQ account and load your account. I got a warning here but I will ignore it.
IBMQ.save_account('IBMQ token')
provider = IBMQ.load_account()
  1. Register the circuit. Finally make this circuit 
qc = QuantumCircuit(4,4)

You have now registered 4 qubits and 4 classical bits. You might wonder why it’s not just qubits, but save the qubit calculation results in this classical bit.

  1. Add the gate
qc.h(0)
qc.h(1)
qc.x(0)
qc.x(1)

Put the first (q_0) and the second (q_1) of the qubit in the overlapping state with the h gate. Also, the x-gate works like a NOT circuit.

qc.ccx(0,1,2)
qc.cx(0,3)
qc.cx(1,3)

Now add the gates needed for the addition. For details, click here (https://eman-physics.net/quantum/computer4.html)

qc.measure(0,3)
qc.measure(1,2)
qc.measure(2,1)
qc.measure(3,0)

measure is very important, it stores the state of the nth qubit in the nth classical bit, as I wrote in 3. If you write so far,

qc.draw()

Let’s see the circuit using. The gate should be firmly applied.

  1. Select the best Backend Here’s a copy-paste from Others. Forgive me
large_enough_devices = IBMQ.backends(filters=lambda x: x.configuration().n_qubits> 3 \
                                     and not x.configuration().simulator)
backend = least_busy(large_enough_devices)
print("The best backend is "+ backend.name())

You should actually see the optimal backend.

  1. Actually throw the job
backends = ['xxxxx','xxxx','least_busy was backend']
backend_q = IBMQ.get_backend(backend[1])
result = execute(qc,backend_q,shots='1024').result()

Multiple backends are set in backends, and elements are specified in backend[n] of backend_q. I’m throwing a job with execute. Depending on how many shots are calculated, it will take quite a while if you increase the number. It seems that it was 4096 at the maximum.

  1. View the results Finally check the result.

print(result.get_counts(qc))
plot_histogram(result.get_counts(qc))

Now you can check. Plot_histogram is good because it can be visualized!
