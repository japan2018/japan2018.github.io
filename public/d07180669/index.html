[Python] Automates AtCoder directory creation, sample case testing, and submission. atcoder-cli and online-judge-tools

Jan 10, 2020 Python AtCoder competitive programming 

Here are two CLI tools to help you solve AtCoder comfortably.

The environment used in this entry is Windows 10, Python 3. Of course, other OS and languages are also supported.

What you can do

Create a contest directory

  • acc new {contest ID}
  • A contest directory will be created.
    • The directory for the contest is cut off for each problem. Under the problem directory, a main.py file that writes the answers and a test file that describes the input and output of the sample case are created.
  • The directory structure is as follows.

Contest ID/ ├ a/ │ └ main.py │ └ test/ │ └ sample-1.in │ └ sample-1.out │ └ sample-2.in │ └ … ├ b/ │ └ main.py │ └ test/ │ └ … └ …

Run the sample case test

  • In the problem directory you want to test oj t -c "python main.py"
  • Execute sample case test and return result. Success in all cases with AC.

hand in

  • acc s in the problem directory you want to submit
  • Submit the completed code to AtCoder. The submission screen opens in a new browser tab and the results are displayed.

How to introduce

Assumptions

  • node.js is installed
  • Python >3.5 is installed

atcoder-cli is node.js and online-judge-tools is Python, so you need these installations. If you haven’t installed it, Chocolatey is recommended (on Windows, Homebrew on Mac). Future updates will be much easier.

Template settings

By setting a template, you can:

  • Submitting answers with acc s
    • If there is no template, you need to specify the file name.
  • Setting the template of the answer file created when acc new {contestId} is executed
    • You can use the code that you use every time, such as reading console input and Shebang.

Set the template file according to the “Template setting” of the tutorial.

main()

#!/usr/bin/env python3 is Shebang.

AtCoder allows you to select Python2 or Python3 as the Python execution environment. You have explicitly specified that you want to use Python3 using Shebang. If you forget this, you will get an error when submitting that the execution environment cannot be limited to one.

open(0).read().split() is a read of console input.

Config changes

You can see the current global settings with acc config. The default is as follows.

$ acc config
oj-path: C:/Python38/Scripts/oj.exe
default-contest-dirname-format: {ContestID}
default-task-dirname-format: {tasklabel}
default-test-dirname-format: tests
default-task-choice: inquire
default-template:

We will change this.

$ acc config default-test-dirname-format test

Change the test directory name to match online-judge-tools. This will allow online-judge-tools to recognize the test directory and allow automated testing of sample cases.

$ acc config default-task-choice all

Sets the selection method for problems creating directories when running acc new {Contest Id}. With the default inquire, you will have to manually select the problem every time. I want it to create all directories at once, so I’ve set it to all.

$ acc config default-template python

Specify the template you want to use by default. I’m using python, but let’s specify the most frequently used template created in “Template setting”.
