Parallel 1 (No STDOUT required)

This is OK if the target to be executed generates something. The run function is targeted for asynchronous execution. When I made it for the first time, I called it with run_until_complete, so I was wondering if create_subprocess_shell would just wait.

import asyncio
from asyncoio.subprocess import DEVNULL

async def run(param):
    # 2 or less will work
    p = await asyncio.create_subprocess_exec(*["some", "command", param],
           stdout=DEVNULL, ...)
    p = await asyncio.create_subprocess_shell("some command %s" %param,
           stdout=DEVNULL, ...)

    await p.wait()

if sys.platform.startswith('win'):
    loop = asyncio.ProactorEventLoop()
    asyncio.set_event_loop(loop)
else:
    loop = asyncio.get_event_loop()

funcs = asyncio.gather(*[run(p) for p in parameters])
loop.run_until_complete(funcs)

Parallel 2 (needs STDOUT)

If you want to do something by using the standard output of the execution target, here. However, the result will be returned to main’s ret in the order in which it was finished, so it must be endured.

import asyncio
from asyncoio.subprocess import PIPE
import sys

async def run(param):
    p = await asyncio.create_subprocess_shell("some command %s" param,
            stdout=PIPE, ...)
    return await (p.communicate())[0].decode('code').splitlines()

async def main():
    funcs = asyncio.gather(*[run(p) for p in parameters]*)
    for f in asyncio.as_completed(funcs):
        ret = await f

if sys.platform.startswith('win'):
    loop = asyncio.ProactorEventLoop()
    asyncio.set_event_loop(loop)
else:
    loop = asyncio.get_event_loop()

loop.run_until_complete(main())
