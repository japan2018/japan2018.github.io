# -*- coding: UTF-8 -*-
import math
import matplotlib.pyplot as plt
import matplotlib.animation as animation
from IPython.display import HTML

fig = plt.figure()

ims = [] # video = list that stores a set of still images
for i in range(360):
    rad = math.radians(i)
    x1, y1 = math.cos(rad), math.sin(rad)
    x2, y2 = math.cos(rad * 2), math.sin(rad * 2)
    im = plt.scatter(x1, y1) # Still image part 1 (not list type)
    im2 = plt.scatter(x2, y2) # still image part 2 (not list type)
    im3 = plt.plot([x1, x2], [y1, y2]) # Still image part 3 (list type)
    image = [im, im2] + im3 # One list represents one still image
    ims.append(image) # Add one still image

# Convert to video
ani = animation.ArtistAnimation(fig, ims, interval=10, repeat_delay=1000)

ani.save("Animation1.gif", writer='pillow') # Save as gif file
HTML(ani.to_jshtml()) # Display on HTML

Function to get the coordinates to draw a straight line between two points

In order to draw a straight line between two points in drawing a graph, create a function that calculates coordinates from a pair of edges (pair of vertices).

def get_edges(routes):
    edges = []
    for route in routes:
        if len(route) == 2:
            town1, y1, x1 = [value for value in japan.values][route[0]]
            town2, y2, x2 = [value for value in japan.values][route[1]]
            edges.append([[x1, x2], [y1, y2]])
    return edges

Depth-first search

Let’s do depth-first search as the first of the graph search algorithms

Function 1 to get neighbor list

In the graph search, it is very important to create a “neighborhood list” (which vertex can go to which vertex). "Connect edges between vertices less than a certain distance (threshold)“ I would like to go with that policy. Basics of graph theory If you use numpy and distance matrix which are not used, it can be calculated as follows.

def neighbor(town, dist_mat=dist_mat, threshold=1): #function 1 to get neighbor list
    return [x[0] for x in enumerate(np.where(dist_mat[town] <= threshold, True, False)) if x[1]]

The usage example is like this.

neighbor(12) # Which city is within 1 distance from Tokyo?

[7, 8, 9, 10, 11, 12, 13]

Graph search function 1In Basics of graph theory, I used the while statement to solve the graph search problem. I have defined the following function to advance one step of the search.

def traverse(i=0): # One step of depth-first search
    if len(stack) != 0: # if the stack is not empty
        next_town, current_town = stack.pop() # Get next route (current location and next city)
        current_direction = [[next_town, current_town]] # for drawing
        if next_town not in visited_towns: # if the next city is not visited
            used_routes.append([next_town, current_town]) # register the route
            visited_towns.append(next_town) # Mark as visited
            for nei in neighbor(next_town): # Take out the cities adjacent to the visited city one by one
                if nei not in visited_towns: # if not visited
                    stack.append([nei, next_town]) # put path on stack
    return current_direction # for drawing

Function 2 to get neighbor list

In the depth-first search above, the “last city on the stack” is the next destination candidate. When you move to a city, the adjacent city(s) will be in the stack. At this time, the behavior changes even in the same depth-first search by considering the stacking order. Let’s modify so that cities with short distances are preferentially selected as destination candidates.

import numpy as np
def neighbor(town, dist_mat=dist_mat, threshold=1): #function 2 to get neighbor list
    return np.argsort(dist_mat[town])[1:np.where(dist_mat[town] <= threshold, 1, 0).sum()][::-1]

Breadth-first searchNext, let’s do a breadth-first search. The basic algorithm is almost the same, just stack (first in, first out) into a queue (queue: first in, first out).

Graph search function 2

Rewrite the function traverse. The only part to be rewritten is the one point shown as the “change.” Since the list used as a stack changes to a queue, I would like to change the variable name stack, but if you do so, you will have to rewrite a lot, so let’s leave stack as it is.

def traverse(i=0): # One step of breadth-first search
    if len(stack) != 0:
        next_town, current_town = stack.pop(0) # Changes
        current_direction = [[next_town, current_town]]
        if next_town not in visited_towns:
            used_routes.append([next_town, current_town])
            visited_towns.append(next_town)
            for nei in neighbor(next_town):
                if nei not in visited_towns:
                    stack.append([nei, next_town])
    return current_direction

Function 3 to get neighbor list

We will also rewrite the function that gets the adjacency list, but basically it is almost the same as the previous one. The stack becomes a queue, so you just flip the order.

import numpy as np
def neighbor(town, dist_mat=dist_mat, threshold=1): #function 3 to get neighbor list
    return np.argsort(dist_mat[town])[1:np.where(dist_mat[town] <= threshold, 1, 0).sum()] # change only the end

#Best-first search and minimum tree

Finally, the best-first search. In the above two searches, when adding to the stack (or queue), we added that cities with short distances are preferentially taken out. Best-first search sorts the entire stack (actually the queue) after it has been added, so that cities with shorter distances are given priority. The result is a “smallest tree”.

Stack reordering function

The only change we’ve made so far is to re-sort the entire stack.

def sort_stack(stack):
    return [stack[i] for i in np.argsort([dist_mat[edge[0]][edge[1]] for edge in stack])]
