get rid of heavy pyls in vim-lsp

Dec 8, 2019 Python Vim LSP 

# vim-lsp and pyls

I tried switching Vim’s Python input completion from jedi-vim to vim-lsp, but the operation is heavy. And after a while after editing a Python file, there are frequent automatic downloads of .py files on OneDrive that have nothing to do with the file you’re editing.

Looking at the process, pylint was a sub process of pyls.exe and it was running with CPU full open and eating up memory. It seems that he’s touching .py files in various places. What are you doing…

Apparently, the pyls plugin pylint starts up automatically and behaves unintentionally. I wonder what is the default. (Is it because of my settings and environment?)

It seems that the plugin can be disabled by setting enabled(enabled instead of enable), so stop with leaving only jedi for completion. I chose This seems to work.

if executable('pyls')
    call lsp#register_server({
        \'name':'pyls',
        \'cmd': {server_info -> ['pyls'] },
        \'whitelist': ['python'],
        \'workspace_config': {'pyls': {'plugins': {
        \'mccabe': {'enabled': v:false },
        \'preload': {'enabled': v:false },
        \'pycodestyle': {'enabled': v:false },
        \'pydocstyle': {'enabled': v:false },
        \'pyflakes': {'enabled': v:false },
        \'pylint': {'enabled': v:false },
        \'rope_completion': {'enabled': v:false },
        \'yapf': {'enabled': v:false },
        \
        \'jedi': {'extra_paths' :[] },
        \'jedi_completion': {'enabled': v:true,'include_params': v:true },
        \'jedi_definition': {'enabled': v:true,'follow_imports': v:true,'follow_builtin_imports': v:true },
        \'jedi_hover': {'enabled': v:true },
        \'jedi_references': {'enabled': v:true },
        \'jedi_signature_help': {'enabled': v:true },
        \'jedi_symbols': {'enabled': v:true },
        \ }}},
        \ })
    autocmd FileType python setlocal omnifunc=lsp#complete
endif

As an aside, vim-lsp has jedi as the complement.
