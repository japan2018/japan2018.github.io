[Python] Steps to develop Django with VSCode

Dec 12, 2019 Python Django beginners VSCode 

2020/03/13 Modified slightly to match VSCode Version 1.43.0

Introduction

A beginner yearned for the word debug, and learned that VS Code can actually do it, and tried setting it. The memorandum. I think that the advantage is that once you set it, you can enter the virtual environment and run the server just by pressing the Debug button. You can save yourself the trouble of going back and forth between the browser and Terminal and VS Code and entering source myenv/bin/activate or python manage.py runserver. A little easier.

Reference URL

Django Tutorial in Visual Studio Code

Environment

Ubuntu 18.04 LTS Python 3.6.9 Django 2.2.8 or 3.0

Steps to develop Django with VSCode

Preparation for Terminal

$ mkdir myproject
$ cd myproject
$python3 -m venv myenv
        # There seem to be various virtual environments, but here is venv one choice.
$ source myenv/bin/activate
        # Now you are in the virtual environment. The prompt in the virtual environment is (myenv)$ below.
(myenv)$ pip install django
        # Django is installed (If you want to specify the version, enter django==2.2.8)
(myenv)$ django-admin startproject config.
        # The project finally started. The name is config. The reason is from the "Django textbook that can be used in the field" book.
        # Don't forget the last ".". If you don't add it, a directory will be created and it will be a little troublesome.
(myenv)$ deactivate
        # Exit the virtual environment

The current directory structure is as follows

.
├── config
│  ├── __init__.py
│  ├── settings.py
│  ├── urls.py
│  └── wsgi.py
├── manage.py
└── myenv

Prepare development environment with VS Code

  1. Start VSCode, click Open Folder… from File, and open the myproject directory.

  2. Install Extention of Python

  3. Select Command Palette… in View and click Python: select interpreter

  4. Select the interpreter of the virtual environment (myvenv in this case)

  5. Click the insect icon on the far left, then click the gear icon and select Python

    • Gear icon disappeared from Version 1.43.0. In that case, click the link of Create a launch.json file → select Python from the display.

  6. Select Django from the display

  7. Press the green triangle button on the right of DEBUG to run server automatically

  8. When you fly to 127.0.0.1:8000 with a web browser, the usual rocket launch

This saves you the trouble of launching multiple Terminals, selling commands every time, and going back and forth between windows. Convenient. I realized that VSCode is a development environment, not just an editor, and it’s amazing.
