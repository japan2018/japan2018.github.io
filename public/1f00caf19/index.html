Why I tried to write this article

I will explain the background. Since I opened Windows 10 in a hurry, I ran without checking the Visual Studio Comminity 2019 update and the application update from Microsft Store also as a security update, and python did not start from Git bash. ..

In the first place, I did not even intend to install python for the Microsoft Store version, but because the python execution from Git Bash caused an error, in the process of searching for the cause the installation status of python (including the remains of the past) investigated.

Classification of installer

version installer type installed path
python 3.7.5 Visual studio community 2019 installer “C:\Program Files (x86)\Microsoft Visual Studio\Shared\Python37_64"
python 3.8.2 Microsoft Store (alias) “%USERPROFILE%\AppData\Local\Microsoft\WindowsApps"
python 3.6.5 Anaconda (Just me) “%USERPROFILE%\Anaconda3”
(All users) “C:\ProgramData\Anaconda3” [I personally changed to “C:\Anaconda" ]
python 3.7.4 R’s reticulate installation of miniconda “%LOCALAPPDATA%\r-miniconda"
python 3.5.2 Official exec installer “%LOCALAPPDATA%\Programs\Python\Python35"

Note:

  • %LOCALAPPDATA% is C:\Users\your_login_name\AppData\Local
  • %USERPROFILE% is C:\Users\your_login_name

Edit environment variables and specify python to be used preferentially

Press [Window key], type in path and select Edit system environment variables. Environment Variables -> Double click on the variable PATH. Adjust the python path that you want to prefer with up.

It should be noted that Windows 10 contains the ultimate horrible bug, which means that if you enter environment variables in the GUI (user environment variables and system environment variables), it will not recognize more than 2048 characters. @IT article is very helpful

Why python startup from Git Bash fails with Permission denied and how to handle it.

python -V
bash: /c/Users/your_account/AppData/Local/Microsoft/WindowsApps/python: Permission denied

This means that when you run python -V etc. from Git Bash, if the MicroSoft Store version of python is installed and python is found first in USERPROFILE%\AppData\Local\Microsoft\WindowsApps as the path. It is an error that occurs in.

approach

There is no root workaround for running from Git Bash (including via vscode terminal) due to a bug from MINGW. (Reported on Python official bug tracker,onstackoverflowExplained)

  1. Don’t use Git Bash because it can be executed from cmd and powershell
  2. Install python other than the Microsoft Store version and lower the priority of the path of USERPROFILE%\AppData\Local\Microsoft\WindowsApps.
