Minimum knowledge required for AI engineer

First, I divided the minimum knowledge required for AI engineers into six. Here, when performing work as an AI engineer in the future, roughly divide it into an overview of the six basic knowledge items. ① Programming skills

  • Python (R, C++, Julia, etc. as needed)
  • Indispensable libraries for data analysis such as numpy, pandas, matplotlib
  • Machine learning libraries and frameworks for scikit-learn, TensorFlow, keras (PyTorch, etc.)

Data pre-processing is essential for machine learning, but it is a convenient library for data pre-processing. ② Mathematics

  • Differentiation, linear algebra, vectors, matrices, probabilities, etc. ③ Knowledge of statistics
  • Standard deviation, variance, probability distribution, estimation, test, etc. ④Basic knowledge of machine learning
  • Supervised and unsupervised learning
  • Preprocessing, feature design, learning and evaluation
  • Simple regression, multiple regression analysis, lasso, ridge, least squares, perceptron, logistic regression
  • Decision tree, random forest, support vector machine, K-means, ensemble learning
  • Deep learning implementation skills and knowledge
  • scikit-learn and XGBoost
  • Knowledge of frameworks such as TensorFlow and Keras
  • Flow of making a model already trained with scikit-learn.

(1. Data collection and data pre-processing Completion of missing values and removal of outliers)

  1. Design of features (selection of features)

  2. Model development (model selection and learning)

  3. Model evaluation: Cross-validation, mixed matrix evaluation, etc.

⑤ Knowledge of operating database using SQL

  • select, insert, update, delete, where, like, limit, sum, avg, max, group by, having, order by, table join, view, subquery, case etc. ⓺ Cloud knowledge
  • Knowledge of cloud infrastructure such as AWS, GCP and Azure You might think that there are 6 big ones, and it seems that there are a lot of them, but we recommend that you first focus on two, ① and ④, instead of doing all at once. The reason is that actually writing the program and making it visible makes it easier to continue learning. If you start from theory, you will be frustrated if you are self-taught.

Recommended way to study artificial intelligence by yourself

The knowledge you need was introduced in the previous section, but how can you learn them? To learn ① to ⑥, we recommend that you acquire the knowledge in the following order. Phase 1 Learning Machine Learning Programming and Artificial Intelligence with python Phase 2 Machine Learning Programming Challenge Phase 3 Kaggle Phase 4 Students will also acquire technology such as SQL, scraping, and cloud. Phase 5 teach I think that learning in each of these phases will be the most enjoyable and least burdensome for you. In Phase 1, we refer to those who are new to programming. If you are new to programming, please read through Phase 1. In Phase 2, we will explain the learning methods related to machine learning programming based on what we actually learned in Phase 1. If you are already doing machine learning programming using scikit-learn, you can skip it. Phase 3 describes how to learn practical programming through competitions such as Kaggle. Phase 4 In doing machine learning, data is often retrieved from the database, so knowledge of SQL is essential. Here, we will introduce how to acquire techniques such as scraping (for data collection) and cloud in addition to SQL. Phase 5 Teaching someone can clarify what you didn’t know. Therefore, it is good to teach machine learning to friends etc. to deepen their understanding.
