Flask introduction ~ Blueprint application note

Mar 2, 2020 Python Flask Beginner Python3 For Beginners 

Let's try the title as easily as possible.

Blueprint allows you to implement the functionality of your application in pieces. Blueprint is officially recommended as a way to organize bloated projects.

Install Flask

$ sudo pip3 install flask

Directory structure

$ cd ~/workspace/sample-flask/

$ls
server.py

from flask import Flask

app = Flask(__name__)

@app.route('/')
def func_1():
    return'Hello world'

@app.route('/test')
def func_2():
    return'Test'

if __name__ =='__main__':
    app.run(host='0.0.0.0', port=80, debug=True)

Split with Blueprint

Try splitting “/test” into a separate file. I created a directory called func and created test.py under it.

Directory structure

$ cd ~/workspace/sample-flask/func

$ls
__init__.py
test.py

Import Blueprint, Create a Blueprint object with the cut out test.py.

from flask import Blueprint

Generate #Blueprint object
app = Blueprint('func', __name__)

@app.route('/test')
def func_2():
    return'Test'

Register the divided Blueprint on server.py side.

from flask import Flask
from func import test

app = Flask(__name__)
# Register the divided blueprint
app.register_blueprint(test.app)

@app.route('/')
def func_1():
    return'Hello world'

if __name__ =='__main__':
    app.run(host='0.0.0.0', port=80, debug=True)

Run it and see if the routing is working.

If you access 127.0.0.1 from the browser in the same way as before, it will be displayed as Hello world, Then, when you access 127.0.0.1/test in the browser, Test is displayed this time.

If all goes well, you can use Blueprint to split.
