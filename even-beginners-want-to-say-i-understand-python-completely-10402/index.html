Docstring

I said, “Next is Docstring! It’s a useful function too! Let’s see what it is!”

def hello(name):
    """ Say hello to the person you put in the argument" ""
    return f"Hello {name}"

“I can embed the document in the code by putting a character string surrounded by three quotation marks in a function (a class or module is acceptable) like this!”

I said, “This is what makes it different from just a comment, but the string embedded here is not stored in the special method __doc__ of that object.”

I said, “You can refer to the documentation by using help() in interactive mode, or by using Sphinx. There are lots of great deals such as automatically creating a document using.! It’s a bit different from just a comment.”

hello(name)
    Say hello to the person you put in the argument

I said, “By further embedding multiple lines of Docstring, I can write the type hint I explained earlier in Docstring as well!”

I’m like this!

def hello(name):
    """ Say hello to the person you put in the argument

    Args:
        name(str): Name of the person you want to greet

    Returns:
        str: Greeting
    """
    return f"Hello {name}"

“I can type-check with an IDE, just like I typed a hint!”

I said, “What I wrote above is what is called Google style, but I can write unexpectedly various arguments and return values, and there are other ways to write it in the first place, so if you are interested please check it out! "

I said, “Use it because it’s convenient when you’re developing with a large number of people!”

Summary

  • Store the string in the special method __doc__ by embedding the string surrounded by three quotation marks in the function, class, or module
  • You can use convenient functions that facilitate development such as help() and reference in IDE
  • You can add type hints by writing multiple lines

Reference: PEP 257-Docstring Conventions

##Generator

I said, “Next is a generator! This is a function that has been around for a long time, but it’s a function that I do not know how to use for beginners. It’s a good idea to make a list without making a generator…”

I thought “I thought, but I feel like I can make a big difference in the following two points.”

  • When you want to make an infinite loop without stopping processing
  • When you want to handle a large amount of text data

I “for example…”

while True:
    print("returns value")

I said, “When I write some code, it immediately enters an infinite loop. It’s possible to keep consuming memory and shutting down the system… I’m usually careful not to do it.”

The same thing may happen even if I am making a big list with heavy mathematical calculation processing, not only for infinite loops.

I said, “But if you use a generator like the one below, you don’t have to press the memory from creating a new element until the next element is requested (calling the special method __next__)!”

I said, “There are two ways to create a generator. Specifically, do it like this!”

How to create a #function
>>> def my_gen():
... x = 0
... while True:
... yield x
... x += 1
# The return value is generator
>>> my_gen()
<generator object my_gen at 0x10df04510>

>>> for i in my_gen():
... print(i)
0
1
2
3
...

#Using a generator expression
>>> my_gen = (i for i in range(10000) if i %2 == 0)
>>> for i in my_gen:
... print(i)
0
2
Four
6
8

I said, “The generator can be used in a for statement because the return value is an iterator!”

“I think it’s really convenient when you want to read in a csv with 1 million records and process it!”

By the way, “By the way, generators can be converted to other container objects! But be careful as they can consume a lot of memory when they are created!”

>>> my_gen = (i for i in range(10000) if i %2 == 0)
>>> my_list = list(my_gen)
>>> type(my_list)
<class'list'>
>>> my_list
[0, 2, 4, 6, 8, 10, 12, 14, 16,...9992, 9994, 9996, 9998]

Summary

  • Generator is an iterator that generates a value each time it is called
  • Not all values are expanded, so memory consumption can be suppressed
  • Convenient for handling large amount of data
