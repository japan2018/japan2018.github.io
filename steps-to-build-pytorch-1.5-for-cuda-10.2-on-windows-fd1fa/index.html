Steps to build PyTorch 1.5 for CUDA 10.2 on Windows

Mar 5, 2020 Python PyTorch VisualStudio2019 CUDA10.2 torchvision 

Here are the steps to build PyTorch 1.5 for CUDA 10.2 on Windows 10. (As of March 1, 2020)

It also contains the torchvision build instructions that are often used with PyTorch. Detailed explanations are omitted here, so if you need more detailed explanations, also refer to Blog-dev.infohub.cc. I hope you will.

#Build environment

  • hardware -CPU: AMD Threadripper 3960X -GPU: nVIDIA TITAN RTX
  • Software -Windows 10 Pro Version 1909 (x64) -Microsoft Visual Studio Community 2019 â€“ Ver 16.4.5 -Git for Windows (anything is OK if you can use git command) -Python 3.8.2 (Python.org instead of Anaconda) -CUDA 10.2 -cuDNN 7.6.5 -Intel MKL 2019.0 (AMD CPU is forced to use MKL)

#Setting environment variables

  • MKL_DEBUG_CPU_TYPE=5 (setting to use MKL on AMD CPU)
  • The PATH to other CUDA etc. must pass

#PyTorch, torchvision build Open x64 Native Tools Command Prompt for VS 2019 of Visual Studio 2019 and perform the following steps. (Explanation is in comment format)

# Create virtual environment (build_pt) for build
python -m venv g:\work\build_pt
g:\work\build_pt\Scripts\activate.bat
cd /d g:\work\build_pt

# Update packages etc.
python -m pip install --upgrade pip

# Introducing packages required for build
pip install numpy ninja pyyaml mkl mkl-include cmake cffi wheel

# Get source
#Unlike the steps on the main site, you will get the submodule later
git clone https://github.com/pytorch/pytorch
cd pytorch

# 2020/03/01 Switch to the state at 13:24 (latest state tried this time)
git checkout ace2b4f

#Get submodule
git submodule sync
git submodule update --init --recursive

# Build (18 minutes with Threadripper 3960X)
python setup.py install

# wheel package creation (pip put the wheel)
# "Torch-1.5.0a0+ace2b4f-cp38-cp38-win_amd64.whl" is created in G:\work\build_pt\pytorch\dist
python setup.py bdist_wheel

# ---- build torchvision from here ----------------------------------

# Move to parent folder (g:\work\build_pt)
cd ..

Introducing additional packages required to build #torchvision
pip install six pillow

Check if CUDA is used in # torchvision
#If this result is True, CUDA will be used (You can also build to force it, see torchvision page for details)
python -c "import torch; print(torch.cuda.is_available())"

Get the source code of # torchvision
git clone https://github.com/pytorch/vision.git
cd vision

git checkout b2e9565

# Build (a few minutes)
python setup.py install

# Create wheel package
python setup.py bdist_wheel

This will create PyTorch and torchvision wheel files (*.whl) in the dist folder.

#Introduction to usage environment This is the procedure to install the built PyTorch and torchvision wheel packages into individual virtual environments. The build folder uses a lot of disk space, so it is convenient to separate the environment that actually uses PyTorch and the virtual environment for build. Here, it is assumed that you will create a virtual environment called ml under c:\venvs\. It is OK if you execute it from the normal command prompt (OK, not the Visual Studio console)

# Create virtual environment (ml)
python -m venv c:\venvs\ml
c:\venvs\ml\Scripts\activate.bat
cd /d c:\venvs\ml

# Update packages etc.
python -m pip install --upgrade pip

# Introducing PyTorch and torchvision (including dependent packages)
# PyTorch and torchvision whl files specify the file created above
pip install numpy mkl six pillow
pip install "torch-1.5.0a0+ace2b4f-cp38-cp38-win_amd64.whl"
pip install "torchvision-0.6.0a0+b2e9565-cp38-cp38-win_amd64.whl"

This completes the PyTorch 1.5.0a0 environment creation for CUDA 10.2.
