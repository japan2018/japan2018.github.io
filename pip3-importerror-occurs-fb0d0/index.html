[pip3] ImportError occurs

May 3, 2020 Python pip3 

An error occurs when executing the # `pip3` command.

ImportError: cannot import name’pkg_resources’ from’pip._vendor’

Cause

pkg_resources is not in the pip._vendor package

When I checked with the CLI whether the package actually existed, it was found to exist. And when I checked the location of the file, I found that it did not exist in the directory managed by pip.

  • Confirm existence of file
Python 3.7.7 (default, Mar 10 2020, 15:43:33)
[Clang 11.0.0 (clang-1100.0.33.17)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
>>> import pkg_resources
>>> pkg_resources.__file__
'/usr/local/lib/python3.7/site-packages/pkg_resources/__init__.py'
>>>
  • Package directory required when starting pip /Users/username/Library/Python/3.7/lib/python/site-packages/pip/_vendor/

solution

  • Copy package file to directory under pip control
  • Create symlink of package file in directory under pip control

This time, we will deal with the idea of creating a symbolic link. ln -s /usr/local/lib/python3.7/site-packages/pkg_resources /Users/username/Library/Python/3.7/lib/python/site-packages/pip/_vendor/pkg_resources
