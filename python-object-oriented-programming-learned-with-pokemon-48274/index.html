What is object-oriented programming?

“Object” is a direct translation of an object. In other words, we try to program as we make things.

Things should have features or parts and movements. The kind of thing is called “class”, each thing is called “instance”, and the behavior of thing is called “method”.

For example, consider Pokemon Charizard. Charizard has status such as name, level, HP, and attack.

The Charizard races (classes) include Charizards (instances) with various names such as “Rizasuke” and “Rizamaru”.

Also, one of the techniques that Charizard remembers is flame radiation. This is the method.

The code is as follows.

The three main characteristics of object-oriented programming are as follows.

·Encapsulation ・Inheritance Polymorphism

Encapsulation (information hiding)

There was a damage calculation formula in the flame radiation method. It looks like a complicated expression.

However, when you actually execute the method, you can easily execute it simply by writing rizasuke.flame_radiation(fussi). You do not need to know the damage calculation formula.

In the general case, the print() function is the same. It’s a function that can be output easily, but there are very few people who know what the processing is like.

The fact that information and processing inside the class can be hidden in this way is called encapsulation. Encapsulation prevents the internal information from being rewritten, and makes classes and methods easier to understand.

In addition, information such as status is written together in one class, which makes management easier.

inheritance

Object-oriented programming is very convenient just because features and operations can be summarized like the code above, but Pokemon is now said to be 890 types (from wikipedia). Writing each method for Pokemon from scratch is a hassle.

That’s where “inheritance” (override) comes in handy. The way is to put the parent class in the argument of the small class and then put the super(). method you want to inherit .

Here, Pokemon class is set in order to share stats such as HP and attacks, techniques such as flame emission.

By implementing a new Pokemon class (parent class) like the code below and inheriting it to child classes such as Charizard and Kairyu, you do not need to write the status and trick codes twice. With this, it seems that 890 kinds of Pokemon can be easily implemented.

#Parent class
class Pokemon(object):

        self.attack = attack
        self.defence = defence

Polymorphism

Polymorphism is, when translated literally, polymorphism, polymorphism, and diversity. I think diversity is the most familiar.

A major feature of this method is that it can produce different results (various results) even if the same command is issued by using it together with inheritance.

Look again at the code from the inheritance example. In addition to super().flame_radiation(enemy), print('The other party got burned!') is included in the flame radiation method of Kairyu. It is not included in Charizard’s flame-radiation method.

By doing this, the flame radiation of Charizard will not be burned, and the flame radiation of Kairyu will always be burned (the actual Pokemon is not such a specification).
