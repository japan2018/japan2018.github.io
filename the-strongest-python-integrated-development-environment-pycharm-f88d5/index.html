# Strongest Python integrated development environment PyCharm

When learning a new language, a good IDE will be the best teacher, so I think it should be used actively without being stingy. So, let’s take a look at PyCharm, the most powerful integrated development environment in Python. I think it is the most, so I wrote it assuming Web application development with Django.

The data on functions and price system are as of PyCharm 5.0.2 on December 15, 2015.

Overview

https://www.jetbrains.com/pycharm/

PyCharm is a Python IDE from Czech JetBrains s.r.o., famous for its high-quality IDE. Works with Widows / OS X / Linux.

JetBrains IDE is based on a Java IDE called IntelliJ IDEA. PyCharm is developed as its Python plugin. We also develop IDEs such as Ruby, PHP, JS, C++, Objective-C/Swift.

PyCharm also comes with Web development related plugins such as JS, CSS, and HTML, so front-end development can be done in the same environment.

Price

There is a paid Professional Edition and a free Community Edition. Community Edition has the same analysis mechanism of Python itself, but the plugins related to Web development and It should be noted that support for WebApplicationFramework, support for DB, SQL, etc. are omitted.

If you want to do web development, you should use Professional Edition.

There is a 30-day free trial so please try it first.

Price system

JetBrains products are offered on a monthly/annual subscription basis. The price is highest in the first year and gradually decreases as it continues, and the amount is the same from the third year onward.

If you go to the purchase page, there are two plans, PyCharm only plan and All Products Pack including all other IDEs such as IntelliJ IDEA Ultimate. If you are developing iOS apps or use Ruby, PHP, Java, C++, etc., the All Products Pack may be worth your consideration.

PyCharm comes with Web-related features, so you don’t have to buy WebStorm separately if you just want to write client-side code.

Install

https://www.jetbrains.com/pycharm/download/ If you go to Nikkei, you can download the image that suits your platform. Then just install as instructed. The OS X version now also ships with Java, so it will continue to work.

Project

PyCharm manages codes and settings in units of Project. It is also possible to create a project from the start screen, load existing code, and fetch from VCS such as git.

In the following sections, we will introduce the recommended features for project settings.

virtualenv management

In Python, the environment is often divided by virtualenv, so create and set the virtualenv environment for Porject that was created. You can specify an existing virtualenv environment or create a new one when creating a project or by going to Preferences -> Project -> Project Interpreter. A list of installed libraries and the latest version are also available, so it is also convenient for batch checking of library updates.

raise PEP8 warning to warning

The warning level for PEP8 violation, which is the Python standard coding standard, is low, so it is better to raise it to a warning. There is no merit to violate PEP8 because I feel nauseous while watching the code that violates PEP8.

Let’s raise week warning to warning from Preferences -> Editor -> Pythons in Inspections -> PEP8 coding style violation. The naming conversion violation level below that should be raised as well.

Change check of file encoding description

Python will follow the rules at the beginning of the code if the file encoding is something other than ascii. Add the following for its description style and checking. Python -> File contains non-ASCII character in Preferences -> Editor -> Inspections and Encoding comment format as you like. Since it is necessary even if there is Japanese in the comment part, it is good to put it in all files by default.

Project Structure

You can add a reference path route such as a search path, template, or image. If you can’t get out with code completion or jump to Template, check it.

Django Setting

When using Django, specify the position of settings and the position of the root as a Django project.

Docstring format

The format used by Docstring. It works well when using type hinting with comments. There is no problem if you select reStructuredText.

Template Language

Besides Django, it also supports Jinja2, Mako, Chameleon, etc. Let’s specify the Template language to use.

VCS

Supports VCS such as Git, Mercurial, Subversion, etc. If set, changes etc. can be confirmed from the editor, and partial restoration is also supported.

Even if the git submodule or outside of Project is the root of VCS, it will recognize that you add the setting here.

Line number display, blank character display,

It can be found in Preferences -> Editor -> General -> Appearance. In general, PyCharm’s search is very good, so if you put it in the search box in the upper left, it will display nicely.

File template

You can specify the template when creating a new file. Preferences -> Editor -> File and Code Templates.

When using Python2.x, it is good to add the following two items.

Absolute_import will be confused by relative reference if the rule at the time of import is Python 2.x, so let’s make it absolute reference only. If you want to make a relative reference, you can explicitly describe it, and it is preferable to add it so that you do not have to worry about it. This is the same as the default behavior of Python3.x.

unicode_literals makes the default string type unicode. It’s a long time to explain, but it’s better if you let Python3.x work, because it doesn’t cause confusion.

from __future__ import absolute_import
from __future__ import unicode_literals

(when writing code to support multiple versions of Python)

You can select multiple versions to check in Python -> Code compatibility inspection in Preferences -> Editor -> Inspections. It is effective in library development.When writing code that runs on 2.x series, it is better to write code that matches 3.4 or above as much as possible.

Function introduction

Search everywhere

This is the search bar that appears when you press Shift twice. It will incrementally search most things related to Project such as file names, symbols and actions. Method names and file names will be searched by ignoring case delimiters such as upper and lower case letters and underscores, so it’s nice to remember. I think there are quite a lot of PyCharm users who rely on this.

Code analysis

You can jump to the definition with Cmd + click or shortcut. It will even look inside the library so you can feel free to read if you have any doubts about the behavior or arguments. Even if the target can not be specified by meta operation etc., it will display the candidate and code reading will be much easier.

In ToolWindow Project or Structure, you can set the linkage with the specified editor. You can specify the operation by selecting the following with the gear mark in the upper right. (Icon in Structure)

Auto Scroll to code
Display the file (line) selected in ToolWindow on the editor side
Auto Scroll from code
Display the file (line) displayed in the editor with ToolWindow

You can set it to your liking. By the way, the path of the file currently in focus appears at the top of the editor, so you can refer to the file in the middle directory.

Code completion

General code completion does everything possible. Moreover, it works with the same logic as Search everywhere, so I’m happy that it complements the latter half of the method name. To be honest, this is pretty strong.

Also, since it works without importing, if you remember the class name or function name, you can write it for the time being and select a candidate with Alt + Enter and add an import statement automatically.

The menu that comes out of Alt + Enter at the warning is full

  • Conversion of double quotation and single quotation etc.
  • The automatic addition system gives various candidates according to the context. The following is an example. -import -Function/class/argument definition -Library installation -docstring -test
  • Change of expression format. tuple -> list and {} -> dict()

etc.

TypeHinting

It’s a type hint that was recently introduced with Python3.5. PyCharm has a TypeHinting function with comments for several years, and you can perform code completion and refactoring with TypeHinting even if it is not Python3.5.

If type information is lost due to meta operations etc., if you add it with TypeHinting, completion will be effective and argument errors etc will be found, so let’s use it aggressively. I can’t finish talking about TypeHinting, so let’s leave it to another article. The Syntax of TypeHinting that can be used in PyCharm is summarized below. ‥

https://www.jetbrains.com/pycharm/help/type-hinting-in-pycharm.html

Database integration

You can connect to RDB such as MySQL and view/manipulate data from GUI. I’m mainly using SequelPro, so I’m not using it much, but I think it’s useful if you’re not using ORM.

Django/SQL Alchemy support

Supports Django/SQL Alchemy ORM completion. These internal implementations have many meta operations, and it is difficult to complete just by analyzing static code, but it is possible to jump to completion and definition.

Template & HTML & JS

You can move from View to Template seamlessly, and complete HTML/CSS/JS completion. It also properly analyzes dependencies between Templates and paths such as images.

It also supports TypeScript and CoffeeScript, so PyCharm is very useful even for clients who do not write server side.

Debugger

PyCharm comes with a debugger, and you can do almost anything you can with a typical debugger, such as stepping, setting breakpoints, stopping and changing variables, and Watch. This debugger is also caught in Template. Of course you can step. The stack trace in the Template is hard to see, so it’s quite appreciated for complicated Templates.

Remote debugger

You can ssh it and run it in a remote server environment with a debugger. It is quite effective for investigating environment-specific problems.

Other

There are too many functions to write, but the accuracy of code analysis and completion is quite good, and the fact that it has all the functions from the beginning and does not require environment construction is a major reason why we recommend PyCharm. Searching, replacing, and automatic refactoring will work pretty fast once the indexing is done on the first project load, and they all work together in a stress-free manner.

I often take out only a part of the function and substitute it, I see a comparison of only the function list, but in the development environment, I think that it is most important that the code analysis area to use most and the search work fast and comfortably .. In that respect, PyCharm will give you a memorable experience.
