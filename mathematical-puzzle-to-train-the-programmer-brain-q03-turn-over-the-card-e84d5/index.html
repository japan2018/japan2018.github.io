Mathematical puzzle to train the programmer brain Q03 Turn over the card

Apr 7, 2020 Python array list comprehension pass by value mutable 

# Problem overview

The first card is 100 cards, the first card is every other card, the second card is every two cards, and so on.

#Code

#Q03 Turn over the card

cards0 = [-1]*100 #First. 1=front, -1=back
cards = [-1, 1]*50 # second
step = 3 # How many sheets to turn over

Turn the card over every #step. End when there are no cards to turn over
while cards0 != cards:
    cards0 = cards[:]
    for i in range(step-1, len(cards), step):
        cards[i] *= -1
    step += 1

#Display how many cards are turned over
backs = [i+1 for i in range(len(cards)) if cards[i] == -1]
print(backs)

Mutable and immutable

Depending on the object, the value can be changed. An object whose value can be changed is called a mutable. An object whose value cannot be changed after it is created is called immutable.

https://docs.python.org/ja/3/reference/datamodel.html

Mutable: list, dictionary, set, etc. Immutable: numbers, strings, tuples, etc.

What that means is that if you try to change the value of an immutable object, a new object will be created. Mutable objects change the values of existing objects. Therefore, the behavior when the variable is copied is different.

a=b=1
print(id(a)) # 140716556333904
a=2
print(id(a)) # 140716556333936 ★A new object is created
print(b) #1 ★a changes do not affect b

a=b=['hoge']
print(id(a)) #2667037271944
a[0] ='fuga'
print(id(a)) #2667037271944 ★Existing object is changed
print(b) #['fuga'] ★A change affects b

##List comprehension

backs = [i+1 for i in range(len(cards)) if cards[i] == -1]
print(backs)

Is the same as

backs=[]
for i in range(len(cards)):
    if cards[i] == -1:
        backs.append(i+1)
print(backs)
