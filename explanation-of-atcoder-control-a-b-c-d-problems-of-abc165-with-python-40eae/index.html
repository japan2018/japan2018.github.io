ABC165A “We Love Golf”

Problem Page: A-We Love Golf Difficulty: ★★☆☆☆ Points: Mathematics and handling of while

If you know how to make all multiples of k less than or equal to b, you can solve it.

How to solve

  1. Think about what to do

Step 1: Read the question

An easy way to come up is to enumerate all the multiples of k less than or equal to b and say “OK” if there is at least one that is greater than or equal to a. I’m ready to start writing code.

However, it is easier to use the method of “finding the maximum multiple of k below b, and if it is a or more, "OK"” written in the official commentary PDF, so if you come up with it Is good. I didn’t come up with it.

code

I will write two methods.

k = int(input())
a, b = list(map(int, input().split()))

x = 0

#I feel like I can write it a little more neatly, but for the time being, infinite loop + break
while True:
    x += k
    if a <= x <= b:
        print("OK")
        break

    if x> b:
        print("NG")
        break


k = int(input())
a, b = list(map(int, input().split()))

# Find "the largest multiple of k less than or equal to b"
largest = (b // k) * k

if a <= largest:
    print("OK")
else:
    print("NG")

ABC165B “1%”

Problem page: B-1% Difficulty: ★★☆☆☆ Point: Handling while, reading the problem statement correctly

Just do it exactly as it is written, but there is an important thing in parentheses: ** “Discard money after the decimal point every year” **.

In C++, etc., you need to be careful about overflow, but in Python, you don’t have to worry about it. As expected, it is Python.

How to solve

  1. Think about what to do

Step 1: Think about what to do

Initially you have 100 yen, and you will increase by 1% per year with compound interest. You can solve it by doing this way.

If you read the question carefully, you will find something very important in parentheses (compound interest, round down decimal points).

Therefore, let’s round off the decimal places.

code

If you can read the problem statement properly, all you have to do is write it.

Even if you miss the rounding down to the right of the decimal point, if you check the sample, the answer will be incorrect, so be aware that something is wrong. I noticed.

Specifically, the output of Sample 2 is 3760, the output is 3703, and the output of Sample 3 is 1706, 1649.

Sample 2 is the maximum constraint of $10^{18}$, so I can be confident that there will be no error if this passes.

x = int(input())

year = 0 # answer
money = 100 # initial value of 100 yen

Turn the loop until it is #x or more
while money <x:
    money *= 1.01
    money = int(money)
    year += 1

print(year)

ABC165D “Floor Function”

Problem Page: D-Floor Function Difficulty: ★★★☆☆ Type: Math

If you are good at mathematics, it is a problem that you can intuitively find the answer. For those of us who aren’t good at mathematics, you’ll find out by experimenting.

things to do

  1. Try and think for the time being

Step 1: Try and think for the time being

If you have a problem like this, try assigning numbers for the time being.

I’m happy that $floor(Ax/B)$ is large and $A\times{floor(x/B)}$ is small.

We see that the latter grows when $x$ is exactly a multiple of $B$. So, if you try various things, you can see that $f(x+B)=f(x)$.

Therefore, $x$ need only be considered in the range of $0 to (B-1)$. At this time, the one to be drawn is $A\times{floor(x/B)} = 0$, so you can forget about its existence.

Now, $floor(Ax/B)$ gets bigger as $x$ gets bigger, so $x$ should be $B-1$. However, $x$ is less than $N$, so the smaller of $N$ and $B-1$ is the answer.

It’s just a type of problem if you know it.
