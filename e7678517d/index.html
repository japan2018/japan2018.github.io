[Python] How to use lambda in Python

Mar 22, 2020 Python 

#Introduction

When you hear lambda, you might think of AWS lambda, I’m going to talk about lambda, which stands for anonymous function in Python.

What is a Python lambda?

In a nutshell, you can define a function without using a def statement.

I think that the def statement has the following structure.

def function (argument):
    return expression

With lambda you can write:

function = lambda argument: expression

How to use

I will describe how to use this lambda.

def statement

First of all, I would like to create a program with def statements without using lambda. As a content, prepare a list containing lowercase and uppercase alphabets, Use the capitalize function to correct all capital letters.

l = ['A','b','C','d','e','F','g']

def change_words(words, func):
    for word in words:
        print(func(word))

def sample_func(word):
    return word.capitalize()

change_words(l, sample_func)

In the change_words function The argument is set to func that passes words and function. In the processing inside, words are rotated in a for loop and the value is put in func(). It changes the word according to the definition of func.

In the sample_func function With word as the argument, inside the beginning of the string capitalize() The process is such that it is returned using a method that capitalizes.

You can see that the output is in all uppercase when you actually run it.

lambda

Let’s write the above program using lambda.

l = ['A','b','C','d','e','F','g']

def change_words(words, func):
    for word in words:
        print(func(word))

#def sample_func(word):
# return word.capitalize()
sample_func = lambda word: word.capitalize()

change_words(l, sample_func)

It becomes possible to describe it in one line like this. If you try it, you will get similar results.

It is better not to name the lambda expression

This time, in order to make it easy to understand, I named it sample_func,

Python coding standard PEP8 It is recommended that def be used when defining a function by name.

Therefore, it is possible to write it directly and pass it as a function without naming it.

l = ['A','b','C','d','e','F','g']

def change_words(words, func):
    for word in words:
        print(func(word))

#def sample_func(word):
# return word.capitalize()
# sample_func = lambda word: word.capitalize()

change_words(l, lambda word: word.capitalize())

With this description, you can use the function without defining it one by one, I think it will be a beautiful program.

More effective

To realize the effect of lambda more Now let’s add a function that modifies the alphabet to lower case.

lambda

Therefore, it is easier to write it in one line using lambda.

l = ['A','b','C','d','e','F','g']

def change_words(words, func):
    for word in words:
        print(func(word))

change_words(l, lambda word: word.capitalize())

change_words(l, lambda word: word.lower())

Executing this will output the same result.

If you use function as argument and define it in () using lambda, Because you don’t have to define the function with def It allows you to write very efficient programs.

Let’s combine with map and filter

Combine the lambda explained above with the built-in functions map and filter. It makes it more efficient.

Let’s use the map function here.

When actually combined with map Change the word according to the definition of func. The change_words function no longer needs to be defined with def.

l = ['A','b','C','d','e','F','g']

change_words = map(lambda c: c.capitalize(), l)

print(list(change_words)

Use list comprehension as much as possible

So far, I have explained how to handle lambda and map, but The book Effective Python recommends using list comprehensions instead of these. (* See item 7 on page 15 for details.)

Below is the program using the list comprehension.

l = ['A','b','C','d','e','F','g']

change_words = [c.capitalize() for c in l]

print(change_words)

There are two main reasons for recommending list comprehension in the book.

The first is readability.

In list comprehension, extra lambda expressions are used compared to the built-in functions map and filter. It is described as clear because it is not required.

Certainly, even if you look at the above program, the list comprehension is more It looks neat and easy to write.

Second, list comprehension is easier to extract elements from the input list.

For example, suppose you want to output only uppercase B from the list.

With list comprehension, all you have to do is add a conditional statement after the loop.

l = ['A','b','C','d','e','F','g']

change_words = [c.capitalize() for c in l if c =='b']

print(change_words)

However, in the case of map, it cannot be described without the help of filter.

l = ['A','b','C','d','e','F','g']

change_words = map(lambda c: c.capitalize(), filter(lambda c: c =='b', l))

print(list(change_words))

You can do that with filter, but it’s not a very readable program.

From the above viewpoint, list comprehension notation is good instead of map and filter.
