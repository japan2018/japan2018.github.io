Qiita: Python lambda is easy to understand

What is #lambda expression? The lambda expression is used to create an anonymous function. As the name implies, an intangible function is a function that has no name. Let’s create a function that calculates the sum of two values, an ordinary function and an anonymous function, and compare the differences.

First is an ordinary function. Define the function with the function name as add.

# Ordinary function
def add(a, b):
  return a + b

# Call the function from the function name "add".
print(add(1, 2)) #==> 3 is output

Now let’s do the same with an anonymous function. I have defined an anonymous function and assigned that function object to a variable (add_lambda). ~~ When using an anonymous function, call the function object from the variable. The anonymous function is assigned to the variable because the anonymous function has no function name to call, so it cannot be used unless it is assigned to the variable. ~~ This time, we are using an anonymous function by assigning it to a variable, but you can also use an anonymous function without using a variable. (Thank you for pointing out @konandoiruasa.)

# Assign the anonymous function object to a variable (add_lambda).
add_lambda = lambda x, y: x + y

Call the function object from the # variable.
print(add_lambda(1, 2)) #==> 3 is output

As you can see, the two examples do the same thing. So why is there a lambda? This will be discussed next.

Why you need a lambda function

**The conclusion is that it may be easier to write. **

Then, I would like to explain in what case it can be easily described.

In Python, you can create a function that can receive a function as a function argument. As an example, let’s consider a function (return_result) that displays the result of the received function.

### Function of disp_result ###
Receives the function (func) and the arguments (*args) used by func as the # argument
Display the result of #func(args)
def return_result(func, *args):
  return func(*args)

Compare the return_result with a function that returns two sums as an argument, and compare it with an ordinary function and with an anonymous function.

First, let’s look at the ordinary function.

def add(a, b):
  return a + b

print(return_result(add ,1, 2)) #==>3 is displayed.

Now let’s look at the case of using anonymous functions.

print(return_result(lambda x, y: x + y, 1, 2)) #==>3 is displayed.

Writing an anonymous function with lambda makes it easier to write because you do not have to define the function separately.

In this way, when using an anonymous function in a function (return_result) that takes a function (func) and arguments (*args) as arguments, the description can be simplified as much as the function definition is unnecessary. However, if you want to use a complicated function as func, you need to define the function separately instead of lambda.

#How to use lambda expression# The usage of lambda is as follows.

#lambda argument a, argument b: Processing using argument a and argument b
func_lambda = lambda a, b: a + b

Addendum (added by your note): How to use anonymous function without using variable

sum_result = (lambda x, y: x+y)(1, 2) # sum_result is assigned 3.

The reason why it can be written like this is that according to the lambda document,

The expression lambda parameters: expression becomes a function object.

There is. That’s right, the lambda expression is a function object, so the (lambda x, y:x+y) part becomes a function object.

Therefore, it can be used as a function in the form of function object (actual argument)!! With this method, anonymous functions can be used once without using variables!! I did not know.
