This article is the same as Omron Environmental Sensor (BAG type) article This article was conducted with the low-priced Inkbird IBS-TH1. The battery used for the mini used last time was bad, so I aimed to improve the endurance with this product using AAA batteries.

① Confirm Bluetooth connection between Raspberrypi and sensor

Confirmation of sensor recognition

・Sensor setup Set the AAA battery in the sensor (be careful as the lid is difficult to remove)

・Scan for Bluetooth devices Run the following command on RaspberryPi

sudo hcitool lescan

LE Scan ...
AA:CC:EE:DD:55:77 sps

So if you see the name “sps”, this is the sensor’s MAC address. If it doesn’t come out, check the battery contact and the Raspberry Pi’s Bluetooth enable. Depending on the model and settings, the name may not be “sps”, so In that case, you can check the MAC address on Inkbird official app.

##** ② Acquire sensor measurement values with Python **

Confirm recognition with bluepy

bluepy is a library for accessing Bluetooth Low Energy (BLE) in Python (Class Definition)

・Installation of required packages Install the following

sudo install libglib2.0-dev

・Installing bluepy

Install with pip with the following command

pip install bluepy

・Authorize bluepy You need to give bluepy Sudo privileges for scanning.

Go to the folder where bluepy is installed,

cd ~.pyenv/versions/3.7.6/lib/python3.7/site-packages/bluepy
  • The above is when Python 3.7.6 is installed with pyenv. Please note that the location varies depending on the environment

Grant Sudo permission to bluepy-helper with the following command

sudo setcap'cap_net_raw,cap_net_admin+eip' bluepy-helper

Creating sensor value acquisition script

Create the following script to get the sensor value

from bluepy import btle
import struct

def get_ibsth1_data(macaddr):
    peripheral = btle.Peripheral(macaddr)
    characteristic = peripheral.readCharacteristic(0x28)
    (temp, humid, unknown1, unknown2, unknown3) = struct.unpack('<hhBBB', characteristic)
    sensorValue = {
            'Temperature': temp / 100,
            'Humidity': humid / 100,
            'unknown1': unknown1,
            'unknown2': unknown2,
            'unknown3': unknown3,
        }
    return sensorValue

The handle of Inkbird IBS-TH1 mini was “0x002d”, but be careful because it is “0x28” this time. Data is acquired by communication with the Peripheral class of bluepy.

The acquired characteristic data is ・1st and 2nd bytes: Temperature (0.01℃ unit) ・3rd to 4th bytes: Humidity (0.01% unit) However, I couldn’t understand the contents of the 5th and 7th bytes, so I will log it as unknown 1 to 3 and try to find out what it is. (We are aiming to be able to obtain the battery level)

Creating the main script

Create a main script to call the sensor value acquisition script

from bluepy import btle
import inkbird_ibsth1_connect

######Omron environmental sensor (BAG type) value acquisition ######
PERIPHERAL_MAC_ADDRESS ='MAC address obtained in ①'
sensorValue = inkbird_ibsth1_connect.get_ibsth1_data(PERIPHERAL_MAC_ADDRESS)

# Display temperature for trial
print(sensorValue['Temperature'])
