[Python] Things from building Python environment to running on Windows

Aug 11, 2017 Python beginner Python3 python2.7 Anaconda 

When building the Python environment, I went through various articles saying "Authentication proxy settings, coexistence environment, and execution confirmation are all necessary", so I think that it would be nice to put them all together.

#Introduction

Content

The contents introduced in this article are as follows. Please skip the authentication proxy and coexistence environment construction if necessary.

  • Install Anaconda (Windows)
  • Authentication proxy settings
  • Different versions coexistence environment construction
  • Run the program in interactive mode
  • Run Python script

For whom

For Python beginners who meet any of the following:

  • Need to use Python on company PC (Windows)
  • I want to coexist with different versions **, such as using both Python2 and 3
  • I want to do ** From environment construction to execution confirmation **

#Environment The environment for which operation has been confirmed is as follows.

  • Windows10(64bit) Version 1607
  • Anaconda 4.4.0 Python 3.6 Version
  • Python 3.6.1 and Python 2.7.13

Procedure

Install Anaconda

  • Download the installer corresponding to the target OS from Download site. (*) Even if you are the main Python2 person, you can use the 3rd person installer.

  • Kick the installed exe and proceed with the installation. __ Check “Add to environment variable PATH” Except for __, the default settings are OK.

Confirm setting information at installation

  • Launch Anaconda Prompt from the Start Menu.

  • Check the existing environment with the following command. “*” is displayed in the currently operating environment.
C:\Users\ username >conda info -e

# conda environments:
#
root * C:\Users\username\Anaconda3
  • Type python to check the version of Python in your current environment. From below, you can confirm that it is Python 3.6.1.
C:\Users\username >python
Python 3.6.1 | Anaconda 4.4.0 (64-bit)| (default, May 11 2017, 13:25:24) [MSC v.1900 64 bit (AMD64)] on win32
Type "help", "copyright", "credits" or "license" for more information.
>>>
  • Type the following to exit interactive mode once.
>>>exit()

Authentication proxy settings

(*) If you are in an authentication proxy environment, you cannot execute “build a coexistence environment” without first passing authentication.

  • Create a configuration file .condarc directly under the HOME directory of Anaconda3 (usually C:\Users\username\Anaconda3). Describe the “user name”, “password”, “IP address”, and “port number” used for proxy authentication in .condarc as follows. 1
proxy_servers:
    http: http:// User name: Password @ IP address: Port number
    https: https://username:password@IP address:port number

(*) If you cannot create it directly, create an appropriate text file and execute the following.

Move to the HOME directory of # Anaconda3 and execute
C:\Users\username\Anaconda3 >ren <text file name> .condarc
  • Execute the following command and check whether the settings are reflected.
C:\Users\ username >conda config --show

add_anaconda_token: True
...

proxy_servers:
  http: http:// User name: Password @ IP address: Port number
  https: https://username:password@IP address:port number

...
verbosity: 0

Construction of coexistence environment of different versions

  • This time, as an example, let’s make a Python2.7 environment coexist. First, execute the following command. If you add anaconda to the end, it will automatically inherit the included libraries when Anaconda is installed. 2
Create a Python execution environment of #2.7 series with the name "py27"
C:\Users\ username >conda create -n py27 python=2.7 anaconda
  • Confirm that the environment has been created with the name “py27”.
C:\Users\ username >conda info -e

# conda environments:
#
py27 C:\Users\username\Anaconda3\envs\py27
root * C:\Users\username\Anaconda3
  • Enter the environment for Python2 with the activate command. 3
C:\Users\ username >activate py27
(py27) C:\Users\username>
  • Type python to show the Python version and make sure the environment is switched.
(py27) C:\Users\username >python
Python 2.7.13 | Continuum Analytics, Inc.| (default, May 11 2017, 13:17:26) [MSC v.1500 64 bit (AMD64)] on win32
Type "help", "copyright", "credits" or "license" for more information.
Anaconda is brought to you by Continuum Analytics.
Please check out: http://continuum.io/thanks and https://anaconda.org
>>>

Run the program in interactive mode

  • Try to display “Hello, world!” in interactive mode while you are in the 2nd system environment. Enter print('Hello, world!').
>>> print('Hello, world!')
Hello, world!
>>>

It was confirmed that “Hello, world!” is displayed.

Run Python script

  • Type the following to exit interactive mode once.
>>>exit()
  • Prepare the following Python script and save it in an appropriate directory.
print('Hello, world!')
  • Execute the following and check that “Hello, world!” is displayed. Enter the path of the script created earlier in <script storage location>.
(py27) C:\Users\ username >python <script storage location>\helloWorld.py
Hello, world!

(py27) C:\Users\username>
  • Use the deactivate command to exit the 2nd environment.
(py27) C:\Users\ username >deactivate py27

C:\Users\username>
  • Type python and confirm that the environment is switched.
C:\Users\username >python
Python 3.6.1 | Anaconda 4.4.0 (64-bit)| (default, May 11 2017, 13:25:24) [MSC v.1900 64 bit (AMD64)] on win32
Type "help", "copyright", "credits" or "license" for more information.
>>>

This completes the operation confirmation of Python2 series. In the same way for the 3rd series following the above, you can check the operation by “Run the program in interactive mode” and “Run Python script”. 4

Supplement (About coexistence environment construction)

A few points will be supplemented regarding the following commands when building different versions of the coexistence environment.

C:\Users\ username >conda create -n py27 python=2.7 anaconda

version specification

  • Of course, the version specified by python= is OK for 3.X series as well.
  • If you do not specify the minor version with python=, the latest version of the target system will be installed.
  • If you specify a version that is not in Anaconda’s library, the following message is output.
C:\Users\ username >conda create -n py21 python=2.1
Fetching package metadata ...........

PackageNotFoundError: Package missing in current win-64 channels:
  -python 2.1*
  • The minor versions that can be specified as of August 11, 2017 are as follows.| Series | Specifiable version | |:–|:–| |2.5 or earlier |None| |2.6|2.6.8, 2.6.9| |2.7|2.7.3 ~ 2.7.13| |3.1|None| |3.2|None| | 3.3 or later | Basically all OK |

Delete the created environment

  • To delete the created environment, execute the following command.
C:\Users\ username >conda remove -n <environment name> --all
