# Build a development environment for DynamoDB and Django with docker-compose

Introduction

DynamoDB is cheap and fast. Did you know that there is a Docker Image for developing such DynamoDB locally? We will inquire about an example of development environment construction using such Docker Image.

Environment

The following image versions are used according to the production environment. Use dynamodb-admin for DynamoDB GUI operation. Node is used to build the environment together.

Environment Version
Python 3.7.4
MySQL 5.7
Node 10.16.3-alpine

Environment

Django and MySQL

Since we’re building from scratch, we’ll start with the Django project. The official Django image is out-of-date, so make it yourself.

$ django-admin startproject dynamodb_example
$ cd dynamodb_example/
$ touch docker-compose.yml
$ touch Dockerfile

FROM python:3.7.4

RUN apt-get update
RUN apt-get install -y --no-install-recommends apt-utils gettext
RUN mkdir /app; mkdir /app/dynamodb_example

WORKDIR /app
COPY dynamodb_example /app/dynamodb_example
COPY requirements.txt /app/
COPY manage.py /app/

RUN pip install -r requirements.txt

EXPOSE 8080
CMD ["python", "manage.py", "runserver", "0.0.0.0:8080"]

version: "3"
services:
  mysql:
    container_name: example-mysql
    ports:
      -53306:3306
    image: mysql:5.7
    command: mysqld --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    volumes:
      -./persist/mysql:/var/lib/mysql
    restart: always
    environment:
      MYSQL_USER: example
      MYSQL_PASSWORD: example
      MYSQL_DATABASE: example
      MYSQL_ROOT_PASSWORD: example

  django:
    container_name: example-django
    build:.
    volumes:
      -.:/app
    working_dir: /app
    command: sh -c "./wait-for-it.sh db:3306; python3 manage.py runserver 0.0.0.0:8000"
    env_file: .env
    ports:
      -58080:8000
    depends_on:
      -mysql

If you come to this point, check whether it will start up for the time being.

$ docker-compose up -d
$ docker-compose exec django bash
$ python manage.py migrate

If there is no problem, I will add DynamoDB Local and dynamodb-admin.

DynamoDB Local

Add dynamodb to docker-compose.yml. Specify dbPath at the end of the command to make it persistent.

  dynamodb:
    container_name: example-dynamodb
    image: amazon/dynamodb-local
    command: -jar DynamoDBLocal.jar -dbPath /home/dynamodblocal/data
    volumes:
      -./persist/dynamodb:/home/dynamodblocal/data
    ports:
      -50706:8000

dynamodb-admin

I had an official image, but I will use it because I made it.

$ mkdir dynamodb-admin
$ touch dynamodb-admin/Dockerfile
$ touch dynamodb-admin/.env

FROM node:10.16.3-alpine

RUN ["apk", "update"]
RUN ["npm", "install", "dynamodb-admin", "-g"]

EXPOSE 50727
CMD ["dynamodb-admin", "-p", "50727"]

In the environment variable DYNAMO_ENDPOINT, specify the container service name of dynamodb and the port on the container side.

DYNAMO_ENDPOINT=http://dynamodb:8000
AWS_REGION=ap-northeast-1
AWS_ACCESS_KEY_ID=ACCESS_ID
AWS_SECRET_ACCESS_KEY=ACCESS_KEY

The final docker-compose.yml file with this dyanamodb-admin added is as follows.

version: "3"
services:
  dynamodb:
    container_name: example-dynamodb
    image: amazon/dynamodb-local
    command: -jar DynamoDBLocal.jar -dbPath /home/dynamodblocal/data
    volumes:
      -./persist/dynamodb:/home/dynamodblocal/data
    ports:
      -50706:8000
  dynamodb-admin:
    container_name: example-dynamodb-admin
    build: dynamodb-admin/
    command: dynamodb-admin -p 8000
    env_file: dynamodb-admin/.env
    ports:
      -50727:8000
    depends_on:
      -dynamodb
  mysql:
    container_name: example-mysql
    ports:
      -53306:3306
    image: mysql:5.7
    command: mysqld --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    volumes:
      -./persist/mysql:/var/lib/mysql
    restart: always
    environment:
      MYSQL_USER: example
      MYSQL_PASSWORD: example
      MYSQL_DATABASE: example
      MYSQL_ROOT_PASSWORD: example
  django:
    container_name: example-django
    build:.
    volumes:
      -.:/app
    working_dir: /app
    command: sh -c "./wait-for-it.sh db:3306; python3 manage.py runserver 0.0.0.0:8000"
    env_file: .env
    ports:
      -58080:8000
    depends_on:
      -mysql
      -dynamodb

It is successful if http://localhost:50727 is displayed in the browser and the following is displayed. All you want to do is Create Table.

Persistence confirmation

I will make a table appropriately. Was created.

Delete the container and restart it.

$ docker-compose down
$ docker-compose up -d

If you open http://localhost:50727 and the previous table remains, the persistence is successful.
