② Processing of acquired data

The file with the package name and the number of downloads obtained earlier is It’s easy and convenient for people to read at hand, To handle it programmatically, you need to parse and process it.

Pay attention to the removal of the cost row of the query at the beginning and the heading row/Total row of the table. Make the LIST format by processing as follows.

f = open('/content/drive/My Drive/PYPI/PYPINFO_365_LIST.txt')
line = f.readline() # read line by line (including: newline character)

pypinfo_list = []
while line:
  #If there are three thresholds = Heading, frame and Total are jams, but other than that can be determined by this condition
  if line.count('|') != 3:
    line = f.readline()
    continue
  else:
    #Remove line feed code, comma, and half-width space
    parsed_line = line.replace('\n', ``).replace('','').replace(',', ``)
    one_data = parsed_line.split('|')
    Use the middle two in the form #['','urllib3', '950108414','']
    #Remark: Numerical values are treated as character strings for now
    one_data = one_data[1:3]
    pypinfo_list.append(one_data)
    line = f.readline()
f.close

#Remove the first two heading lines and the last total line
pypinfo_list = pypinfo_list[2:-1]

③Technology to make package name katakana (alkana.py & work hard)

Now, the package names can be listed, For example, urllib ⇒ Url Lrib python-dateutil ⇒ Python-Date Util To convert the package name to katakana, such as What do you do?

The policy is the following four steps.

  1. Katakana words that exist as English words
  2. Katakana IT special terms such as Python, AWS, and GIT
  3. Apply “Romaji conversion” as much as possible
  4. Convert the remaining fractional characters appropriately: f⇒F, etc.

The first, “English words in katakana” is I used the conversion table of alkana.py. https://github.com/cod-sushi/alkana.py/blob/master/README_ja.md

About 2-4, mainly from the Roman alphabet rule table, I created a conversion table of about 330 lines. Add the above data from alkana.py, Make a conversion table as alkana_list.

The point here is to use the length of the English string as a key, Sort alkana_list in descending order.

Put the length of the character string in the item of ```py:x[0] beforehand. alkana_list = sorted(alkana_list, key=lambda x: x[0], reverse=True) For high-priority items such as #py ⇒ pie and python ⇒ python,

[30,‘py’,‘pie’] and so on will increase the priority by registering as long.


The transformations will now be applied in order from longest words.
The actual conversion is as follows.
Due to the amount, it takes about 50 minutes each time.
You can display the progress in the middle with tqdm as follows,
It will be easier to save it with pickle after processing.

```py: Add Katanaka reading information to all modules
from tqdm import tqdm

pypinfo_jp_list = []
for pypinfo in tqdm(pypinfo_list):
  #Japanese module name storage variable (English is stored at this point)
  jp_module_name = pypinfo[0]
  for data in alkana_list:
    #Convert the conversion table in order.
    jp_module_name = jp_module_name.replace(data[1], data[2])
  pypinfo_jp_list.append([pypinfo[0], jp_module_name, int(pypinfo[1])])
