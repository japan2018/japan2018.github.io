#C problem

Since I want to count how many each character string is, I manage it in a dictionary. After enumerating all the strings in the dictionary, if you make that dictionary a list, you will have a list with tuples of “key-value pairs” of the dictionary as elements. Since we want to find the maximum number of strings, we can sort by the second element of the tuple. Also, when outputting multiple character strings, they are output in alphabetical order, so sort by the first element of the tuple first. Implementing the above, it becomes as follows.

d=dict()
n=int(input())
for i in range(n):
    s=input()
    if s in d:
        d[s]+=1
    else:
        d[s]=1
d=list(d.items())
d.sort(key=lambda x:x[0])
d.sort(key=lambda x:x[1],reverse=True)
m=len(d)
for i in range(m):
    if d[i][1]==d[0][1]:
        print(d[i][0])
    else:
        break

#D problem

First of all, k is 10^10 at maximum, so you can see that methods that start from the front will not be in time. Here, monotonicity and range of numbers are fixed, so you can use binary search.

When we look at the problem with the binary search in mind, we find the product, so if the product of positive numbers or negative numbers is positive, and if the product of positive and negative numbers is negative, then at least one You can see that if is 0, it becomes 0. Therefore, we considered different cases in each case.

First, if the kth number is negative, you can choose one from the set of positive numbers (b3) and one from the set of negative numbers (b1). Also, since the product of the maximum absolute numbers is the smallest and -1 is the maximum here, the kth smallest number in that interval is searched for by the binary search. At this time, the countk1 function is used to count the number below a certain number t. The countk1 function counts the number of negative numbers x or less given a set of positive numbers and a set of negative numbers. Here, we will look at the elements of the set of negative numbers in order, and then sequentially check how many positive numbers have a product with x that is less than or equal to x by a binary search ($O(n \ log n)$).

Next, if the kth number becomes 0, 0 is calculated as it is.

Finally, if the kth number is positive, you can choose two numbers from a set of positive numbers or a set of negative numbers, which is a little more complicated. First, the minimum number is 1, but the maximum number is the product of the larger of the two sets with the larger absolute value of each set, and the number of elements of either set is two. Note that there must be at least one. Under this condition, just like the case where the k-th number is negative, it is sufficient to search for the k-th number (when the number becomes negative and when it becomes 0) by a binary search. Use the countk2 function instead. As with the countk1 function, one can be fixed and a binary search can be performed, but the binary search must be performed for each of a negative number set and a positive set. Note that in order to select two numbers without duplication, the index to start searching with the binary search must be i+1 instead of 0.

If the above case is divided and the binary search code is carefully written, it becomes the following code. (It was difficult to implement…I feel like I became a full-fledged person when I could write this much code…)

ll countk1(ll x,vector<ll> b11,vector<ll> b13){
    ll ret=0;
    REP(i,l1){
        # Binary search part (whether x or less is satisfied)
        ll lx=0;ll rx=l3-1;
        while(lx+1<rx){
            ll t=(lx+rx)/2;

ll countk2(ll x,vector<ll> b21,vector<ll> b23){
    ll ret=0;
    REP(i,l1-1){
        # I+1 not binary at the beginning of binary search (no duplication allowed)
        ll lx=i+1;ll rx=l1-1;
        while(lx+1<rx){
            ll t=(lx+rx)/2;
            if(b21[t]*b21[i]<=x){lx=t;} else{rx=t;}
        }
        if(b21[rx]*b21[i]<=x){
            ret+=(rx-i);
        }else if(b21[lx]*b21[i]<=x){
            ret+=(lx-i);
        }
    }
    REP(i,l3-1){
        # I+1 not binary at the beginning of binary search (no duplication allowed)
        ll lx=i+1;ll rx=l3-1;
        while(lx+1<rx){
            ll t=(lx+rx)/2;
            if(b23[t]*b23[i]<=x){lx=t;} else{rx=t;}

            ret+=(rx-i);
        }else if(b23[lx]*b23[i]<=x){
            ret+=(lx-i);
        }
    }
    return ret;
}

n=[int(x) for x in list(input())]

dp=[[-1,-1] for _ in range(k)]
dp[0]=[min(1+(10-n[0]),n[0]),min(1+(10-n[0]-1),n[0]+1)]

    dp[i]=[min(dp[i-1][1]+(10-n[i]),dp[i-1][0]+n[i]),min(dp[i-1][1]+(10-n[i]-1),dp[i-1][0]+n[i]+1)]
