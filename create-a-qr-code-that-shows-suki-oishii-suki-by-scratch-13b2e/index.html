Decide the type of QR code

There are actually many types of QR codes, though they are generally referred to as QR codes. There are more than 40 versions, from compact and large grids to very large and detailed ones.

What is different is basically two points, capacity and strength of error correction. The capacity is from 5 bytes to about 3700 bytes, and error correction recovers from 7% to 30% of the total even if part of it cannot be read.

This time, I want to create a simple code that includes the text “Suki Oishizumi”, so select the small one. Specifically, it is a QR code of 25 squares on each side, called version 2-H. The capacity is 16 bytes (Table 7 on page 31 of the standard manual). In addition, we chose the highest error correction power of 30%. Enter 9 characters for kanji and hiragana. “Anastasia plow” is safe. Unfortunately “Eve Santa Claus Suki” does not enter.

About GF2

Before I start the problem, I think I need to explain a mathematical concept called GF(2). I can’t explain it easily because it’s something I just remembered, but in a nutshell, it means “a world with special calculation rules that only has two numbers 0 and 1”. In this world, the rules for addition and subtraction are as follows.

addition

+ 0 1
0 0 1
1 1 0

subtraction

- 0 1
0 0 1
1 1 0

Yes, addition and subtraction have exactly the same result. As you can see from this result, this is in agreement with XOR (exclusive OR) of bits. At this point, there are some things that are convenient for computers. By expanding this steadily, various applications will become possible.

By the way, GF is an abbreviation for Galois Field, which is called Galois field or finite field in Japanese.

About primitive polynomials and GF(28)

A primitive polynomial is a special polynomial used when expanding and applying GF(pm). It is also used when extending GF(2) (p=2,m=1).

GF(28) is an extension of GF(2), which is also a world with special calculation rules. GF(2) contains only two numbers 0 and 1, but GF(28) contains 28=256 numbers .. It seems that it is often used because it is very convenient for 8-bit arithmetic.

When extending GF(2) to GF(28), we use primitive polynomials, but there are several types of primitive polynomials. Since the calculation result changes depending on the usage, x8+x4+x3+x< when creating a QR code You are clearly instructed to use sup>2+1.

And it is a very important entity that constitutes the 256 numbers (also called the original in technical terms) contained in GF(28). Since α satisfies F(α)=0, the following equation holds.

α^8+α^4+α^3+α^2+1 = 0 \tag{1}

At this time, all the coefficients of each term belong to the world of GF(2). That is, it takes only two values, 0 and 1, and the result of both addition and subtraction is equal to XOR. Based on that, (1) is transformed.

α^8 = α^4+α^3+α^2+1 \tag{2}

You may think it’s a little bit, but it’s not wrong that there is no minus when you transfer. Since addition and subtraction are the same, it is safe to say that there is no distinction of +/- in this world (α8=1α8=(0-1)α 8=-α8).

Choosing the right mask (hard)

The QR code has a function called “mask” to prevent patterns that are very difficult to read depending on the content of the data. It is intended to partially invert the placed black and white pattern using eight different patterns to finally generate the pattern that seems to be the most readable.

So, in the software (encoder) that creates the QR code, you have to try all eight masks and select the best one.

Mask type

There are the following 8 types of masks. (Calculation formula is omitted. For details, refer to P.49 of the standard document.)

The part included in the mask target

  • Data part
  • Error correction code

Parts not included in the mask target (other than those listed above)

  • Function pattern (position detection, timing, alignment)
  • Format information
  • Residual bit

Evaluation

After applying each mask, score evaluation is performed according to 4 kinds of evaluation criteria. The evaluation is a deduction system, and the one with the fewest goals is actually used.

The evaluation target is the entire QR code. (However, there is a part where the color is not decided yet, so it is unclear what to do with it…)

The error correction level is 10 because’H’ was selected this time. For other levels, refer to the specifications (Table 12 on page 54).

The BCH code is calculated based on this 5-bit information 10111. This is also a kind of error correction code similar to RS code. You may think that it’s mathematics again, but it’s okay because the flow of processing is similar and this one is fairly easy.

BCH code

In this case as well, the functions I(x) and G(x) are defined and solve the following problem.

N = 15, K=5 \\
I(x) = 1x^4 + 0x^3 + 1x^2 + 1x + 1 \\
G(x) = x^{10} + x^8 + x^5 + x^4 + x^2 + x + 1 \\
At \\
P(x) = x^{N-K}I(x) \quad mod \quad G(X) \\
Ask for

Each coefficient of I(x) is an array of bits of error correction level and mask number. Thankfully this time, all the coefficients are 0 or 1, so it’s very easy to calculate.

If you write only the result, you will get 0000101001. It is completed by connecting this to the end of the existing 5 bits and taking the XOR with the specified mask 101010000010010.

  & 101110000101001 \\
\oplus & 101010000010010 \\

& 000100000111011
\end{align}
\end{array}
