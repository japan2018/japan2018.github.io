Use jupyter-lab installed in python virtual environment (venv)

Nov 24, 2019 Python venv JupyterLab 

There was already a post on Qiita, but the information was scattered, so I arranged it.

There are two points.

  • Use kernel installed in virtual environment
  • Add a virtual environment to the execution environment (python) path

It’s obvious if you understand it, but often you install it in the original environment without checking it.

environment

  • MacOS 10.15.1
  • python 3.7.4 (pyenv/anyenv)

Create virtual environment

The module that creates the virtual environment is assumed to be installed.

$ virtualenv test
$ source test/bin/activate
$ pip install keras
$ pip install jupyterlab

Install kernel in virtual environment

$ pip install ipykernel
$ ipython kernel install --user --name=test

run

Start jupyter-lab.

$ test/bin/jupyter-lab

The installed kernel is displayed in the list, so select the kernel (test) created this time. The kernel can also be selected from the menu (Kernel/select kernel).

You can see that the kernel on the boot screen (right side) is test.

An error (ModuleNotFoundError) occurs when importing the module installed in the virtual environment.

Display the Python path.

Add the path of the virtual environment and run it. Import is successful.
