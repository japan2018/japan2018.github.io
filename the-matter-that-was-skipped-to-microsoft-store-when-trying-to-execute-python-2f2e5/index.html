The matter that was skipped to Microsoft Store when trying to execute Python

Nov 11, 2019 Python Windows Python3 

## Target audience
  • People who started to run Python with PowerShell and were skipped to the Microsoft Store
  • People who ran a Python script with PowerShell and nothing happened
  • Those who have already set the path and were able to run Python originally

##environment

  • Windows10 Home
  • Anaconda3

Conclusion

~~ Although the cause is unknown ~~ It seems that the following path is referenced. Let’s change the order of environment variables. Windows 10 May 2019 Update seems to be the cause (Thanks to @Miyukiumoo) https://forest.watch.impress.co.jp/docs/serial/yajiuma/1188376.html

C:\Users\user\AppData\Local\Microsoft\WindowsApps\python.exe

Background

Python cannot be executed

When I tried to execute a Python script with PowerShell as usual, nothing happened and the process returned. Not even at the command prompt. I tried running the interpreter for a trial, and it was skipped to the Microsoft Store.

** Why? ? **

By the way, at this time, I was able to execute scripts normally in the virtual environment of Anaconda.

Annoying the day before, I tried updating conda and it took a long time, it was displayed as a conflict, it was displayed as a conflict, Ctrl-c interrupted and restarted, and the default environment was broken ~ Anaconda I tried putting it in again, but no.

Check the executable file and environment variables

Anaconda It was something I had to confirm before re-inserting it. .. ..

Check environment variables. The path is up to python.exe.

** So why? ? **

Check the path referenced by the command

I finally noticed here. It’s not an error in the first place, so I’m going to the Microsoft Store. If you check the path that the command refers to,

> where python
C:\Users\user\AppData\Local\Microsoft\WindowsApps\python.exe
C:\Users\user\Anaconda3\python.exe

** Something **

When I checked it with Explorer, it seemed to be a relatively recent file. I tried to delete it with anger, but it was played.

I thought that if I execute it with administrator privileges, it will be erased.

Change the order of environment variables

When you run the command, it will scan the path of environment variables and execute the first file it finds. The python.exe of Anaconda3 is found before the python.exe which is skipped to Microsoft Store.

Now you can run Python safely.

Summary

If the command cannot be executed, first check the path that the command refers to.
