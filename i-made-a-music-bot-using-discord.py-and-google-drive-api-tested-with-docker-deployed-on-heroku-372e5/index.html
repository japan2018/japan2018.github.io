I made a music bot using discord.py and Google Drive API (tested with Docker → deployed on Heroku)

Jan 28, 2020 Python Heroku Docker GoogleDrive discord 

The used file is placed in [here](https://github.com/ysk0832/hobby/tree/master/discordbot-citron)(excludingauthenticationrelated)

Basically, I made it for my family to enjoy.

Thing you want to do

  1. Send a message on discord
  2. bot finds song file from google drive
  3. If you find it, download it and play it with your voice.

Also, there are two personal requirements for music bots,

  • You can request songs other than yourself
  • I want to keep the bot running

I decided to use google drive and heroku to satisfy these.

#Environment

  • macOS Mojave 10.14.6
  • Docker 19.03.5
  • discord.py 1.3.0
  • Google Drive API v3

Directory structure

.
├── Procfile
├── citron.py
├── client_secrets.json
├── commands.py
├── requirements.txt
├── runtime.txt
└── token.pickle

Description of each file

  • citron.py: bot itself
  • commands.py: List of commands that the bot responds to
  • client_secrets.json, token.pickle: google drive authentication related
  • Procfile, runtime.txt, requirements.txt: Required for Heroku deployment

Build test environment (Docker container)

docker pull python:3.7
docker images
# Container creation
docker run -d -it --name python --mount type=bind,source=path/to/mydir,target=/home/ python:3.7

# Enter container
docker exec -it python bash

#The following operations in the container
python3 -m pip install -U discord.py
pip install --upgrade google-api-python-client google-auth-httplib2 google-auth-oauthlib
apt update
apt -y upgrade
apt install ffmpeg
