[Python] I will create a Python directory structure that I will not regret later

Apr 13, 2020 Python PyPI Python3 

I've learned a little about directory structure a little after I started programming in Python, so I'll summarize it.

At first, I didn’t understand it at all, and I was doing a lot of googling, so I wrote the steps so that even beginners can understand.

The following is a memo by an engineer who has been developing in Python for about 6 months. ** We would appreciate if you could give us a lot of suggestions, suggestions, and questions from beginners. **

Introduction

When I first used Python, I wrote a lot of methods in hoge.py for the time being, and then there were multiple files and directories increased. However, doing this made it difficult to resolve the dependencies in my own program, and the composition became more and more incomprehensible, making it difficult to correct it later.

Therefore, unless the programming is urgent, I think that “preparing directories and files so that it can be easily made into a library” is important first. I will make the template in this article.

First try writing from setup.py

First, create a python_package directory as a project folder and create a new setup.py. Directly below the project will be as follows.

.
└── setup.py

**setup.py is absolutely necessary to provide your own program as a library. **

For now, let’s fill in what can be put in under the present conditions. You decide the name of the project you are trying to create and the location of the source.

from setuptools import setup

setup(
    name="koboripackage",
    version='1.0',
    description='For testing Python directory structure',
    author='Kobori Akira',
    author_email='private.beats@gmail.com',
    url='https://github.com/koboriakira/python_package',
)

If the product to be created is clear, you can create README.md at this stage.

Make a library for the time being

After writing setup.py, the next step is to prepare the source directory.

Create a directory with the same name as that written in name of setup, and insert __init__.py for the time being. You can use an empty file. Also create a Python file with the same filename as the directory name. Let’s say this is a module used as a library.

│ └── koboripackage.py
└── setup.py

Make it usable in CLI

Libraries made in Python can also be executed from the command line from the terminal.

Let’s add a setting for that. First, create cli.py directly under the source directory.

def execute():
    print('The command has been executed!')

Then edit setup.py.

  1. Set version to 1.1
  2. Add find_packages to import and add packages=find_packages()
  3. Add the entry_points. The format is written as command name = module:method.
from setuptools import setup, find_packages

setup(
    name="koboripackage",
    version='1.1',
    description='For testing Python directory structure',
    author='Kobori Akira',
    author_email='private.beats@gmail.com',
    url='https://github.com/koboriakira/python_package',
    packages=find_packages(),
    entry_points="""
      [console_scripts]
      koboripackage = koboripackage.cli:execute
    """,
)

When you are done, run python setup.py sdist again. Then a new v1.1 tar is created in dist, and it has the following structure.

.
├── dist
│ ├── koboripackage-1.0.tar.gz
│ └── koboripackage-1.1.tar.gz
├── find_package.txt
├── koboripackage
│ ├── __init__.py
│ ├── cli.py
│ └── koboripackage.py
├── koboripackage.egg-info
└── setup.py

As before, try installing the library with pip install dist/koboripackage-1.1.tar.gz. Then,

$ koboripackage
The command has been executed!

You can see that the execute method of cli.py can be executed only by the command name as above.

(Supplement) About __init__.py and find_packages()

It may be wrong, but write down your understanding.

find_packages() is a method to search for necessary sources when creating a tar with python setup.py sdist. And __init__.py is needed for searching.

For this chapter only, I’ll try adding the sub directory to the source directory, and sub_module.py.

.
├── koboripackage
│ ├── __init__.py
│ ├── cli.py
│ ├── koboripackage.py
│ └── sub
│ ├── __init__.py
│ └── sub_module.py
├── setup.py

If you don’t have __init__.py in the sub directory, find_packages() won’t find the files in or in the sub directory.

Use existing library

Try importing the existing library to cli.py.

import requests


def execute():
    print('The command has been executed!')
    response = requests.get('https://www.google.com/')
    print(response.status_code)

To use requests (locally) you need to import the library with pip install requests. Similarly, in order to provide it as a library, it is necessary to specify “what must be imported”.

The requirements.txt is prepared for this. Place it directly under the project.

requests==2.23.0
```By the way, you can check the required libraries and versions with the `pip freeze` command.
However, since all the libraries currently installed are output, if you copy and paste as it is, you may include unnecessary libraries.

Next, make two settings so that requirements.txt is used correctly when creating a tar with `python setup.py sdist`.

1. Create a new MANIFEST.in
2. Add `install_requires` to setup.py (and version to 1.2)

. ├── MANIFEST.in ├── dist ├── koboripackage │ ├── init.py │ ├── cli.py │ └── koboripackage.py ├── koboripackage.egg-info ├── requirements.txt └── setup.py


```MANIFEST.in
include requirements.txt

from setuptools import setup, find_packages

setup(
    name="koboripackage",
    version='1.2',
    description='For testing Python directory structure',
    author='Kobori Akira',
    author_email='private.beats@gmail.com',
    url='https://github.com/koboriakira/python_package',
    packages=find_packages(),
    entry_points="""
      [console_scripts]
      koboripackage = koboripackage.cli:execute
    """,
    install_requires=open('requirements.txt').read().splitlines(),
)

MANIFEST.in allows you to “add what is not normally included when packaged (and vice versa)". Since I wrote include requirements.txt, I will add requirements.txt when packaging.

Then, in install_requires added to setup, specify the required libraries. You can directly specify the library name as an array, but it seems to be common to load the contents of requirements.txt as above.

*When specifying directly

install_requires=['requests']

Prepare Docker and git

Finally, I will prepare a Docker image because it is a big deal. After adding the necessary files, create an image with the library installed in advance.

FROM python:3.7.5-slim

WORKDIR /work

ADD koboripackage koboripackage
ADD tests tests
ADD requirements.txt requirements.txt
ADD setup.py setup.py
ADD MANIFEST.in MANIFEST.in

RUN pip install --upgrade pip \
  && pip install -r requirements.txt \
  && pip install pytest

CMD bash

Create an image with docker build -t python_package . and a container with docker run -it --rm python_package. You can now run pytest inside a Docker container. If you want to develop it as it is, you can share the source as follows.

docker run -it --rm -v $(pwd)/koboripackage:/work/koboripackage -v $(pwd)/tests:/work/tests python_package
