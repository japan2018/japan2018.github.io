What happens to the access log when accessing the website automatically with selenium-webdriver

Nov 13, 2019 Python selenium-webdriver headless-chrome 

#Purpose

Actually, I was secretly automating part of the internal work, but I suddenly looked into the server log and thought that it would be obvious that it was automated, so I verified it.

Verification environment

  • Use PaizaCloud as a web server
  • Use Selenium-webdriver and headless-chrome as an automatic access method
  • Use Python as a program to automate

What you did

Introducing Selenium and Headless Chrome

There are many articles about this, so I will omit it.

Set up a web server

Access PaizaCloud and log in. You can log in with your Google account, and some plans can be used free of charge. This time I am using this free plan.

On the server creation screen, select an appropriate server name and application to install. This time I have Apache installed.

After creating the server, click the browser button on the left to launch the internal browser.

Of course, you can copy and paste the URL and use another browser. For now, I’ve confirmed that the server is up.

Check the location of the server log

The Apache server log is located at /var/log/apache2/access.log by default. Launch the console by clicking the console button on the left.

Hit the following command and check the access log.

sudo tail /var/log/apache2/access.log

This time, there was no access log yet.

Writing code in Python

#Import module
from selenium import webdriver

#Start Chrome
options = webdriver.ChromeOptions()
driver = webdriver.Chrome(options=options)

# Throw a request to the web server
driver.get('https://bowtin-testserver.paiza-user.cloud/')

When you run the above program, Google Chrome should launch automatically and access the web page.

Check the access log

Check the access log on the server side again.

It is written as Chrome, and at this point I do not know if it is manual or automatic.

Try changing the code so that the Chrome screen does not start

Add some code to make Chrome start in the background. With this, Chrome does not appear on the screen, so it feels like it is automated.

#Import module
from selenium import webdriver

#Start Chrome
options = webdriver.ChromeOptions()
options.add_argument('--headless') # add
options.add_argument('--disable-gpu') # add
driver = webdriver.Chrome(options=options)

# Throw a request to the web server
driver.get('https://bowtin-testserver.paiza-user.cloud/')

After modifying the code, try again.

Check the server log again

This time, I was able to confirm that the browser type is Headless Chrome.
