Using anaconda’s virtual environment with Zsh (defect support)

Jan 1, 2020 Python Zsh Anaconda 

# Overview

Using Jupyter Notebook of anaconda installed with pyenv with zsh (defect support), following trial and error as a memorandum for using Jupyter Notebook with Zsh Keep.

Defect status

  • $ conda info -e can confirm the virtual environment, but cannot activate it with $ conda activate <virtual environment name>

#Environment Mac OS Mojave

Method

1. Try running the virtual environment

$ conda activate <virtual environment name>

Error message returned

CommandNotFoundError: Your shell has not been properly configured to use'conda activate'.
To initialize your shell, run

    $ conda init <SHELL_NAME>

Currently supported shells are:
  -bash
  -fish
  -tcsh
  -xonsh
  -zsh
  -powershell

See'conda init --help' for more information and options.

IMPORTANT: You may need to close and restart your shell after running'conda init'.

2. Initialize

$ conda init <SHELL_NAME> *This time "zsh"

3. Restart the shell

$ exec $SHELL -l

Aside. Response after restarting the shell

After restarting the shell, the prompt was (base) <username>%. It seems that it has entered the Python base environment, so exit the virtual environment with $ conda deactivate.
