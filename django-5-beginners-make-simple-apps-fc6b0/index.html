1. Implementation of Update part

rough image

When the log is displayed on the movie detail screen page and you want to edit the log for a moment, you can edit it by pressing the button.

When creating a new page, create it in the order of urls.py → views.py → template. In this way, I think that leaking thoughts will be lessened by making a way to make it as a mold. First, describe the address update/log/<int:pk>/ when updating the log (impression) in urls.py.

urls.py (update part)

from django.urls import path, include
from myapp import views

app_name ='myapp'
urlpatterns = [
    path('', views.IndexView.as_view(), name='index'),
    path('movie/<int:pk>/', views.MovieDetailView.as_view(), name='movie_detail'),
    # (Omitted)
    path('update/log/<int:pk>/', views.UpdateLogView.as_view(), name='updatelog'), # add this line
]

views.py (update part)

Next, write the code of UpdateLogView which is Class-based-view in views.py.


class UpdateLogView(generic.UpdateView):
    model = Log
    form_class = LogForm
    template_name = "myapp/register.html"
    def get_success_url(self):
        return reverse('myapp:movie_detail', kwargs={'pk': self.object.movie.pk })

# If you write in Function-view, it is as follows

def updatelog(request, pk):
    obj = get_object_or_404(Log, id=pk)
    if request.method == "POST":
        form = LogForm(request.POST, instance=obj)
        if form.is_valid():
            form.save()
            return redirect('myapp:movie_detail', pk=obj.movie.pk)
    else:
        form = LogForm(instance=obj)
        return render(request,'myapp/register.html', {'form': form})

There are four items that should be specified in Class UpdateLogView (same as the previous CreateView).

  1. What to use for model
  2. What form do you use
  3. What to use for the displayed template
  4. Where to fly if successful

Then, you can understand how it actually works by understanding with def updatelog of Function-view. The request part of (request, pk) is an HttpRequest object that can be created by pressing the edit button of detail.html. The id number of the log is entered in the pk part. Then, using this pk, a number is entered in the <int:pk> part of update/log/<int:pk>/ of urls.py. Note below.

2. Implementation of Delete part

rough image

I want two functions: a function to erase the log (impression) and a function to erase the movie data. Then you’ll have to write two chunks of code in urls.py and two in views.py.

urls.py (delete part)


from django.urls import path, include
from myapp import views

app_name ='myapp'
urlpatterns = [
    path('', views.index, name='index'),
    # (Omitted)
    path('delete/log/<int:pk>/', views.deletelog, name='deletelog'), # add this line
    path('delete/movie/<int:pk>/', views.deletemovie, name='deletemovie'), # add this line
]

The address described here is the confirmation screen for deletion. The place that says “Can I erase this?” Next, write the code in views.py.

views.py (the part that erases impressions)


class DeleteLogView(generic.DeleteView):
    model = Log
    def get_success_url(self):
        return reverse('myapp:movie_detail', kwargs={'pk': self.object.movie.pk })

# From here Function-view

def deletelog(request, pk):
    obj = get_object_or_404(Log, id=pk)
    movie_id = obj.movie.pk
    if request.method =="POST":
        obj.delete()
        return redirect('myapp:movie_detail', pk=movie_id)
    context = {'obj':obj}
    return render(request, "myapp/delete.html", context)

With Class-based-view, all you have to do is specify the model and the page to be skipped on success. However, with only these two items, it is necessary to name the delete confirmation page “XX_confirm_delete.html”. The name of model is entered in ○○. This time it becomes “log_confirm_delete.html”. In Function-view, “delete.html” was created and used as the deletion confirmation page.

Class-based-view does it automatically, so what I had to devise in Function-view was to display the detailed screen after deleting the log. If you don’t save the id of the movie in the log before clearing the log, you won’t be able to return to the movie details screen. Note below.

views.py (the part that erases movie data)

    model = Movie

def deletemovie(request, pk):

The code is almost a repeat of deletelog. The only difference is that it will jump to index after deletion.

def writingthismovielog(request, movie_id):
    obj = get_object_or_404(Movie, id=movie_id)
    form = LogForm({'movie':obj})
    if request.method == "POST":
        form = LogForm(request.POST)
        if form.is_valid():
            l = form.save(commit=False)

        return render(request,'myapp/register.html', {'form': form})
