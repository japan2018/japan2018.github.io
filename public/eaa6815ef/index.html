Why isn’t Language X the last thing you learn in your life?

Is rust the last thing to learn in life?

I will probably end my adult life by continuing to write scala. When learning a JVM language like Scala, sometimes I’m envious of languages that can be natively compiled. Especially modern rust looks good. …But I feel something different when an old man writes rust, which is a lower language, in a rigorous work. (Of course, it would be cool if there was such an old man) So in my case, I rejected it because of my age. Oh, I think young people can earn money by doing rust properly ~

Is elm the last thing to learn in life?

It’s the opposite of rust, and I’m quite attracted to the front-end language elm in the standing position. (For those who like elm, read herefirst.) Since it is specialized for the web front end, the description is simple, but on the other hand, it is possible to write naturally with type safety. Elm is great because I feel like I can put a code with security even in the year when dementia approaches. Unfortunately, in the case of me, who is vulnerable to UX, I could not see the future of earning tens of millions of yen with elm after retirement. If you are younger than the front end, you will definitely be happy if you do elm properly.

#Why are you thinking of learning groovy3.x at the end of your life?

Well, it is groovy3. I think few people are paying attention. After all, groovy’s history as Alt-Java is longer than other JVM languages, so groovy itself is not much talked about anymore.

It is worth noting that what is crucially different from other JVM languages such as Scala/Kotlin is the scripting language of ruby-bar, but its notational compatibility with Java. In this area, you should read the introduction article of NTT Technocross, who is using groovy in practice. https://www.ntt-tx.co.jp/column/tec/java_01/ (…In other words, in the case of developing firmly with a Spring-based framework, It has the implication that it is easy to make proposals such as using groovy together to increase development productivity. )

Although it is such an old groovy, Groovy 3.0 that corresponds to the evolution of Java itself since Java 8 is about to be released. …Because of the large language specifications of Java itself, but it has been around for quite some time. http://groovy-lang.org/releasenotes/groovy-3.0.html
