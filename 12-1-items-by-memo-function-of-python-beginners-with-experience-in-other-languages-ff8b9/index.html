Import and namespace

Suppose you have a file called foo/bar.py that looks like this: (The description of the beginning part is omitted)

import inspect

def baz():
print('module=', __name__,', func=', inspect.currentframe().f_code.co_name)

def qux():
print('module=', __name__,', func=', inspect.currentframe().f_code.co_name)

(inspect.currentframe().f_code.co_name is the code that gets the function name from inside the function.)

Note: It seems that you need to put a file called __init__.py in the foo/ directory to use this foo as a package, as it can be empty. However, this example works even without __init__.py.

And if you want to call a function in it, it must have the full name. Only the module is imported, not the namespace.

>>> import foo.bar
>>> bar.baz()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name'bar' is not defined
>>> foo.bar.baz()
module= foo.bar ,func= baz
>>> foo.bar.qux()
module= foo.bar ,func= qux
>>>

To import bar as a namespace from within foo, from foo import bar will do. Also, it is difficult to use a long name, so you can give an alias with as. Of course, you can also combine the two.

>>> from foo import bar
>>> import foo.bar as fb
>>> from foo import bar as b
>>> bar.baz()
module= foo.bar ,func=baz
>>> bar.qux()
module= foo.bar ,func=qux
>>> fb.baz()
module= foo.bar ,func=baz
>>> fb.qux()
module= foo.bar ,func=qux
>>> b.baz()
module= foo.bar ,func=baz
>>> b.qux()
module= foo.bar ,func=qux
>>>

If you use from foo.bar import baz, you can directly import the baz function, but it is not recommended to use it directly because the namespace is confused except for some standard functions. Would be good (By the way, the qux function is not imported in this case.)

I want to rewrite a global variable from another scope

Variables defined in the module that are not closed in a block are global variables.

(I’m a little uncertain about the exact definition of this area. Maybe the default namespace is the main module.)

When a new variable with the same name as a global variable is defined in the function, it becomes a local variable. This means that if you try to rewrite (reassign) a global variable in a function, it will be treated as a local variable.

What if you want to rewrite a global variable in a function?

>>> gv ='global'
>>> def foo():
... gv ='local'
... print(gv)
...
>>> gv
'global'
>>> foo()
local
>>> gv
'global'
>>>

You can declare a global variable to be used in that scope using the keyword global.

>>> gv ='global'
>>> def foo():
... global gv
... gv ='reassign'
... print(gv)
...
>>> gv
'global'
>>> foo()
reassign
>>> gv
'reassign'
>>>

bonus: The globals() and locals() functions return a dictionary of global and local variables, respectively. There are a lot of global variables, so please check by actually executing them.

Property

You can hook access to class attributes (member variables), like C# properties, Java Bean accessors, etc.

When the Foo class has an attribute called text, you can declare a property using the property function.

>>> class Foo:
... def __init__(self, text):
... self.__text = text
...def get_text(self):
... print('get_text')
... return self.__text
...def set_text(self, text):
... print('set_text')
... self.__text = text
...text = property(get_text, set_text)
...
>>> foo = Foo('foo')
>>> foo.text
get_text
'foo'
>>> foo.text ='FOO'
set_text
>>> foo.text
get_text
'FOO'
>>>

By the way, if you put two underscores (__) at the beginning of the name, you cannot access the name as it is. It cannot be completely hidden like a private member of a controllable language, but it can convey its intent to not be exposed.

>>> foo.__text
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError:'Foo' object has no attribute'__text'
>>> foo._Foo__text
'FOO'
>>>

Properties can also be set in the decorator. This is a new style.

Just a little about decorators. The decorator looks like C# attributes or Java annotations, but I think it’s more like a macro or AOP. Please google for details.

>>> class Bar:

Relationship between list and tuple

Python is a dynamically typed language, so in principle both can be used the same way as sequences, except that lists are mutable and tuples are immutable.

By the way, Python tuples can be written without parentheses, except for empty tuples, but it is better to add parentheses to indicate that they are tuples.

>>> t1 = (1, 2, 3)
>>> t2 = 1, 2, 3
>>> t1
(one two Three)
>>> t2
(one two Three)
>>> type(t1)
<class'tuple'>
>>> type(t2)
<class'tuple'>
>>>

Lists and tuples can be converted to each other using the list and tuple functions.

>>> a = [1, 2, 3]
>>> type(a)
<class'list'>
>>> aa = tuple(a)
>>> aa
(one two Three)
>>> type(aa)
<class'tuple'>
>>> b = (4, 5, 6)
>>> type(b)
<class'tuple'>
>>> bb = list(b)
>>> bb
[4, 5, 6]
>>> type(bb)
<class'list'>
>>>

String format format

The so-called printf-like format seems to be old style. However, it is not deprecated to be old, it seems that old styles are just frozen features, and new styles are more flexible.

The old style is written like format string % tuple of values.

>>>'x=%d, y=%d' %(10, 20)
'x=10, y=20'
>>>

The new style uses the string format method. This style can be used in various ways, such as changing the order of passing and passing a dictionary as an argument.

>>> dict = {'x': 10,'y': 20,'z': 30}
>>>'z={z:d}, y={y:d}, x={x:d}'.format(**dict)
'z=30, y=20, x=10'
>>>

There is also a template string, but I was not sure how to use it.

>>> from string import Template
>>> s = Template('--- $foo --- $bar ---')
>>> s.substitute(foo='Foo', bar='Bar')
'--- Foo --- Bar ---'
>>> d = {'foo': 100,'bar': 200}
>>> s.substitute(d)
'--- 100 --- 200 ---'
>>>

define a string representation of the object

How can I define methods like to_s in Ruby and toString in Java or JavaScript?

Python classes have special methods, __str__ and __repr__, which you can implement to define a string representation of an object.

__str__ defines the character string representation when converting into a character string with the str function and outputting with the print function.

__repr__ is Python specific and defines the string representation on the interpreter. (abbreviation for representation?)

>>> class Foo:
... def __init__(self, text):
... self.text = text
...def __str__(self):
... return self.text
...def __repr__(self):
... return'%s(\'%s\')' %(self.__class__.__name__, self.text)
...
>>> Foo('foo')
Foo('foo')
>>> print(Foo('foo'))
foo
>>> str(Foo('foo'))
'foo'
>>>

First, the result of the __repr__ method is output as an expression on the interpreter. Second, the string representation of Foo(), that is, the result of the __str__ method is printed by the print function. Thirdly, the result of the __str__ method is also converted to a string by the str function and output as a representation on the interpreter.

If you use the repr function, you can get the result of the __repr__ method as a string.

I want to use a dictionary that maintains the registered order

If you need a dictionary that can be iterated in the registered order, like LinkedHashMap in Java, Use OrderedDict from the collections module instead of the built-in dictionary.

>>> from collections import OrderedDict
>>> d = dict([('y', 0), ('e', 0)])
>>> d
{'e': 0,'y': 0}
>>> d['s'] = 0
>>> d
{'s': 0,'e': 0,'y': 0}
>>> for x in d:
... print(x)
...
s
e
y
>>> od = OrderedDict([('y', 0), ('e', 0)])
>>> od
OrderedDict([('y', 0), ('e', 0)])
>>> od['s'] = 0
>>> od
OrderedDict([('y', 0), ('e', 0), ('s', 0)])
>>> for x in od:
... print(x)
...
y
e
s
>>>
