[Python] A note on creating a python development environment on macOS Catalina

Mar 22, 2020 Python Mac pyenv Python3 python2.7 

*Memo will be updated at any time*

Aiming to build a python development environment

In the environment I upgraded from Mojave to Catalina, when I did it normally, it did not work as expected with python3 or pip3 due to the difference in version, so I try using pyenv and see if it works.

It’s actually better to use Docker, but for now…

homebrew

Start from here.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

pyenv install

$ brew install pyenv
$ pyenv -v
pyenv 1.2.17

Profile settings

$ echo'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bash_profile
$ echo'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bash_profile
$ echo -e'if command -v pyenv 1>/dev/null 2>&1; then\n eval "$(pyenv init -)"\nfi' >> ~/.bash_profile

https://github.com/pyenv/pyenv

By the way, in the case of zsh, it seems that the same setting should be done for .zshrc.

Zsh note: Modify your ~/.zshrc file instead of ~/.bash_profile. https://github.com/pyenv/pyenv

I don’t know why it is .zshrc instead of .zprofile.

Reflection

$ source ~/.bash_profile

Install python by specifying version

Check installable version

$ pyenv install -l

Install the version you want to use

$ pyenv install 3.7.7

Confirm installation

$ pyenv versions
* system (set by /Users/ykinomt/.pyenv/version)
  3.7.7

Switch version

Status quo confirmation

$python -V
Python 2.7.16

switching

$ pyenv global 3.7.7
$python -V
Python 3.7.7

Use local to switch to that directory.

$python -V
Python 2.7.16
$ pyenv local 3.7.7
$python -V
Python 3.7.7

If you use local, a file called .python-version will be created in that directory, and that directory will work with the specified version. It may be better to specify it as local when creating an application.

vscode settings

Although not particularly necessary, the version that appears in the interpreter settings seems to have been installed by the time VSCode starts, so if you newly install with the above settings, you may not be able to select it without restarting vscode once.
