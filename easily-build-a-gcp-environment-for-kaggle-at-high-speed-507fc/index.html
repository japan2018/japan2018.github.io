Easily build a GCP environment for Kaggle at high speed

Dec 18, 2019 Python gcp Jupyter Kaggle 

#GCP had a great feature added

Previously, I wrote an article Easily build a GCP environment for Kaggle. This time, I found an easier way to build it, so I’ll write it as an improved version of the previous article. A notebook function has been added to the lower AI platform. What this means is that you can get it ready for use without having to do the work to make the jupyter notebook introduced in the previous article. To be precise, Jupyter Lab comes up, but if you’ve used a jupyter notebook, you’ll be familiar with it immediately. #How to use You can create an instance by pressing the button called “New Instance” at the top. This instance seems to be shared with GCE. In this environment part, you can set up the execution environment as easily as the Deep Learning VM by selecting the framework you want to use.

Once you have configured your instance, click Open JUPYTER LAB. If you select Notebook here, the jupyter notebook will start up. It’s really easy :smiley:

After that, you can input commands from Terminal, and you can create a Python file from Text File and rename it as .py, and you can create it without problems.

As for files, the GUI is well maintained, so you can easily upload and download files. #in conclusion Since this function was implemented, it became much easier to create a computing environment with GCP. Recently, Kaggle’s Kernel has severe restrictions on use, so I think that many people are using Google Colab, but I can get a coupon for 30,000 yen, so I think it is good to try GCP at this opportunity. I will. If you have any questions, please feel free to comment. #P.S Currently, I am using this function in a competition, but it seems that there is a problem with handling large files. For example, if you try to download a heavy file from Jupyter lab, it will obviously download a small amount, or if you try to pip install PyTorch, it will stop halfway. The measure is to write it to storage using gsutil, or set up an instance containing PyTorch from the beginning… I look forward to future improvements!
