Get a Python development environment with Poetry

May 11, 2020 Python setuptools Pipenv Poetry 

## background

Poetry reached 1.0.0 on December 12, 2019, so I tried it.

Overview

Poetry is a packaging and dependency management tool for Python.

As a similar tool, There are setuptools and Pipenv, but Poetry one can cover both functions.

I am attracted to the point that “one toml file is all I need”. Or rather, I thought it was wonderful.

version

poetry v1.0.5 (latest as of May 11, 2020)

Creating a project

https://python-poetry.org/docs/basic-usage/ https://python-poetry.org/docs/cli/#new

$ poetry new <project name>

After execution, the following folders will be created.

<project name>
├── pyproject.toml
├── README.rst
├── poetry_demo
│ └── __init__.py
└── tests
    ├── __init__.py
    └── test_poetry_demo.py

The pyproject.toml included in the created folder is the poetry configuration file. It is a file that replaces setup.py/setup.cfg/Pipfile, and the contents written in each file can be described in pyproject.toml. If you look it up, it will be replaced, so if you are interested, please see the link on the official page. https://python-poetry.org/docs/pyproject/

[tool.poetry]
name = "<project name>"
version = "0.1.0"
description = ""
authors = ["Sébastien Eustace <sebastien@eustace.io>"]

[tool.poetry.dependencies]
python = "*"

[tool.poetry.dev-dependencies]
pytest = "^3.4"

Add Library

https://python-poetry.org/docs/cli/#add

# Add dependent library
$ poetry add requests
# Add dependent library for development
$ poetry add --dev requests

You can also specify the version. It seems that it is not so different from requirements.txt/setup.py/Pipfile. https://python-poetry.org/docs/dependency-specification/

Create virtual environment, install in development mode

https://python-poetry.org/docs/cli/#install

$ poetry install

Command execution in virtual environment

https://python-poetry.org/docs/cli/#run https://python-poetry.org/docs/cli/#shell

# Run in one shot
$ poetry run python -m pip list
run with # shell
$ poetry shell

It seems that you can do things like npm-scripts by specifying the Python module you want to execute in pyproject.toml.

[tool.poetry.scripts]
my-script = "my_module:main"

$ poetry run my-script

Packaging

https://python-poetry.org/docs/cli/#build

$ poetry build

By default, both sdist/wheel are created. It is possible to use only one of them with the --format option.

Output to requirements.txt

https://python-poetry.org/docs/cli/#export

$ poetry export -f requirements.txt >requirements.txt

It seems that v1.0.2 only supports the requirements.txt format.

Cooperation with PyCharm

PyCharm (2019.03) recognizes pyproject.toml and detects the virtual environment.

Cooperation with Vitual Studio Code

Visual Studio Code doesn’t recognize it automatically.

It’s a hassle, but one of the following tasks is required.

  1. Specify the path where the virtual environment is saved in settings.json
  2. Create a poetry virtual environment in the automatically recognized .venv folder (* Personally, I prefer this)

https://github.com/microsoft/vscode-python/issues/8372#issuecomment-549924849

When setting in settings.json

This is a Mac example.

{
    "python.venvPath": "~/Library/Caches/pypoetry/virtualenvs"
}

When creating a virtual environment in the .venv folder

# Delete the Python environment that was created once
$ poetry env list
$ poetry env remove <environment name>
It is output to the setting value change --> poetry.toml so that it is saved in # .venv.
$ poetry config virutalenv.local true --local
# Create virtual environment --> Output to .venv folder.
$ poetry install

After that, if you reload the Python Interpreter settings of Visual Studio Code, you can select the .venv folder.
