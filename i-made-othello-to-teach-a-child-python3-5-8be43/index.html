Let’s talk about types

There are blood types, right? Until now, I have explained various things about variables, but I have not touched on types. I’ve explained that arrays are of class list(), and dictionary variables are also of class dict(). The data type in a program, like the blood type, represents the nature of the data.

keyboard input

First of all, it is a function that performs keyboard input. Display the string passed as an argument and wait for keyboard input. If you hit the Enter key blank or enter a value other than 1 to 8, you will return to keyboard input again. When the correct number is entered, convert it back to an integer with int(). The important values for this Othello class are the numbers 1 to 8 for X and Y. Even if it becomes a graphical Othello, even if the place is clicked with the mouse, it is converted to the X coordinate and the Y coordinate of 1 to 8 internally, and the function of the current class can be used as it is.

    # Input

The ot_changeturn()` function is a function that switches the attacker’s turn.

    # It's your turn
    def ot_changeturn(self):
        # Store offensive pieces
        self.ot_offdef ='●' if self.ot_offdef =='○' else'○'
        # Store the pieces opposite to the attackers
        self.ot_search ='●' if self.ot_offdef =='○' else'○'

There is a ternary operator in the program. It’s similar to writing an IF statement with an Excel macro, but the logic above and the logic below are exactly the same. By writing value when condition is met if condition else value when condition is not met, you can condense the if condition statement on one line. The ternary operator has the effect of making the program clear depending on the writing method, but if you use it forcibly, it will be a very unreadable source, so it is recommended to use it carefully. If you find it difficult to understand, you can replace it with the following source.

Function to think about the next operation of the Othello game

The ot_checkendofgame()` function is a program that considers whether the Othello game will end. See (2) of this series of articles for the operation.

What is being done in this is as follows.

  1. Use the ot_canplacemypeace() function to check if there is a place to put the attacking piece.
  2. If you cannot put the attacking piece, use ot_changeturn() to switch the attacking piece to the opponent’s turn.
  3. Use the ot_canplacemypeace() function to check if there is a place to put the attacking piece.
  4. If both decide that there is no place to put it, compare the numbers of ● and ○ to determine which one won and the draw, display the result and return 1 to tell that the game is over
  5. If there is no space for the attacker in 1. above, the message “Change to opponent’s turn” is displayed and -1 is returned to notify you.
  6. If the attacker determines that the piece can be placed as it is, 0 is returned and the fact is notified.

In this function, count() called myot.ot_bit.count('●') is a function that returns the number of data contained in the array of list() class.

    # Determine if it's over (change turn if you can't just put your hand in it)
    def ot_checkendofgame(self):
        #There is no place to put your own hand, and you can not put your opponent's hand either
        if 0> self.ot_canplacemypeace():
            self.ot_changeturn()
            if 0> self.ot_canplacemypeace():
                bc = myot.ot_bit.count('●')
                wc = myot.ot_bit.count('○')

                    bws = "● victory"
                elif (bc-wc) <0:
                    bws = "○ victory"
                else:
                    bws = "draw"
                print("{}. Thanks for your hard work!".format(bws))
                return 1
            else:
                print("There is no place to put {}. It will change to the opponent's turn!!".format(self.ot_search))
                return -1
        else:
            return 0

A function that finds all places that can be turned over

In the previous ot_checkendofgame(), it was determined whether the game was over, there was a place to put one’s hand, or if there was a place to put the other hand. It is the ot_canplacemypeace() function that thinks “Is there a place to put my hand?”

   # Find out if there is a place to put your hand
    def ot_canplacemypeace(self):
        for n in range(64):
            if self.ot_bit[n] !='・':
                continue
            for d in self.ot_direction:
                if 1 == self.ot_next_onepeace(int(n%8)+1,int(n/8)+1, d):
                    return 0
        # Come here if there is no place to put one
        return -1

What I am doing is as follows.1. Skip the place where the frame is already placed on the 8x8 board. 2. If there is a place where no frame is placed, convert that position from 0 to 63 into X and Y coordinates. The value obtained by adding 1 to the remainder divided by 8 is the relative X coordinate, and the value obtained by dividing 1 by 8 is the Y coordinate. 3. Look in all directions (8 directions) to see if you can put your hand on the converted X and Y coordinate positions (is there a place where you can flip the opponent’s frame?). In the for d in self.ot_direction: part, it is repeated for the number of dictionary variables, and the key string indicating the direction stored in the dictionary is stored in the variable d. 4. If a place to put the piece is found, 0 is returned and the fact is notified. 5. If there is no place to put a piece, -1 is returned to notify that fact.

Place a frame at the specified X and Y coordinates

As I talked about last time, if you put a piece in a designated place, you must judge whether it is a place where you can put it, and if it can put it, you must turn over the opponent’s piece. And if the place where you put it is a place that you should not put in the first place, you have to display that fact.

    # Put your hand on the board of Othello
    def ot_place(self, ot_x, ot_y):
        ot_change = False
        for d in self.ot_direction:
            if 1 == self.ot_next_onepeace(ot_x,ot_y,d):
                self.ot_peace(ot_x,ot_y,self.ot_offdef)
                self.ot_changepeace(ot_x,ot_y,d)
                ot_change = True
        # If there is no valid direction
        if not ot_change:
            print("Could not place {}".format(self.ot_offdef))
            return -1
        # Display the screen and change hands to the opponent's turn
        self.ot_display()
        self.ot_changeturn()
        return 0

This function is called when you try to put a piece in a predetermined place. The designated location is the coordinate where X and Y are specified by the value of 1 to 8, respectively. Now let’s explain the program of this function.

  1. Initialize ot_change with False as a flag variable indicating whether or not a frame has been placed. This is True when you can put a piece.
  2. Iterate for the number of ot_direction arrays (that is, for 8 directions). The variable d stores the key value of ot_direction.
  3. Call the ot_next_onepeace() function to see if the current direction can be flipped, and if so, how far can it be flipped. A return value of 1 from the function indicates that this direction is a tipping direction.
  4. Specify your hand as the third argument and call the ot_peace() function to put your frame at the specified coordinates. The ot_peace() function is also a function that returns one frame at the specified coordinates if the third argument is omitted. If the third argument is specified, the frame at the specified coordinates will be replaced with the specified character.
  5. Call the ot_changepeace() function and flip over to the place where it can be flipped over.
  6. If the frame can be placed, set the ot_change flag to True. If all eight directions cannot be turned over, it remains False.
  7. Check if the ot_change flag is False when exiting the iteration. In the case of False, the coordinates that cannot be placed are specified in the first place, so a message is displayed indicating that the location cannot be placed and the process returns.
  8. If the ot_change flag is True when exiting from the iteration, the overturning process has completed rewriting the ot_bit, which is the board data, so call the ot_display() function to redisplay the board. Then, call the ot_changeturn() function to change the turn of the attacker.
