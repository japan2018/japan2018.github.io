Virtual environment construction/settings

This item creates a virtual environment and installs Django in the virtual environment. (Roughly speaking, a virtual environment is a "pseudo OS". We will create web applications on this pseudo OS.)

First, create a directory on your desktop for this tutorial. The directory name can be anything, but this time it is "django_work".

Promptly enter this.
Open the terminal in Finder → Applications → Utilities → Terminal.

Did you see a black screen? When it comes out, enter the command below to enter the django_work directory. 

$ cd Desktop/django_work

Let's create a virtual environment here! Enter the following command in the terminal. 

$python3 -m venv django_venv

There should be a directory called django_venv in the django_work directory. This is a virtual environment.

The construction of the virtual environment is now complete. It was easy!

If this is the case, it's just a virtual environment. You have to start a virtual environment.

Type the following command while still in the django_work directory. Start the virtual environment. 

$ source django_venv/bin/activate

The virtual environment is now running. It was easy again!

In

, it's finally time to install Django. Enter and execute the following command line by line. 

(django_venv) $ pip install --upgrade pip

(django_venv) $ pip install django==2.2.6

Django is now installed in your virtual environment. By the way, the version of Django is 2.2.6.

Construction and setting of the virtual environment are complete.

Create project (directory)

project is a folder that contains setting files and applications to be created in the future.
The configuration file is automatically created by typing the following command.

Enter the following command while the virtual environment is running. The project name is "sakusaku_blog". 

(django_venv) $ django-admin startproject sakusaku_blog

The project is now created. If you look into the django_work directory on your desktop, you will see that a directory called "sakusaku_blog" has been created.

In

, enter the following command in the terminal to enter the project. 

(django_venv) $ cd sakusaku_blog

This completes the creation of the

project (directory). Leave the terminal in this state for a while. It will be used in the following item “Display Django page”.

Correspond to Japanese time and Japanese

When you execute the django-admin startproject command, the Django configuration file is automatically generated. The name of the settings file is "settings.py". Rewrite the relevant part in this settings.py to correspond to Japanese and Japan time.

Open the sakusaku_blog directory in the django_work directory with a text editor (VSCode, Atom, etc.). Then, I think that there is a "settings.py" file in the sakusaku_blog directory. Look at the contents of this file.

Around line 106, I think there is a code "LANGUAGE_CODE ='en-us'".
You can set the language to Japanese by rewriting "'en-us'" to "'ja-JP'".
Also, if you change the value of the code "TIME_ZONE ='UTC'" to "'Asia/Tokyo'" around line 108, you can set it to Japan time.

You have now changed to Japanese and Japan time!

The above is the correspondence between Japan time and Japanese.

Display Django page

Start the server and finally display the Django page!

Open the previous terminal. You should be in sakusaku_blog now.

Enter the following command in the terminal to start the server. 

(django_venv) $python3 manage.py runserver

The server has started! Try accessing http://localhost:8000!
You should see the Django default page.

Congratulations! With that, the Django environment has been built up!
