Let’s find the maximum value python (modified ver)

Nov 12, 2019 Python max fix 

This is a modified version of "[Look for the maximum value] python"(https://qiita.com/banana19980702/items/08b30cc81602917e8407)" that I posted the other day. Here are the bad points and changes in the previous code:

Bad points

・When the list elements are all negative, the maximum value becomes 0. ・Slow processing speed ##change point ・Set the initial value of the maximum value candidate to an appropriate element of the list ・From while sentence to for sentence

I tried refactoring based on two bad points

num = int(input("How many number values do you want to substitute?"))
value_list = [] # prepare an array to store numbers

for i in range(num):
    value = int(input("substitute a number"))
    value_list.append(value) # Add values to the prepared list

max_value = value_list[0] # Initialize with list elements

for i in range(num):
    if value_list[i] >max_value:
        max_value = value_list[i]

print(max_value) # print maximum value

Modified part

while i <num:
    value = int(input("substitute a number"))
    value_list.append(value) # Add values to the prepared list
    i += 1

for i in range(num):
    value = int(input("substitute a number"))
    value_list.append(value) # Add values to the prepared list

max_value = 0 # Initialize maximum value to 0
i = 0
while i <num:
    if value_list[i] >= max_value: # compare stored numbers in order
        max_value = value_list[i]
    i += 1

max_value = value_list[0] # Initialize with list elements

for i in range(num):
    if value_list[i] >max_value:
        max_value = value_list[i]

By setting the initial value to the input element, the maximum value will be output normally even if all the input elements are negative. Furthermore, by changing the while statement to the for statement, unnecessary initialization and addition processing has been eliminated.
