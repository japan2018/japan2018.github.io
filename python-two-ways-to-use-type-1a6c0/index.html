[Python] Two ways to use type()

Feb 3, 2020 Python 

## Introduction

When I read the code around Django’s Manager at work I found an unfamiliar usage of type().

GitHub link is here [https://github.com/django/django/blob/master/django/db/models/manager.py#L100-L108)


@classmethod
def from_queryset(cls, queryset_class, class_name=None):
    if class_name is None:
        class_name ='%sFrom%s' %(cls.__name__, queryset_class.__name__)
    return type(class_name, (cls,), {
        '_queryset_class': queryset_class,
        **cls._get_queryset_methods(queryset_class),
    })

Three arguments are passed to type…??? what’s this?

When I checked it, it was written in the Documentation of the Python standard library. (Yeah)

Although it looks almost like a quotation, I will briefly summarize it.

Conclusion

The behavior of __type() changes depending on the number of arguments. __

1. (Maybe) common usage

With only one argument, return the type of object. The return value is a type object, typically the same object returned by object.class. (Quoted from Python documentation)


>>> class Bar:
... pass

>>> class Foo(Bar):
... pass

Prepare class Bar and class Foo which inherits it.

At this time, the behavior of type() is as follows.

>>> foo = Foo()
>>> type(foo)
<class'__main__.Foo'>

>>> type(foo) is Foo
True

>>> type(foo) is Bar
False

Depending on the case, let’s use it properly with isinstance().

That was the story I knew.

2. Unexpected usage

class type(name, bases, dict)

With three arguments, returns a new type object. It is essentially a dynamic form of the class statement. (Quoted from Python documentation)

The documentation says that the following two statements create the same type object.


>>> class X:
... a = 1

>>> X = type('X', (object,), dict(a=1))

First argument: class name to generate Second argument: inherited class Third argument: object attributes

Has become.

I will try it.


>>> class Bar:
... a=1
...
...def print_hogefuga(self):
... print('hogefuga')
...

>>> Foo = type('Foo', (Bar,), dict(b=2))

>>> foo = Foo()
>>> foo.print_hogefuga()
hogefuga
>>> foo.a
1
>>> foo.b
2

I was able to create a FooClass that inherits BarClass like this.

Summary

The above is about the usage of type() that I didn’t know.

There are many surprising uses other than type().
