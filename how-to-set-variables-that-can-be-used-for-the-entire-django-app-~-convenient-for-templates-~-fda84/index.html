How to set variables that can be used for the entire Django app ~ convenient for templates ~

May 3, 2020 Python Django 

I think it is often useful to have constants (variables) that can be used throughout the site when creating apps with Django.

For example, the text that is displayed on the entire site, such as the app name and version.

Django uses the Context Processor to retrieve the constants you define. Of course, you can set more than one.

#Environment | Software | Version | |—|—| | Nginx | 1.16.1 | | uWSGI | 2.0.18 | | Python | 3.8.2 | | Django | 3.0.5 |

That’s the way it was in Django 3.0.x.

Variables handled by # Template are usually passed through the view, but… Template fetches and displays variables like {{variable}}, which is usually passed through the view. With this method, it is inefficient to write the process of passing variables for each application.

Normal reference method

def viewFunc(request):
  template_name = "template.html"
  context = {"variable" :"hello world!"}
  return render(request,template_name,context)

Create a view function in your app’s view and pass the value as context.

<p>{{variable name}}</p>

It will be displayed in the template like this.

#Set constants that can be used throughout the app By creating a Context Processor, you can pass values/objects to the entire application.

TEMPLATES = [
    {
        'BACKEND':'django.template.backends.django.DjangoTemplates',
        'DIRS': [
            os.path.join(BASE_DIR,'templates'),
        ],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                â€¦
                'mlapp.context_processors.constant_text',, # add this
            ],
        },
    },
]

The function we will add, constant_text, will get the template value.

def constant_text(request):
    return {
        'APP_NAME':'TEST APP',
    }

We have added APP_NAME to the return part. Create context_processors.py in the same directory as settings.py.

<p>{{ APP_NAME }}</p>

Now you can get the variables from the entire app and display them in the template. It’s that simple, so it’s easy.

If you want to set the value in # settings.py

APP_DESCRIPTION='This is a test app.'

Create a variable in settings.py and assign it a value.

from django.conf import settings

def constant_text(request):
    return {
        'APP_NAME':'TEST APP',
        'APP_DESCRIPTION': settings.APP_DESCRIPTION,
    }

Read the values from settings.py.

<h1>{{ APP_NAME }}</h2>
<p>{{ APP_DESCRIPTION }}</p>

By doing this, the value of settings.py can be handled.

It’s very easy.
