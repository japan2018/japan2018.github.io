Use: Django-MySQL

Dec 5, 2019 Python Django MySQL 

This article is the 5th day of [Django Advent Calendar 2019](https://qiita.com/advent-calendar/2019/django).

#Introduction Django’s ORM can issue sufficient queries for development by making full use of select_related, prefetch_related, Preftch, Q object, etc.

However, there are some things that cannot be done, and there are cases where you have to write raw SQL. For example, when multiple indexes are pasted, USE_IDNEX to specify the index you want to use, IGNORE_INDEX to specify the index you do not want to use, etc. In that case, you need to write raw SQL.

Django-MySQL

  • Django-MySQL

However, by using Django-MySQL, you can extend Django’s ORM without writing raw SQL and issue queries such as USE_IDNEX and IGNORE_INDEX.

setup

install

pip install django-mysql

Edit ### settings.py

Add django-mysql to INSTALLED_APPS


INSTALLED_APPS = (
    "django.contrib.admin",
    "django.contrib.auth",
    ...
    ...
    "django_mysql",
)

Enable DJANGO_MYSQL_REWRITE_QUERIES


DJANGO_MYSQL_REWRITE_QUERIES = True

sql_mode and innodb_strict_mode settings

DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "HOST": "127.0.0.1",
        "NAME": "hogehoge",
        "USER": "fugafuga",
        "OPTIONS": {
            "charset": "utf8mb4",
            "init_command": "SET sql_mode='STRICT_TRANS_TABLES', innodb_strict_mode=1",
        },
    }
}

Note that with this change, if STRICT mode is enabled, an error will occur when you try to insert/update data that exceeds max_length. When STRICT mode is not enabled, if you insert/update data that exceeds max_length, the excess will be truncated and the process will be performed without error, so if validation is not done properly there Needs attention.

It should be noted that this change is a setting when connecting from Django to MySQL, so you do not need to restart migration again.

Let’s use

model settings

Let’s create a model called hoge in the users application. Put index on both fuga and piyo of the field.

from django.db import models
from django_mysql.models import QuerySet


class Hoge(models.Model):
    fuga = models.IntegerField(db_index=True)
    piyo = models.IntegerField(db_index=True)

    objects = QuerySet.as_manager()

The difference from usual is that you use from django_mysql.models import QuerySet and use it as the ORM manager. By doing this, you can issue a query extended with Django-MySQL.

Try IGNORE INDEX

In the case of issuing a query without using fuga index, proceed as follows.

Hoge.objects.all().ignore_index("users_hoge_fuga_8a4fba19")

The SQL issued by this is as follows

SELECT
  `users_hoge`.`id`,
  `users_hoge`.`fuga`,
  `users_hoge`.`piyo`
FROM
  `users_hoge` IGNORE INDEX(`users_hoge_fuga_8a4fba19`);

The ignore index has been issued properly.
