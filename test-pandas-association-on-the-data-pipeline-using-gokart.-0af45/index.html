Confirm if the pd.Dataframe of input is empty

  • The code below causes an error when pd.Dataframe is empty.
  • I was supposed to write a unit test, but I often couldn’t pick it up.
class DataTask(gokart.TaskOnKart):
    task_namespace ='sample'

    def run(self):
        df = pd.DataFrame(dict(user=[1, 2], item=['a','b']))
        self.dump(df)


class TaskA(gokart.TaskOnKart):
    task_namespace ='sample'

    def requires(self):
        return DataTask()

    def run(self):
        df = self.load_data_frame()
        If #df is empty, `KeyError` is raised.
        df['user'] = df['user'].apply(lambda x: f'user_{x}')
        self.dump(df)


if __name__ =='__main__':
    gokart.run()
  • Run the following to see if it works with empty.
  • Add --test-run-pandas and --test-run-namespace=sample.

Make sure each column has the expected type when dumping

  • Check if the pandas type has been changed by an unintended operation.
  • Although you can check for each task, define a combination of column_name and type for each namespace.
  • Inherit gokart.PandasTypeConfig and define rules in namespace.
class SamplePandasTypeCheck(gokart.PandasTypeConfig):
    task_namespace ='sample'

    @classmethod
    def type_dict(cls) -> Dict[str, Any]:
        return {'user': int}


class DataTask(gokart.TaskOnKart):
    task_namespace ='sample'

    def run(self):
        df = pd.DataFrame(dict(user=[1, 2], item=['a','b']))
        self.dump(df)


class TaskA(gokart.TaskOnKart):
    task_namespace ='sample'

    def requires(self):
        return DataTask()

    def run(self):
        df = self.load_data_frame()
        The type of # user is changed from int to str.
        df['user'] = df['user'].apply(lambda x: f'user_{x}')
        self.dump(df)


if __name__ =='__main__':
    gokart.run()
  • If you run the task normally, the type will be checked automatically.
  • In this case, user is changed to str, so TaskA fails.
