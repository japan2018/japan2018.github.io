Go History

Unlike Python, which started out as a personal hobby, Go was born out of the global giant Google. Designed by three Google engineers, Robert Griesemer, Rob Pike, and Ken Thompson, in an experimental project to develop new programming (Go-en.wikipedia.org).

The design started in 2007, and Go1.0 was just released in 2012, which is still a short history of new entry in the programming language world.

Go is open source, so anyone can read the code.

$ git clone https://github.com/golang/go.git # (Note!) About 200MB
$ git log --reverse

If you execute the above, you can see the commit around 2008. (By the way, the first commit was the specification.)

Rob Pike, one of the three, can read a PDF of an unreleased slide presented at Stanford EE Computer Systems Colloquium in 2010 (Another Go at Language Design).

Looking at this PDF, it looks quite interesting, but roughly it solves the problems with Java and C++ (redundant code, compile time, large binaries, etc.). It seems that Go was invented in order to do **.

The Target: Go aims to combine the safety and performance of a statically typed compiled language with the expressiveness and convenience of a dynamically typed interpreted language.

Especially when translating this part,

**Go aims to connect the following two. **

  • ** Safety and performance of statically typed compiled languages **
  • ** Expressiveness and convenience of dynamically typed interpreter language **

It’s a pretty neat story, but Go is the language that really wants to do that.

The PDF also mentions Go’s features to solve other language challenges.

  • Fast compilation
  • Expressive type system
  • Concurrency (strong in concurrent programming)
  • Garbage collection
  • System programming language (direct access to hardware)
  • Be clear
  • Orthogonality (package is well independent of other packages)

You can set getter and setter using @property decorator. Now you can implement a private variable (something like that).

As for name, we didn’t implement the setter, so we can’t access it from outside (though we can).

class Programmer:
    def __init__(self, name, favorite_language):
        self.__name = name
        self.__favorite_language = favorite_language
        
    @property
    def name(self):
        return self.__name
   
    @property
    def favorite_language(self):
        return self.__favorite_language
    
    @favorite_language.setter
    def favorite_language(self, favorite_language):
        self.__favorite_language = favorite_language

    def teach_favorite_language(self, prog):
        prog.favorite_language = self.favorite_language


pythonista = Programmer("zawawahoge", "Python")
go_lover = Programmer("go_lover", "Go")

pythonista.teach_favorite_language(go_lover)

In Python, you want to make it a private variable by prefixing it with __.

Go “struct”

I tried implementing exactly the same functionality as implemented in Python in Go. I try to get as close as possible to the code I’m writing at work.

package main

type Language string

// Programmer is the programmer's interface (global variable)
type Programmer interface {
TeachFavoriteLanguage(target Programmer) error
SetFavoriteLanguage(lang Language) error
FavoriteLanguage() Language
}

// programmer implements Programmer interface (only visible from within package)
type programmer struct {
Programmer
name string
favoriteLanguage Language
}

// NewProgrammer returns a new Programmer
func NewProgrammer(name string, lang Language) Programmer {
return &programmer{
name: name,
favoriteLanguage: lang,
}
}

// getter
func (p *programmer) FavoriteLanguage() Language {
return p.favoriteLanguage
}

// setter
func (p *programmer) SetFavoriteLanguage(lang Language) error (
p.favoriteLanguage = lang
return nil
}

// Implementation of TeachFavoriteLanguage
func (p *programmer) TeachFavoriteLanguage(target Programmer) error {
err := target.SetFavoriteLanguage(p.FavoriteLanguage())
if err != nil {
return err
}
return nil
}

// In the main function, you can interact only with the Programmer interface
func main() {
pythonista := NewProgrammer("zawawahoge", Language("Python"))
go_lover := NewProgrammer("go_lover", Language("Go"))

err := pythonista.TeachFavoriteLanguage(go_lover)

if err != nil {
panic ("An error occurred during missionary work")
}
}

The error handling is completely different!

Python error handling try ~ exceptIn Python, you use the try ~ except syntax.

try:
    do_something()
    res = get_something()
except Exception as e:
    print(f"e={e}, type(e)={type(e)}")

Error handling of the try ~ catch system has a large overhead for error detection, and performance may drop even when no error occurs. It is hard to enclose everything in try, so it is necessary to enclose error-prone parts in try to write error handling. Then what do you do when an error occurs where you haven’t tried it? It will be a story.

Go error handling

Unlike other languages, Go does not handle error handling by try ~ catch or try ~ except, but judges whether it is a success or an error by the return value of type error.

The following error handling frequently appears in Go code.

If you’re unfamiliar, it’s probably feeling (early author).

err := DoSomething()
if err != nil {
        return errors.Errorf("failed to do something; err=%#v", err)
}

res, err := GetSomething()
if err != nil {
        return errors.Errorf("failed to get something; err %#v", err)
}

As in Go, error detection is possible only by checking if err is nil (None in Python), so the overhead is very small and there is a performance degradation due to error detection. I don’t need much.

On the downside, you’ll have to write error handling in almost every piece of code, so it’s somehow painful before you get used to it. However, by the time I’m used to it, I’m more relieved to be able to handle all the code errors! **
