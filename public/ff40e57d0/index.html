[Python] Update Python on Mac from 2 to 3

Nov 3, 2019 Python Mac homebrew pyenv Python3 

# Overview

This article is a step-by-step article on how a beginner himself developed a coupon distribution service for iPhone with RESTful API and swift. It was a very detouring implementation because we implemented it while examining the technical elements one by one.

Python is originally installed on the Mac, but the version is old as of OS 10.15. When actually writing Python, everyone seems to have been updated to 3 series. So I also updated to the 3 series, so I will record the procedure at that time.

Reference URL

https://prog-8.com/docs/python-env

Work environment

Mac OS 10.15 Catalina Fixed network line

Python update procedure

The recommended procedure for Mac environment is as follows.

  • Check the currently installed Python version
  • Install Homebrew
  • Install pyenv using Homebrew
  • set pyenv
  • Install Python 3 series using pyenv

Check the currently installed Python version

First of all, check the version of Python on your Mac just in case. Open the terminal and enter the following command to display the version.

$Python -V

The version installed on my Mac was version 2.7.16.

Homebrew installation

_It is recommended to update Xcode to the latest version in advance. In my case, when I tried to install Homebrew with Xcode 10.3 (latest version is 11.1 at the time of work), the update of “Command tools for Xcode 11” that runs during installation did not complete well, and Xcode latest version It was resolved by updating to. _

Homebrew is a tool that installs and manages middleware and other packages required for development. Only compatible with Mac. Now, I will install Homebrew.

Go to Homebrew site and copy the installation script displayed on the top page.

Paste the copied script into the terminal and run it. You may be asked to enter the Enter key or the Mac password on the way. Also, updates such as “Command tools for Xcode 11” may be executed. If the Homebrew installation is successful, you will see something like this in the terminal.

Make sure Homebrew is installed in the terminal. Execute the command to display the version, and if the version is displayed, the installation is successful.

$brew -v

Install pyenv using Homebrew

pyenv is a tool for installing Python. First, install pyenv using Homebrew.

$ brew install pyenv

Installation was completed in a minute or two. Make sure it is installed. Execute the command to check the version, and if it is displayed, the installation is successful.

$ pyenv -v

set pyenv

To install python with pyenv, you need to add settings to a file called “.bash_profile” on Mac. Add the configuration code using the command. Run three commands. Nothing is displayed if it can be set successfully with the command.

$ echo'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bash_profile
$ echo'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bash_profile
$ echo'eval "$(pyenv init -)"' >> ~/.bash_profile

Install Python using pyenv

First, check the version of python that can be installed. Execute the following command to display the list of versions.

$ pyenv install --list

This time I decided to install 3.7.5rc1. The reason for using 3.7 series is that it is a major version with minor updates in Python 3 series. I selected the latest 3.7.5rc1 for 3.7 series. The 3.8 series still has only 3.8.0 and 3.8-dev, so let’s take a look.

Install Python 3.7.5rc1 with the following command.

$ pyenv install 3.7.5rc1

Use the command to check if the installation was successful.

$ pyenv versions

Then, the installed Python version is displayed, but the one displayed as “system (~)” is marked with “*”. In this state, the installed Python is not enabled. Requires a global setting, otherwise the old version will be used in the system, the command is:

$ pyenv global 3.7.5rc1

Execute the pyenv versions command again to confirm that the settings have been reflected. If the globally set Python version is enabled, the globally set Python version will be displayed when the command to check the Python version is executed.

$python -V

This completes the Python update.
